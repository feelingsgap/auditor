정보시스템감리사 보안 영역 세부 공부 내역 (2026년 기준 Part 1 토픽 분류 체계 적용)

### 보안 영역 세부 공부 내역

- **정보보호 이론**

    - [기초 이론] 정보보호 3대 목표: 기밀성(Confidentiality), 무결성(Integrity), 가용성(Availability) — 각 목표를 침해하는 위협 유형과 대응 기술을 연계하여 이해
    - [기초 이론] 접근통제 모델: DAC(임의적 — 사용자 신원 기반), MAC(강제적 — 비밀 등급/허가 등급 매트릭스), RBAC(역할 기반 — 역할과 정보 객체 간 관계) 구분 [2022(103)]
    - [기초 이론] ACL(Access Control List): 객체에 접근통제 정보 저장, 주체 수 증가 시에도 효율적 관리 가능
    - [기초 이론] OTP(One-Time Password) 생성 방식: S/KEY(해시 체인), 시간 동기화, 도전-응답(서버 난수→클라이언트 OTP 생성), 이벤트 동기화 구별 [2021(102)]
    - [기초 이론] BCP(Business Continuity Planning) / DR(Disaster Recovery): 재해 복구 계획과 업무 연속성 계획의 차이
    - [암호학] 대칭키 암호: AES(SPN 구조, 블록 128비트, 키 128/192/256비트, SubBytes/ShiftRows/MixColumns/AddRoundKey), DES, SEED, ARIA [2022(102)]
    - [암호학] 블록 암호 운용 모드: ECB, CBC(이전 암호문 XOR 후 암호화), CTR, GCM(인증 태그 생성, 병렬 처리 가능) — GCM은 기밀성 제공하나 메시지 무결성을 별도의 MAC 값(GMAC)으로 보장 [2024(106)], [2025(104)]
    - [암호학] 공개키 암호: RSA(소인수분해 어려움, 공개키(E,N)로 암호화, 개인키(D,N)로 복호화), ECC(이산대수 기반, 짧은 키로 RSA급 보안), ElGamal [2021(104)], [2024(115)], [2025(101)]
    - [암호학] 해시 함수: SHA-3(스펀지 구조, SHA1/SHA2와 다름, SHA3-224/256/384/512, 의사난수/스트림 암호로도 활용), 암호학적 해시의 충돌 저항성·역상 저항성 [2021(106)], [2025(102)]
    - [암호학] 전자서명: 개인키로 서명 → 공개키로 검증, 송신자 인증·무결성·부인 방지 제공, 기밀성은 제공하지 않음, 복사 사용 불가(디지털 정보이지만 재사용 방지 포함) [2021(103)], [2022(101)]
    - [암호학] MAC / HMAC: 공유 비밀키로 메시지 인증코드 생성, 무결성·송신자 인증 제공, 부인 방지는 제공하지 않음 (제3자 증명 불가) [2022(104)], [2024(116)]
    - [암호학] 하이브리드 암호 시스템: 대칭키(세션키)로 메시지 암호화 + 수신자 공개키로 세션키 암호화 전달, PGP에서 활용 [2024(117)]
    - [암호학] PBE(Password Based Encryption): 패스워드 → 솔트(salt) 추가 → 키 스트레칭 → KEK(Key Encryption Key) 생성 → CEK(Contents Encryption Key) 암호화, 솔트로 사전 공격 방어 [2021(117)]
    - [암호학] 부채널(Side Channel) 공격: 전력 분석, 타이밍 공격, 전자기 분석 — 암호 알고리즘 자체가 아닌 구현·하드웨어에서 정보 추출, 선택 암호문 공격은 부채널 아님 [2023(101)]

- **네트워크보안**

    - [보안프로토콜] SSL/TLS: 핸드셰이크(ClientHello→ServerHello→인증서→키교환→ChangeCipherSpec→Finished), 레코드 프로토콜, HTTPS는 브라우저-서버 간 SSL/TLS, HTTP 프록시 경유 HTTPS는 CONNECT 메서드로 터널 형성 [2021(115)], [2025(105)]
    - [보안프로토콜] IPSec VPN: AH(인증 헤더), ESP(암호화+인증), 터널 모드(IP 패킷 전체 암호화·캡슐화)·전송 모드(페이로드만) [2023(104)]
    - [보안프로토콜] VPN 터널링 프로토콜: L2F(CISCO, 링크 계층 캡슐화) → L2TP(L2F+PPTP 장점, Remote Dial-Up 사설망 연결) → PPTP(링크 계층 PPP에 보안 추가, 네트워크 계층 적용); L2TP는 IPSec과, SSTP는 TLS와 함께 사용 [2023(104)], [2024(112)], [2025(107)]
    - [보안프로토콜] WLAN 보안: WPA2(CCMP/AES 기반, AP 접속 단말마다 별도 암호화 키 재사용), WPA-EAP(사용자별 인증, 세션마다 고유 암호화 키) [2021(105)]
    - [보안프로토콜] PGP / S/MIME: 이메일 암호화·서명, PGP(하이브리드 암호+전자서명), S/MIME(전자결제 등에 활용) [2023(102)]
    - [보안프로토콜] 인증 프로토콜: SSO(Single Sign On) — SSO 서버가 인증 후 접근 권한 위임, Kerberos(대표 구현), Single Point of Failure 존재; 지식·소유·생체·이중 인증 방식 구별 [2024(114)], [2024(118)]
    - [네트워크] 방화벽 패킷 필터링: ACL 규칙 테이블(From/Service/To/Action), 포트 번호와 프로토콜 기반 Allow/Deny 판단 [2022(107)]
    - [네트워크] IDS/IPS — Snort: alert/drop 지시어, tcp/udp/icmp 프로토콜, 방향(→), 포트 지정, msg 옵션 [2024(110)], [2025(103)]
    - [네트워크] IP 필터링: Ingress/Egress 필터링, Unicast RPF, Blackhole 라우팅 — 비정상 출발지 IP 차단 [2025(103)]
    - [네트워크] NAC(Network Access Control): MAC 주소 기반 엔드포인트 접근 제어, 회사 내부망 접속 통제 [2025(108)]
    - [네트워크] VLAN: 브로드캐스트 도메인 분리, ARP Request 격리, 목적별 네트워크 분리로 보안 수준 향상 [2024(109)]
    - [네트워크] ARP Poisoning 방지: 정적 ARP 테이블 사용, ARP Request/Reply 검증, 외부 PC 로컬 접근 차단 [2022(110)]

- **시스템보안**

    - [보안모델] BLP(Bell-LaPadula): 기밀성 보호 모델 — 단순보안규칙(낮은→높은 등급 읽기 가능), 스타보안규칙(높은→낮은 등급 쓰기 금지), 강한 스타보안규칙(동일 등급만 읽기/쓰기) [2023(116)]
    - [보안모델] Biba: 무결성 보호 모델 — BLP의 무결성 버전 (높은 등급이 낮은 등급에 쓰기 금지, 낮은 등급이 높은 등급 읽기 금지)
    - [보안솔루션] DLP(Data Leakage Prevention): 내부 정보 외부 유출 탐지·통제, 블루투스·테더링 통신 채널 차단 [2022(119)]
    - [보안솔루션] DRM(Digital Rights Management): 디지털 자산 암호화·접근 통제, Enterprise DRM(EDRM), 인가된 사용자만 접근, 사용자·프로세스별 권한 관리 가능 [2023(113)]
    - [보안솔루션] DB 암호화 방식: Plug-In 방식(DBMS 독립적), API 방식(응용 프로그램 수준 암호화) [2025(112)]
    - [보안솔루션] DB 보안 요구사항: 논리적 일관성, 기밀성, 사용자 통계집합 추적 불가(Inference 공격 방지), 트랜잭션 병행처리 무결성 [2021(107)]
    - [보안솔루션] False Positive / False Negative: 정상 접근을 공격으로 오인 → 가용성 저하(False Positive), 공격을 정상으로 오인 → 탐지 실패(False Negative) [2024(119)]
    - [OS 보안] Linux iptables: `iptables -A INPUT -p icmp --icmp-type echo-request -j DROP` (Ping of Death 방어), `iptables -A INPUT <IP> -j DROP` (특정 IP 차단) [2021(108)], [2022(109)]
    - [OS 보안] Linux root 원격 접속 금지: `/etc/pam.d/login` + `/etc/securetty` 파일 수정으로 원격 root 접속 차단 [2022(112)]
    - [OS 보안] 모바일 OS 보안: iOS(샌드박스, 원격 명령 실행 불허), Android(샌드박스, 앱 간 데이터 전달 자유로움, 코드 서명 필수, 일반 사용자 권한으로 실행) [2021(109)]

- **응용보안**

    - [개발보안] XSS(Cross-Site Scripting): Stored XSS(DB 저장 후 실행), Reflective XSS(URL에 스크립트 노출, 클릭 유도), DOM XSS — 입력값 검증·인코딩·CSP로 방어 [2021(113)]
    - [개발보안] SQL Injection: Prepared Statement, Stored Procedure로 방어; 입력값 검증·파라미터화 쿼리 [2025(111)]
    - [개발보안] 파일 업로드 취약점: 실행 속성 제거, 화이트리스트(허용 확장자만), 저장 경로를 웹 루트 외부로, 임의의 파일명으로 저장 [2024(104)]
    - [개발보안] 버퍼/힙 오버플로우: C코드 `scanf("%99s")` 이후 `strcpy` → 힙 오버플로우, EBP(스택 베이스 포인터) 오버라이트로 제어권 탈취 [2021(119)], [2022(115)], [2024(105)]
    - [개발보안] 포맷 스트링, 경쟁 조건(Race Condition), 경로 탐색(Path Traversal), 하드코딩 비밀번호, 널 포인터 역참조 등 SW 취약점 유형 [2025(115)]
    - [개발보안] OWASP TOP 10: Injection, XSS, CSRF, Broken Access Control, Security Misconfiguration 등; 웹 서버 보안 강화(WebDAV 비활성화, 최소 권한, 디렉토리 목록 비공개) [2023(111)]
    - [개발보안] SW 보안 취약점 점검 도구: SpotBugs(Java 정적 분석, XSS/HTTP 응답 분할/Path Traversal/하드코딩 탐지), Burp Suite/Paros/WebScarab(웹 취약점 진단) vs Hydra(패스워드 크래킹 — 웹 취약점 진단 도구 아님) [2023(114)], [2025(116)]
    - [공격기법] DoS / DDoS: SYN Flooding(3-Way Handshake 미완료 → TCB 큐 포화), Smurf(IP 위장 + ICMP 브로드캐스트), TearDrop(IP 패킷 시퀀스 중첩 조작), Bonk(동일 출발지/목적지 IP) [2023(105)]
    - [공격기법] 스푸핑: ARP 스푸핑(IP-MAC 매핑 조작, ARP Redirect/Switch Jamming), IP 스푸핑, DNS 스푸핑(캐시 포이즈닝), DNS Amplification [2024(113)], [2025(106)]
    - [공격기법] TCP 세션 하이재킹: 시퀀스 넘버 예측, 패킷 유실·재전송 급증, 세션 리셋 탐지로 식별; SYN Storm은 탐지 방법 아님 [2021(111)]
    - [공격기법] 네트워크 기반 공격: Evil Twin AP(강한 신호의 가짜 AP로 사용자 유인), Switch Jamming(MAC 테이블 오버플로우), ARP Redirect [2022(105)], [2022(108)]
    - [공격기법] 악성코드: 트로이 목마(정상 프로그램 속 악성 코드, 자가복제 안 함), Dropper(압축/암호화된 악성코드 생성), Spyware(정보 탈취 전송), PUP(사용자 동의 설치, 성능 저하) [2022(111)], [2025(109)]
    - [공격기법] 살라미 공격(Salami Attack): 금융 프로그램에서 미세 금액을 지속적으로 탈취 [2021(116)]
    - [공격기법] 블루투스 공격: BlueSnarfing(펌웨어 취약점으로 데이터 무단 접근), BlueBorne(블루투스 취약점 원격 코드 실행), Bluejacking(스팸 메시지 전송), Blueprinting(기기 정보 수집) [2023(106)]
    - [공격기법] finger 명령어: 외부 시스템 등록 사용자 정보 확인, 권한 탈취 전 사전조사(Reconnaissance) 단계 [2024(101)]
    - [침해대응] 디지털 포렌식 원칙: 신속성, 재현성, 무결성(수집 증거 위변조 불가), 연계보관성(각 단계별 책임자 명확), 정당성(적법절차) — 정당성은 수사관 동행 시 위법 증거도 인정 안 됨 [2023(115)]
    - [침해대응] 파일 슬랙(File Slack): 파일 시스템의 클러스터 끝과 파일 데이터 끝 사이 빈 공간, 삭제된 파일 내용 복구 가능 → 포렌식 분석 대상 [2023(109)]
    - [침해대응] 리버스 엔지니어링: 장치/시스템 구조 분석으로 동작 원리 이해, 보안 문제 검토; OllyDBG(실행 파일 리버스 엔지니어링/디버거)와 Wireshark(패킷 캡처) 구별 [2021(101)], [2023(110)]
    - [침해대응] 보안 취약점 점검 도구: netstat(네트워크 상태 확인), tcpdump(패킷 출력), nmap(포트 스캔·OS 탐지), Wireshark(패킷 캡처·분석) [2021(110)]

- **표준,법령,지침**

    - [정보보안 표준] ISMS-P 위험관리 절차: 위험 식별 → 위험 분석 → 위험 대응 계획 → 위험 모니터링 및 통제 [2022(113)], [2023(120)]
    - [정보보안 표준] 정보자산 식별: 조직의 업무 특성에 따라 **관리체계 범위 내 모든 중요 정보자산**을 식별하고 중요도 산정 (일부 중요 정보자산만 식별하는 것은 오류) [2023(120)]
    - [정보보안 표준] PIA(개인정보 영향평가): 대규모 개인정보 처리 시스템 구축·변경 시 수행, 위험 분석 및 개선 방안 마련 [2025(117)]
    - [정보보안 표준] 보안 거버넌스: 조직이 적합한 보안 정책을 수립하고 보안 관련 조직을 구성하여 일련의 보안 활동을 수행하는 것 — ISO 27014(보안 거버넌스), ISO 27001(ISMS) [2024(107)]
    - [정보보안 표준] 비식별화 기법 (DB 개인정보 보호): k-익명성(k-Anonymity), ℓ-다양성(ℓ-diversity), t-근접성(t-closeness), m-군집화(m-clustering은 비식별화 기법 아님) [2021(114)], [2023(108)]
    - [법령/지침] 개인정보보호법 원칙: 목적 명확화·최소 수집, 정확성·완전성·최신성, 사생활 침해 최소화, 처리 방침 공개·열람 청구권 보장, 익명처리 우선 → 불가 시 가명처리 [2021(120)], [2023(119)]
    - [법령/지침] 개인정보 제3자 제공(법 제17조): 정보주체 동의 원칙, 제공 받는 자·목적·항목·기간 고지 [2025(118)]
    - [법령/지침] 개인정보 유출 시 조치(신고·통지 사항): 유출된 개인정보 항목, 유출 시점과 경위, 유출로 인한 피해 최소화 방법, 대응 조치 및 피해 구제 절차 — 피해가 발생한 경우 신고 접수 창구 연락처 등 [2024(111)]
    - [법령/지침] 가명정보 처리 특례(법 제3절): 통계작성·과학적 연구·공익 기록 목적 동의 없이 처리 가능, 특정 개인 식별 정보 생성 시 즉시 파기·처리 중단, 가명정보 처리 목적·항목·이용내역·제3자 제공 기록 관리 [2022(114)], [2024(108)]
    - [법령/지침] 안전성 확보조치 기준(암호화): 주민등록번호·비밀번호 → AES-256, 바이오정보 → SHA-256, 일반 개인정보 → AES-128 이상 [2021(118)]
    - [법령/지침] OECD 개인정보보호 원칙: 수집 제한, 정보 품질, 목적 명확화, 이용 제한, 안전 보호, 공개, 개인 참가, 책임 [2025(119)]

- **기타(신기술)**

    - [블록체인] 특성: 암호기술로 인증·검증 가능한 트랜잭션, 참여 노드 합의를 통한 신뢰 형성, 분산 저장으로 임의 조작 어려움, 스마트 계약(블록체인 DB에 포함된 비즈니스 규약) — 블록에 기록된 민감 정보의 수정은 불가 [2021(112)], [2022(117)]
    - [블록체인] 하드포크 vs 소프트포크: 소프트포크는 후방 호환성 지원(기존 클라이언트 업그레이드 불필요), 하드포크는 후방 호환성 없음(모든 클라이언트 필수 업그레이드) [2023(107)]
    - [블록체인] 블록체인 대상 보안 공격: 이중 지불(Double Spending Attack), 허위 정보 전파(Eclipse Attack), 재진입 공격(Reentrancy Attack) — 스머프(Smurf) 공격은 블록체인 공격 유형 아님 [2024(120)]
    - [AI/딥페이크] 딥페이크(Deepfake): AI를 이용한 가짜 이미지·영상 제작 기술, 가짜 뉴스·불법 콘텐츠 제작 악용, 사회·정치 분야 영향 — 부작용 없이 사회에 도움이 된다는 설명은 옳지 않음 [2022(116)]
    - [AI 보안] AI Inversion Attack: AI에 수많은 쿼리 수행 → 결과 분석 → 학습에 사용된 데이터 추출 [2023(112)]
    - [클라우드 보안] 주요 위협: 제3자 위탁 저장으로 관리 부주의 유출, DDoS/APT 집중 공격 목표, 가상화 자원 재분배 문제, IaaS 가상머신 간 도청·해킹 — 논리적 자원의 물리적 통합은 가상화 문제이지 물리적 통합과 재분배가 아님 [2022(118)]
    - [Zero Trust] 개념: 내부 네트워크도 이미 침해된 것으로 간주, 불확실성 최소화 설계, 보안 위협이 언제 어디서든 발생 가능 전제, 신뢰도 평가·인증·권한부여 갖추지 않은 접근 차단 — 경계 기반 보안체계 강화가 아닌 Zero Trust로의 전환이 핵심 [2024(102)]
    - [IoT 보안] De-synchronization 공격: 잘못된 시간 정보를 IoT 디바이스에 지속 전송하여 시간 교정에 자원을 소모시키는 서비스 거부 공격 [2025(110)]
