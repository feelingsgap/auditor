- **데이터베이스 기본**
    - [DB이론] DBMS의 역할과 3계층 스키마(외부/개념/내부 스키마) 개념을 이해해야 합니다.
    - [DB이론] 데이터베이스 구축 공정: 메타데이터 입력항목 협의, 한자 및 다국어 지원 협의, 표준안 수립 및 작업지침 작성 등의 태스크가 **계획수립** 세그먼트에 해당함을 알아야 합니다 [2022(51)].
    - [DB이론] 릴레이션이 가질 수 있는 최대 투플 개수: 기본키가 아닌 속성들의 도메인 크기 곱으로 계산합니다 [2025(56)].
    - [데이터저장] 고정 길이 레코드 방식: 레코드 길이, 블록 크기, 블록 헤더 길이를 이용하여 블록당 저장 가능한 레코드 수 및 남는 바이트를 계산하는 능력이 요구됩니다 [2023(52)].
    - [데이터저장] 비순서 히프파일(Heap File): 레코드 개수·길이·블록 크기 등이 주어졌을 때 평균 검색 비용을 계산할 수 있어야 합니다 [2022(62)].
    - [데이터저장] B-트리(B-tree): 삽입 시 노드 분할 발생, 루트 공간 사용률, 높이 증가 가능성 등을 이해해야 합니다 [2021(58), 2024(73)]. B+-트리의 노드 구조와 키 값 분포 특성(내부/리프 노드 역할)을 파악해야 합니다 [2025(63)].
    - [데이터저장] 확장 가능 해싱(Extendible Hashing): 전역 깊이(global depth)와 지역 깊이(local depth)의 개념을 이해하고, 특정 레코드 삽입·삭제 시 변경되는 깊이 값과 디렉터리/버킷 상태를 계산할 수 있어야 합니다 [2021(66), 2024(71), 2025(64)].
    - [데이터저장] 인덱스 ROWID 구성 요소(오브젝트 번호, 데이터 파일 번호, 블록 번호)와 ROWID를 이용한 테이블 블록 읽기 메커니즘을 파악해야 합니다 [2024(59)].

- **데이터베이스 설계**
    - [함수종속/정규화] 함수적 종속성(Functional Dependency): 주어진 릴레이션의 함수적 종속성을 파악하고 **후보 키**를 식별할 수 있어야 합니다 [2021(53), 2025(51)]. 항상 성립하는 추론 규칙(암스트롱 공리)을 식별할 수 있어야 합니다 [2022(54), 2023(58)].
    - [함수종속/정규화] 정규형(Normal Form): 1NF(도메인 원자성)·2NF(부분 함수 종속 제거)·3NF(이행 함수 종속 제거)·BCNF(결정자가 모두 후보키)의 조건을 이해하고 정규화 과정을 수행할 수 있어야 합니다 [2021(57), 2022(55), 2023(61), 2024(57), 2025(54)].
    - [함수종속/정규화] 다치 종속(Multivalued Dependency)과 제4정규형(4NF): 다치 종속이 존재하는 릴레이션을 4NF로 분해하는 방법을 이해해야 합니다 [2024(58)].
    - [함수종속/정규화] 무손실 분해(Lossless Decomposition): 릴레이션을 분해 후 자연 조인으로 원래 릴레이션 복구 가능 여부를 판단해야 합니다 [2021(74), 2025(52)].
    - [모델링] 개체-관계 모델(E-R Model): 약한 엔티티 타입(key 형성에 불충분)·범주(category/union type)·집단화(aggregation) 개념을 이해하고 특징을 파악해야 합니다 [2022(53), 2023(55), 2024(64)].
    - [모델링] ER 다이어그램을 관계 스키마로 변환: 1:1·1:N·N:M 관계 처리 규칙과 생성되는 릴레이션 개수를 파악해야 합니다 [2021(52), 2021(72), 2023(56), 2024(56), 2025(55)].
    - [모델링] 관계 데이터 모델의 제약조건: 도메인 무결성·엔티티 무결성·참조 무결성 제약조건의 개념을 이해하고, 투플 삽입/삭제 시 위배 가능한 제약조건을 식별할 수 있어야 합니다 [2022(52), 2022(56)].
    - [모델링] 최소 카디널리티(부분/전체 참여): 1:N 관계에서 부분 참여·전체 참여 조건을 DDL로 표현하는 방법(NOT NULL 외래키, 트리거)을 이해해야 합니다 [2024(69)].

- **데이터베이스 언어**
    - [데이터연산] 관계대수 연산(셀렉션·프로젝션·조인·디비전·집합연산): "모든 과목에 수강 중인 학생을 검색"과 같은 디비전(division) 연산의 의미와 기본 연산자(프로젝션·차집합·곱집합)로 표현하는 방법을 이해해야 합니다 [2021(59), 2025(60), 2025(61)].
    - [데이터연산] 연산 결과의 카디널리티: theta join·natural join·외부 조인·카르테시안 곱 등 다양한 연산 결과의 카디널리티 최솟값·최댓값·합을 계산할 수 있어야 합니다 [2021(68), 2022(59), 2023(54), 2024(52)].
    - [데이터연산] 관계해석 및 동등 규칙(Equivalence Rule): 질의 최적화에서 관계 대수의 동등 규칙을 이해하여 식을 변환할 수 있어야 합니다 [2024(67)].
    - [데이터연산] 외부 조인(Outer Join): 자연 조인·세미 조인·외부 조인의 차이와 완전 외부 조인(full outer join)의 결과를 파악해야 합니다 [2021(61), 2023(59), 2025(53)].
    - [SQL] SQL 함수: `COUNT(*)`은 중복 포함 투플 수, `COUNT(속성명)`은 NULL 제외, `COUNT(DISTINCT 속성명)`은 중복 제거 후 카운트 동작을 이해해야 합니다 [2021(54)].
    - [SQL] VIEW(뷰): 뷰 생성(CREATE VIEW)과 뷰의 갱신 가능 조건(두 개 이상 테이블 관련·DISTINCT/GROUP BY 사용·기본키 미포함·상수/산술연산자 사용 → 변경 불가)을 이해해야 합니다 [2021(55), 2022(68), 2023(57), 2025(65)].
    - [SQL] 조인: INNER JOIN·LEFT/RIGHT OUTER JOIN·FULL OUTER JOIN 결과로 생성되는 레코드 개수를 파악할 수 있어야 합니다 [2022(57), 2022(58), 2024(60), 2025(53)].
    - [SQL] DDL 참조 무결성: `FOREIGN KEY ... REFERENCES ... ON DELETE (SET NULL/SET DEFAULT/CASCADE)`의 동작을 이해하고 특정 DML 실행 후 릴레이션 상태를 예측할 수 있어야 합니다 [2021(65), 2023(60), 2024(55), 2025(62)].
    - [SQL] 부속질의(Subquery): EXISTS·NOT EXISTS·IN·NOT IN·ALL·ANY 등을 활용한 질의를 정확히 해석하고 작성할 수 있어야 합니다. NOT IN과 NULL 관계 주의 [2021(67), 2023(65), 2024(68), 2024(70), 2025(58)].
    - [SQL] 집계 함수·GROUP BY·HAVING: ORDER BY·GROUP BY·HAVING절을 포함한 질의 결과와 카디널리티를 예측할 수 있어야 합니다 [2022(61), 2024(60), 2025(57), 2025(59)].
    - [SQL] 자체 조인(Self Join)·다중 테이블 조인: 직원-관리자, 3개 릴레이션 조인 등 복잡한 조인 질의 결과를 파악해야 합니다 [2021(63), 2022(64), 2023(62), 2023(63)].
    - [SQL] 트리거(Trigger)와 저장 프로시저(Stored Procedure): 입력 인자/결과 반환 여부, 실행 시점(자동 vs 명시 호출), 할당 대상 등 특성을 구분할 수 있어야 합니다 [2024(65)].
    - [SQL] 삽입 SQL·JDBC: JDBC를 이용한 데이터베이스 프로그램 작성 순서(SQLPrepare→SQLBindParameter→SQLExecute→SQLBindCol)를 이해해야 합니다 [2025(72)].

- **데이터베이스 관리**
    - [DB성능] 조인 구현 방법: 중첩 루프 조인(Nested Loop Join)·정렬-합병 조인(Sort-Merge Join)·분할-해시 조인(Partition-Hash Join)의 특징과 적용 조건(정렬 여부·인덱스 존재 여부)을 이해해야 합니다 [2021(56), 2024(53)].
    - [DB성능] 질의 최적화 동등 규칙: 셀렉션 분배·프로젝션 분배·조인 교환·조인 결합 등 관계대수 동등 규칙을 이해해야 합니다 [2024(67)].
    - [백업/회복] Checkpoint 회복: 시스템 장애 후 REDO(재수행)·UNDO(취소) 대상 트랜잭션 판별 규칙을 이해해야 합니다. 검사시점 이전 완료 트랜잭션은 회복 불필요, 수행 중 트랜잭션은 UNDO [2021(71), 2022(63), 2023(64)].
    - [백업/회복] 지연 갱신(Deferred Update)·즉시 갱신(Immediate Update): 커밋 이전 실패 시 지연 갱신은 UNDO 불필요, 즉시 갱신은 UNDO 필요 [2022(66)].
    - [백업/회복] WAL(Write-Ahead Logging): AFIM(After Image)은 REDO에 대비, BFIM(Before Image)은 UNDO에 대비하기 위함. 즉시 변경 로그 기록에서 복구 후 데이터 값을 파악해야 합니다 [2024(74), 2025(70)].
    - [병행제어] 트랜잭션 동시성 문제: 오손 데이터 읽기(dirty read)·갱신 손실(lost update)·반복불가 읽기(unrepeatable read)·유령 데이터 읽기(phantom read) 등의 현상을 식별할 수 있어야 합니다 [2021(64), 2023(66)].
    - [병행제어] 로킹(Locking): 로킹 단위가 작아지면 동시성 수준 높아짐, 로킹 오버헤드 증가. 데이터베이스·파일·레코드 등이 단위가 될 수 있습니다 [2023(67)].
    - [병행제어] 엄격한 2단계 로킹(strict 2PLP): 모든 독점 로크는 트랜잭션 완료 시까지 유지. 낙관적 로킹(트랜잭션 완료 후 로크 획득)과 비관적 로킹 장단점 비교 [2023(68), 2024(54)].
    - [병행제어] 다중 단위 로킹(Multiple Granularity Locking): 의도 로크(IS·IX·SIX)의 개념과 로크 양립성 행렬을 이해해야 합니다 [2025(66)].
    - [DB보안] 필수 접근 제어(MAC): 각 데이터 객체에 비밀 등급, 사용자에게 허가 등급을 부여하여 접근을 통제합니다 [2022(65)].
    - [DB보안] GRANT/REVOKE: `GRANT SELECT, UPDATE ON 테이블 TO 사용자 WITH GRANT OPTION` 형식으로 권한 부여 및 재부여 권한을 제어합니다 [2023(70)].

- **데이터베이스 응용**
    - [정보분석] 데이터 품질 관리: ISO 8000의 데이터 품질 기준(완전성·유효성·일관성·정확성·적시성 등)과 각 오류 유형의 정의를 이해해야 합니다 [2021(51), 2023(51)].
    - [정보분석] 연관 규칙 마이닝(Association Rule Mining): Apriori 알고리즘으로 빈발 항목집합(frequent itemset) 및 신뢰도(confidence) 계산. 반단조(anti-monotone) 성질(상위 집합이 비빈발이면 하위도 비빈발)을 이해해야 합니다 [2021(75), 2022(67), 2022(73), 2025(68)].
    - [정보분석] 나이브 베이지안 분류기: 조건부 확률(사전 확률·우도·사후 확률)을 이용한 P(C|X) 계산 [2021(69), 2025(69)].
    - [정보분석] TF-IDF: TF(단어가 해당 문서에 등장하는 빈도)와 IDF(단어가 전체 문서에 등장하는 역수)의 개념을 이해하고 값을 계산할 수 있어야 합니다 [2022(71)].
    - [정보분석] 의사결정 트리(Decision Tree): Gini 불순도·Gini Index·분류 오류(classification error) 계산 방법을 알아야 합니다 [2022(70), 2025(74)].
    - [정보분석] 혼돈행렬(Confusion Matrix): 정확도(accuracy)·정밀도(precision)·재현율(recall)·특이도(specificity)·F-score 등 성능 척도를 혼돈행렬로부터 계산할 수 있어야 합니다 [2022(74)].
    - [정보분석] 검색 엔진 평가 척도: 리콜(recall)·정확도(precision)·평균 정확도(average precision)·F-스코어(정확도와 재현율의 조화 평균) [2024(62)].
    - [정보분석] k-Means 군집화(Clustering): 초기 중심점 선택에 따라 결과 달라짐, 이상치에 취약, 구형이 아닌 군집에 불리. 유사도(점과 점 사이 거리) 계산 횟수 공식을 알아야 합니다 [2023(75), 2024(75)].
    - [정보분석] 멀티미디어 데이터 검색 기법: 수학 함수로 저장된 데이터와 질의 조건 간 유사도 계산으로 검색하는 **매칭 기법(matching technique)** [2023(73)].
    - [DB응용] 분산 데이터베이스 투명성: 위치 투명성·분할 투명성·중복 투명성·장애 투명성·병행 투명성 개념을 이해해야 합니다 [2021(75), 2022(75), 2023(71)].
    - [DB응용] ORDB/OODB: 객체-관계 데이터베이스에서 OID(Object Identifier)의 역할 [2021(57)].
    - [DB응용] NoSQL 데이터 모델 유형: Key-Value·문서 기반(Document, JSON, MongoDB)·컬럼 기반·그래프 기반의 특징과 차이를 이해해야 합니다 [2021(62), 2024(51)].
    - [DB응용] NoSQL 특징: 높은 확장성(노드 추가)·고장 허용(복제)·샤딩을 통한 부하 분산. CAP 이론(Consistency·Availability·Partition Tolerance)과 NoSQL의 궁극적 일관성(eventual consistency) [2022(72), 2023(72), 2025(73)].
    - [DB응용] 관계형 DB vs NoSQL 비교: 스키마 유연성·클러스터 적합성·단순 검색·트랜잭션 일관성 측면 [2023(74)].
    - [DB응용] HDFS(Hadoop Distributed File System): NameNode(메타정보 관리)·DataNode(실제 데이터 블록 저장) 구조와 읽기/쓰기 성능 특성을 이해해야 합니다 [2023(69), 2025(67)].
    - [DB응용] Apache Spark: RDD(Resilient Distributed Dataset) 데이터 표현 방식, 반복 계산에 적합, HDFS와 연동 [2024(72)].
    - [DB응용] XML: XML DTD(Document Type Definition) 구문을 이해하고 파싱 오류를 식별할 수 있어야 합니다 [2025(71)].
    - [DB응용] 다양한 데이터 저장 형식: Key-Value Store·JSON·XML·CSV의 구조적 특징과 활용 분야를 구분할 수 있어야 합니다 [2025(75)].
    - [DB응용] 데이터 웨어하우스(DW): 통합성·비휘발성·시계열성·주제 지향성 특징과 OLAP 분석 [2022(69)].
