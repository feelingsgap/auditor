정보시스템감리사 필기시험의 데이터베이스 영역은 DB 모델링/설계, 관계 대수/SQL, 정규화, 트랜잭션 관리, 데이터 무결성, 인덱싱/해싱, NoSQL/빅데이터, 데이터 마이닝/분석, 그리고 데이터 보안/복구 등 다양한 세부 범위로 구성되며, 각 분야에서 중요한 개념과 실제 문제 해결 능력을 평가하는 문항이 출제됩니다.

다음은 시험범위 및 기출 분석 자료를 바탕으로 데이터베이스 영역의 세부 공부 내역을 정리한 것입니다.

- **DB 모델링/설계**
    
    - **개체-관계 모델 (E-R Model)**: **약한 엔티티 타입**의 개념(키 형성에 불충분한 애트리뷰트, 강한 엔티티 타입에 의존)과 특징을 이해해야 합니다 [291, 106, 2022(53)]. EER 모델에서 여러 하위 개체 타입이 상위 개체 타입을 형성하는 **범주(category)** 개념을 이해하고, 이를 통해 모델링된 결과를 파악할 수 있어야 합니다 [250, 2024(64)]. ERD를 관계 모델로 변환할 경우 생성되는 릴레이션의 개수를 정확히 파악해야 합니다 [177, 2023(56)]. ER 스키마를 관계 모델의 릴레이션으로 사상하는 방법을 이해해야 합니다 [2025(55)].
    - **데이터베이스 구축 공정**: 메타데이터 입력항목 협의, 한자 및 다국어 지원 협의, 표준안 수립 및 작업지침 작성 등의 태스크가 진행되는 단계가 **계획수립** 세그먼트에 해당함을 알아야 합니다 [105, 2022(51)].
    - **관계 데이터 모델의 제약조건**: **도메인 무결성, 엔티티 무결성, 참조 무결성 제약조건**의 개념을 이해하고, 릴레이션에 대한 투플 삽입 연산 시 위배될 수 있는 제약조건들을 식별할 수 있어야 합니다 [291, 105, 2022(52)].
    - **고정 길이 레코드 방식**: 레코드 길이, 블록 크기, 블록 헤더 길이를 이용하여 블록당 저장 가능한 레코드 수 및 남는 바이트를 계산하는 능력이 요구됩니다 [176, 2023(52)].
    - **릴레이션의 최대 투플 개수**: 주어진 조건(도메인 크기, 기본키) 하에서 릴레이션이 가질 수 있는 최대 투플 개수를 계산할 수 있어야 합니다 [2025(56)].
    - **ER 모델**: ER 다이어그램을 관계 스키마로 변환하는 규칙(1:1, 1:N, N:M 관계 처리)을 이해해야 합니다 [288, 2021(52)]. 주어진 ER 스키마를 릴레이션으로 사상할 때 외래키 설정 등을 파악해야 합니다 [2025(55)].
    - **후보키(Candidate Key)**: 릴레이션의 함수적 종속성을 분석하여 후보키를 식별할 수 있어야 합니다 [2025(51)].
    - **카디널리티(Cardinality)**: 도메인 크기가 주어졌을 때 릴레이션이 가질 수 있는 최대 투플 수를 계산할 수 있어야 합니다 [2025(56)].
    - **뷰(View)**: 뷰의 정의(CREATE VIEW)와 그룹화(GROUP BY)를 이용한 뷰 생성 구문을 이해해야 합니다 [2025(65)].
- **관계 대수/SQL**
    
    - **SQL 함수**: `COUNT(*)`는 투플의 집합을 처리 대상으로 하며, 그 결과는 투플의 수이고, 중복 투플을 포함합니다 [291, 34, 2021(54)]. `COUNT(속성명)`과 `COUNT(DISTINCT 속성명)`의 차이를 이해해야 합니다 [33, 2021(54)].
    - **VIEW (뷰)**: **뷰에 대한 투플의 삽입은 시스템에 의해 거부될 수 있으며** [291, 34, 2021(55)], **집계(aggregation) 함수에 의해 생성된 뷰의 경우 일반적으로 갱신을 허용하지 않습니다** [291, 34, 2021(55)]. `GROUP BY`절을 사용해도 뷰 생성은 가능하며, 뷰의 갱신 가능성 수준에 따른 유형과 포함 관계를 이해해야 합니다 [119, 291, 2022(68)]. 두 개 이상의 테이블이 관련되거나, `DISTINCT`, `GROUP BY`, 상수/산술 연산자/함수가 사용된 뷰, 기본 테이블의 기본 키를 구성하는 속성이 포함되어 있지 않은 뷰는 변경할 수 없습니다 [178, 2023(57)].
    - **조인 연산**: 질의 처리에서 **가장 많은 비용이 소요되는 연산 중 하나**이며 [291, 34, 2021(56)], 중첩 루프 조인, 정렬-합병 조인, 분할-해시 조인 등의 구현 방법이 있습니다 [35, 2021(56)]. `INNER JOIN`, `LEFT OUTER JOIN`, `RIGHT OUTER JOIN`, `FULL JOIN` 등 다양한 조인 연산의 결과로 생성되는 레코드의 개수를 파악할 수 있어야 합니다 [109, 2022(57), 2025(53)].
    - **디비전 연산**: "모든 과목에 수강하고 있는 학생의 학번과 이름을 검색하라"와 같은 질의에 사용되는 관계 대수 연산입니다 [291, 36, 2021(59)]. 릴레이션 간 연산 결과의 카디널리티(cardinality) 최소값과 최대값을 계산하는 능력이 요구됩니다 [114, 2022(59)].
    - **SQL DDL (Data Definition Language)**: 테이블 정의 시 `FOREIGN KEY (속성) REFERENCES 참조테이블 (참조속성) ON DELETE (SET NULL/SET DEFAULT/CASCADE)`와 같은 **참조 무결성 조치**를 명시하는 방법을 알아야 합니다 [40, 291, 2021(65), 243, 2024(55)]. `ON DELETE CASCADE`는 참조되는 투플이 삭제될 때 해당 예금자 릴레이션의 투플도 같이 삭제되도록 합니다 [243, 2024(55)].
    - **SQL 질의문 작성 및 해석**: `ORDER BY`, `GROUP BY`, `HAVING`절을 포함한 질의 [113, 2022(61)], `EXISTS`, `NOT EXISTS`, `IN` 등의 서브쿼리를 활용한 질의 [42, 186, 2021(67), 2023(65), 2025(58)]를 정확히 해석하고 작성할 수 있어야 합니다. 자체 조인(self join)을 통해 직원-관리자 관계 등을 검색하는 질의 결과를 예상할 수 있어야 합니다 [38, 2021(63)].
    - **관계 대수 연산 결과의 카디널리티 계산**: `theta join`, `natural join`, `left outer join`, `right outer join` 등 여러 관계 연산을 수행했을 때 결과 릴레이션의 카디널리티 합을 계산할 수 있어야 합니다 [43, 2021(68)].
    - **SQL 질의 최적화의 동등 규칙**: 주어진 식을 다른 식으로 변경하기 위해 사용되는 관계 대수(relational algebra)의 **동등 규칙(equivalence rule)**을 이해해야 합니다 [252, 2024(67)].
    - **SQL 질의 결과**: `LEFT JOIN`, `GROUP BY`, `HAVING` 등을 포함한 복잡한 SQL 질의의 결과를 예측할 수 있어야 합니다 [2025(57)].
    - **SQL 조인(Join)**: 내부 조인, 외부 조인(Left, Right, Full), 교차 조인 등의 결과 집합을 예측할 수 있어야 합니다 [288, 2021(54)]. 완전 외부 조인(Full Outer Join)의 결과를 파악해야 합니다 [2025(53)].
    - **관계 대수 연산**: 셀렉트, 프로젝트, 조인, 디비전(Division) 등의 연산 결과와 SQL 변환을 이해해야 합니다 [288, 2021(59)]. 디비전 연산을 수행하기 위해 필요한 기본 연산자(프로젝트, 차집합, 곱집합)를 알아야 합니다 [2025(60), 2025(61)].
    - **SQL 집계 함수**: COUNT, SUM, AVG 등 집계 함수와 GROUP BY, HAVING 절의 동작 원리를 이해해야 합니다 [2025(57), 2025(59)].
    - **상관 부속 질의(Correlated Subquery)**: 메인 쿼리와 서브 쿼리가 연관된 질의의 동작을 이해하고, 관계 대수(차집합 등)로 변환할 수 있어야 합니다 [2025(58)].
    - **JDBC**: JDBC를 이용한 데이터베이스 프로그래밍 순서(드라이버 로드 -> 연결 -> 문장 생성 -> 실행 -> 결과 처리 -> 해제)를 알아야 합니다 [2025(72)].
- **정규화**
    
    - **함수적 종속성 (Functional Dependency)**: 주어진 릴레이션의 함수적 종속성을 파악하고 **후보 키**를 식별할 수 있어야 합니다 [291, 33, 2021(53)].
    - **정규형 (Normal Forms)**: 제2 정규형, 제3 정규형, **BCNF(Boyce-Codd Normal Form)**의 개념과 특징을 이해하고, 주어진 릴레이션을 BCNF로 정규화한 결과를 파악할 수 있어야 합니다 [291, 107, 2022(55), 180, 2023(61), 244, 2024(57)].
    - **바람직한 분해 (Decomposition)**: **손실 없는 분해(lossless decomposition)**가 되어야 하며, 함수적 종속성이 보존되어야 합니다 [291, 47, 2021(74)].
    - **정규형(Normal Form)**: 제1정규형(도메인 원자성), 제2정규형(부분 함수 종속 제거), 제3정규형(이행 함수 종속 제거), BCNF(결정자가 모두 후보키)의 조건을 이해하고 정규화 과정을 수행할 수 있어야 합니다 [288, 2021(57)]. 주어진 릴레이션을 제2정규형으로 분해했을 때 생성되는 릴레이션 개수를 파악해야 합니다 [2025(54)].
    - **무손실 분해(Lossless Decomposition)**: 릴레이션 분해 후 자연 조인을 통해 원래 릴레이션으로 복구 가능한지(무손실 분해 조건)를 판단해야 합니다 [2025(52)].
- **트랜잭션 관리**
    
    - **트랜잭션의 ACID 속성**: **원자성(Atomicity), 일관성(Consistency), 고립성(Isolation), 지속성(Durability)**을 이해해야 합니다.
    - **동시성 제어 (Concurrency Control)**:
        - **로킹 (Locking)**: **로킹 단위가 작아지면 동시성(병행) 수준은 높아지고** [187, 2023(67)], 로킹 오버헤드는 증가합니다 [291, 187, 2023(67)]. 데이터베이스, 파일, 레코드 등이 로킹 단위가 될 수 있습니다 [187, 2023(67)]. **엄격한 2단계 로킹 규약(strict 2PLP)**에서는 모든 로크가 트랜잭션 완료 시까지 유지됩니다 [188, 2023(68)]. **낙관적 로킹과 비관적 로킹**의 장단점 및 적용 사례를 비교할 수 있어야 합니다 [241, 2024(54)].
        - **트랜잭션 문제점**: 동시성 제어를 하지 않을 때 발생할 수 있는 **오손 데이터 읽기(dirty read), 갱신 손실(lost update), 반복할 수 없는 읽기(unrepeatable read), 모순성(inconsistency)** 등 다양한 문제점을 이해해야 합니다 [291, 39, 2021(64)]. 특정 시나리오에서 **유령 데이터 읽기(phantom read)**와 같은 현상을 식별할 수 있어야 합니다 [185, 2023(66)].
    - **회복 (Recovery) 기법**:
        - **검사시점 (Checkpoint) 회복**: 시스템 장애 발생 후 트랜잭션들의 REDO 연산과 UNDO 연산 수행 규칙을 이해해야 합니다. 예를 들어, 검사시점 이전에 완료된 트랜잭션은 회복 작업에 관련될 필요가 없으며, 수행 중이던 트랜잭션은 UNDO를 수행해야 합니다 [291, 45, 2021(71)]. 검사시점 회복의 주요 작업들의 순서를 파악할 수 있어야 합니다 [115, 2022(63)].
        - **지연 갱신 (Deferred Update) 및 즉시 갱신 (Immediate Update)**: **지연 갱신에서는 완료(commit) 이전에 실패한 트랜잭션의 경우 UNDO가 필요하지 않습니다** [291, 118, 2022(66)]. **즉시 갱신에서는 완료 이전에 실패한 트랜잭션의 경우 UNDO를 수행해야 합니다** [291, 118, 2022(66)]. **WAL(Write-Ahead Logging)** 기법에서 AFIM(After Image)은 REDO에, BFIM(Before Image)은 UNDO에 대비하기 위함입니다 [256, 2024(74)].
        - **즉시 변경(immediate update) 회복**: 로그 기록을 보고 시스템 장애 후 복구되었을 때 데이터 값을 파악할 수 있어야 합니다 [2025(70)].
    - **동시성 제어(Concurrency Control)**: 로킹(Locking), 타임스탬프 순서 기법, 다중 버전 동시성 제어(MVCC) 등의 기법을 이해해야 합니다 [288, 2021(64)].
    - **다중 단위 로킹(Multiple Granularity Locking)**: 의도 로크(Intention Lock: IS, IX, SIX)의 개념과 로크 간의 양립성(Compatibility) 행렬을 이해해야 합니다 [2025(66)].
- **데이터 무결성**
    
    - **키**: 모든 릴레이션에는 최소한 하나의 수퍼키와 후보키가 존재합니다 [291, 46, 2021(72)]. 릴레이션에서 **외래키는 널(NULL) 값을 갖지 않을 수도 있으며**, 하나의 릴레이션에 여러 개가 존재할 수 있습니다 [291, 46, 2021(72)]. 후보키의 개수는 수퍼키 개수보다 많을 수 없습니다 [291, 46, 2021(72)].
    - **참조 무결성 제약조건**: 외래 키를 사용하여 테이블 간의 관계를 설정하며, 참조하는 테이블의 투플이 삭제될 때 참조되는 테이블의 투플을 어떻게 처리할지(SET NULL, SET DEFAULT, CASCADE 등)를 정의할 수 있습니다 [40, 2021(65), 108, 2022(56), 243, 2024(55)].
    - **함수 종속**: 릴레이션 내 속성 간의 함수 종속 관계를 파악하고, 주어진 함수 종속성으로부터 항상 성립하는 추론 규칙을 식별할 수 있어야 합니다 [107, 2022(54), 179, 2023(58)].
    - **손실 없는 분해**: 릴레이션을 두 개의 릴레이션으로 분해했을 때, 손실 없는 분해가 되는 경우를 식별할 수 있어야 합니다 [2025(52)].
    - **무결성 제약조건**: 개체 무결성, 참조 무결성, 도메인 무결성 등의 개념과 SQL(DDL)에서의 정의 방법(PRIMARY KEY, FOREIGN KEY, CHECK 등)을 이해해야 합니다 [288, 2021(53)].
    - **참조 무결성 옵션**: 외래키 설정 시 `ON DELETE`, `ON UPDATE` 옵션(`CASCADE`, `SET NULL`, `SET DEFAULT`)의 동작을 이해해야 합니다 [2025(62)].
- **인덱싱/해싱**
    
    - **B-트리 (B-tree)**: 대용량 데이터 저장 및 검색에 효율적인 트리 구조로, 삽입 및 삭제 시 노드 분할 또는 병합 발생, 루트 공간 사용률, 높이 증가 가능성 등을 이해해야 합니다 [291, 35, 2021(58), 256, 2024(73)].
    - **확장 가능 해싱 (Extendible Hashing)**: 전역 깊이(global depth)와 지역 깊이(local depth)의 개념을 이해하고, 특정 레코드 삽입 시 변경되는 깊이 값을 계산할 수 있어야 합니다 [291, 41, 2021(66), 255, 2024(71)].
    - **인덱스**: 인덱스에 저장되어 있는 ROWID의 구성 요소(오브젝트 번호, 데이터 파일 번호, 블록 번호 등)를 이해하고, 인덱스 ROWID를 이용한 테이블 블록 읽기 메커니즘을 파악해야 합니다 [245, 2024(59)].
    - **비순서 히프파일 (Heap File)**: 레코드 개수, 길이, 블록 크기 등의 매개변수가 주어졌을 때, 주어진 특정 레코드를 찾기 위해 평균적으로 걸리는 시간을 계산할 수 있어야 합니다 [113, 2022(62)].
    - **B-트리**: 데이터 삽입 후 트리의 상태 변화를 이해하고, 특정 노드의 키 값을 파악할 수 있어야 합니다 [2025(63)].
    - **확장 가능 해싱**: 특정 키 값 삭제 후의 디렉터리 및 버킷 상태를 파악할 수 있어야 합니다 [2025(64)].
    - **B-Tree / B+-Tree**: 인덱스 구조의 특징과 삽입/삭제 시 분할 및 병합 과정을 이해해야 합니다 [288, 2021(58)]. B+-Tree의 노드 구조와 키 값 분포 특성을 파악해야 합니다 [2025(63)].
    - **해싱(Hashing)**: 정적 해싱과 동적 해싱(확장성 해싱 등)의 차이와 동작 원리를 이해해야 합니다 [288, 2021(62)]. 확장성 해싱(Extendible Hashing)에서 전역 깊이와 지역 깊이의 변화를 이해해야 합니다 [2025(64)].
- **NoSQL/빅데이터**
    
    - **NoSQL 데이터 모델 유형**: **그래프 기반(graph-based)** 데이터베이스는 관계 데이터 모델과 유사하게 데이터 간의 관계 표현에 적합하며, 트랜잭션을 통해 ACID를 지원합니다 [291, 38, 2021(62)]. **문서 기반(document-based)** 데이터 모델은 JSON 기반으로 복잡한 구조의 객체를 저장하며 MongoDB가 대표적입니다 [240, 2024(51)].
    - **NoSQL 특징 및 장점**: 노드 추가를 통한 높은 확장성, 데이터 복제를 통한 고장 허용(fault tolerance), 샤딩(sharding)을 사용한 부하 분산, 읽기/쓰기 성능 개선 등을 제공합니다 [291, 122, 2022(72)].
    - **CAP 이론**: 분산 시스템의 요구사항을 설명하는 이론으로, **일관성(Consistency), 가용성(Availability), 분할 감내(Partition Tolerance)** 성질을 의미합니다. **NoSQL 시스템에서는 CAP의 세 가지 성질을 모두 보장하기 어렵고**, 보통 일관성의 완화된 형태인 **궁극적 일관성(eventual consistency)**을 수용합니다 [291, 190, 2023(72)].
    - **관계형 데이터베이스와 NoSQL 비교**: 스키마 유연성, 클러스터 환경 적합성, 단순 검색 기능, 트랜잭션 일관성 보장 어려움 측면에서 NoSQL이 관계형 데이터베이스와 다릅니다 [192, 2023(74)].
    - **Apache Spark**: 데이터 표현 방식으로 **RDD(Resilient Distributed Dataset)**를 사용하며, 맵리듀스보다 계산 속도가 빠르고 파이썬, 스칼라, 자바 등으로 개발 가능하며, 기계 학습, 그래프 연산 등 반복적인 계산에 적합합니다 [255, 292, 2022(98)]. 하둡 2.0(YARN 환경)에서도 Spark를 지원합니다 [292, 2022(98)].
    - **NoSQL과 관련된 데이터베이스 성질**: **분할 감내성(partition tolerance), 가용성(availability)** 등을 이해해야 합니다 [2025(73)].
    - **데이터 저장 형식**: **Key-Value Store, JSON, XML, CSV** 등 다양한 데이터 저장 형식의 특징을 구분할 수 있어야 합니다 [2025(75)].
    - **NoSQL**: CAP 이론(Consistency, Availability, Partition Tolerance)과 NoSQL 데이터베이스의 유형(Key-Value, Document, Column, Graph)을 이해해야 합니다 [288, 2021(51)]. NoSQL과 관련된 성질(BASE 등)을 파악해야 합니다 [2025(73)].
    - **하둡(Hadoop)**: HDFS(Hadoop Distributed File System)의 아키텍처(NameNode, DataNode)와 읽기/쓰기 성능 특성을 이해해야 합니다 [2025(67)].
    - **데이터 포맷**: JSON, XML, CSV, Key-Value Store 등 다양한 데이터 저장 및 교환 형식의 특징을 비교할 수 있어야 합니다 [2025(75)].
    - **XML**: XML DTD(Document Type Definition) 구문을 이해하고 파싱 오류를 식별할 수 있어야 합니다 [2025(71)].
- **데이터 마이닝/분석**
    
    - **연관 규칙 마이닝 (Association Rule Mining)**: Apriori 알고리즘을 사용하며, 빈발 항목집합(frequent itemset) 및 신뢰도(confidence)를 계산하는 방법을 이해해야 합니다 [291, 47, 2021(75), 118, 2022(67), 123, 2022(73)].
    - **나이브 베이지안 분류기 (Naive Bayesian Classifier)**: 특정 조건부 확률 값을 계산하는 능력이 요구됩니다 [44, 2021(69)].
    - **TFIDF (Term Frequency-Inverse Document Frequency)**: 문서 집합에서 특정 단어의 중요도를 나타내는 통계적 수치로, TF(term frequency)와 IDF(inverse document frequency)의 개념을 이해하고 TFIDF 값을 계산할 수 있어야 합니다 [291, 121, 2022(71)].
    - **의사결정 트리 (Decision Tree)**: 불순도(impurity)를 표현하는 Gini 값 또는 엔트로피 계산 방법을 알아야 합니다 [291, 120, 2022(70)].
    - **혼돈 행렬 (Confusion Matrix)**: 예측 모델의 성능을 평가하는 지표(정확도, 정밀도, 재현율, 특이도 등)를 혼돈 행렬로부터 계산할 수 있어야 합니다 [291, 124, 2022(74)].
    - **검색 엔진 평가 척도**: **리콜(recall)**, **정확도(precision)**, 평균 정확도(average precision), F-스코어(F-score)의 정의와 계산 방법을 이해해야 합니다 [248, 2024(62)]. F-스코어는 정확도와 리콜의 조화 평균으로, 두 값 중 더 작은 쪽에 가까워지는 경향이 있습니다 [248, 2024(62)].
    - **k-Means 군집화 (Clustering)**: 초기 중심점 선택에 따라 군집 결과가 달라질 수 있으며, 이상치(outlier)를 포함하는 경우 잘 동작하지 않습니다 [291, 192, 2023(75)]. 유사도(점과 점 사이의 거리) 계산 횟수를 계산할 수 있어야 합니다 [257, 2024(75)].
    - **멀티미디어 데이터 검색 질의**: 수학 함수로 저장된 데이터와 질의 조건으로 주어진 데이터 간의 유사도를 수학 함수로 계산하여 유사도가 높은 데이터를 검색하는 기법은 **랭킹 기법(ranking technique)**에 해당합니다 [191, 2023(73)].
    - **Apriori 알고리즘**: 비-단조(anti-monotone) 성질의 핵심 원리를 이해해야 합니다 [2025(68)].
    - **베이즈 정리(Bayes theorem)**: 주어진 조건 하에서 특정 클래스에 속할 확률을 계산할 수 있어야 합니다 [2025(69)].
    - **의사결정 트리**: Gini Index와 분류 오류(classification error)의 개념을 이해하고, 특정 노드의 불순도를 계산할 수 있어야 합니다 [2025(74)].
    - **연관 규칙(Association Rule)**: 지지도(Support), 신뢰도(Confidence), 향상도(Lift)의 개념과 계산 방법을 알아야 합니다 [288, 2021(67)]. Apriori 알고리즘의 비-단조(anti-monotone) 성질을 이해해야 합니다 [2025(68)].
    - **분류(Classification)**: 의사결정 트리(Decision Tree)의 불순도(Impurity) 측정 지표인 지니 지수(Gini Index)와 엔트로피를 이해해야 합니다 [2025(74)]. 베이즈 정리(Bayes Theorem)를 이용한 사후 확률 계산을 수행할 수 있어야 합니다 [2025(69)].
- **데이터 보안/복구**
    
    - **데이터베이스 보안 요구사항**: 데이터에 대한 허용 값 통제, 중요 데이터의 기밀성 보호, 인가된 사용자만 접근 허용, 트랜잭션 병행 처리 시 데이터의 논리적 일관성 보장 등이 포함됩니다. 다만, **사용자가 통계 집합적 데이터로부터 개별적인 데이터 항목에 대한 정보를 추적할 수 있어야 한다는 설명은 보안 요구사항과 거리가 니다** [292, 64, 2021(107)].
    - **접근 통제**: 필수 접근 제어(MAC) 기법은 각 데이터 객체에 비밀 등급을 부여하고 사용자도 허가 등급을 지정하여 접근을 통제합니다 [291, 117, 2022(65)].
    - **가명 정보**: 안전한 관리를 위한 기술적 보호 조치(추가 정보와 물리적 분리, 접근 권한 최소화, 이용 내역 기록 보관, 담당자 엄격 통제)를 이해해야 합니다 [291, 142, 2022(114)].
    - **트리거(trigger)와 저장 프로시저(stored procedure) 비교**: 입력 인자/결과 반환 여부, 할당 대상(테이블/뷰 vs 데이터베이스), 실행 시점(DBMS에 의해 vs 필요시), 저장 및 컴파일 방식 등 그 특성을 구분할 수 있어야 합니다 [251, 2024(65)].
    - **SQL DDL**: `ON DELETE SET NULL ON UPDATE CASCADE`와 같은 참조 무결성 제약조건의 동작을 이해하고, 특정 DML 문장 실행 후의 릴레이션 상태를 예측할 수 있어야 합니다 [2025(62)].

이러한 세부 공부 내역을 바탕으로 각 개념에 대한 깊이 있는 이해와 문제 해결 능력을 향상시키는 것이 정보시스템감리사 자격 검정 준비에 도움이 될 것입니다.
