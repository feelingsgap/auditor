"시험범위 및 기출분석" 자료를 바탕으로 시스템 구조 영역의 세부 공부 내역을 정리해 드리겠습니다. 시스템 구조 영역은 **컴퓨터 구조, 운영체제, 네트워크 구조, 프로토콜**이 주요 출제 분야이며, 최근에는 **머신러닝/AI, 가상화/클라우드** 관련 문항이 증가하는 추세입니다. **저장장치/백업, 네트워크 보안**에 관한 문항도 꾸준히 출제되고 있으며, 특히 **SDN, 마이크로서비스 아키텍처, 컨테이너 기술** 등 새로운 IT 기술에 관한 문항도 중요한 비중을 차지합니다.

다음은 시스템 구조의 세부 범위별 공부할 내역입니다.

- **컴퓨터 구조**
    
    - **하드 드라이브, RAID 배열, 비디오-캡쳐 박스 및 네트워크 인터페이스와 같은 주변장치들을 위한 데이터, 비디오, 오디오 및 전력을 하나의 고속 연결선으로 통합한 초고속 주변 장치 접속 기술**을 이해해야 합니다 [48, 2021(76)]. (예: 선더볼트)
    - **캐시 메모리**에 대한 이해가 필요합니다. 특히 **직접 사상(direct mapped) 캐시**와 **완전 연관(fully associative) 캐시**의 특징 및 성능 차이점을 알아야 합니다 [50, 2021(81)].
    - **MTTR(Mean Time To Repair)** 계산과 같은 **시스템 가용성 지표**를 파악해야 합니다 [51, 2021(82)].
    - **컨볼루션 신경망(CNN)에서 원본 이미지와 필터 행렬의 합성곱 연산 결과로 생성되는 특징 맵 행렬의 크기 계산**을 알아야 합니다 [52, 2021(85)].
    - **디지털논리회로(예: 멀티플렉서, 인코더, 디코더, 디멀티플렉서)**의 기능 및 특징을 이해해야 합니다 [258, 2024(77)].
    - **CPU가 I/O 장치와 기억장치 간 데이터 전송에 개입 없이 수행하도록 지원하는 메커니즘** (예: DMA)에 대해 알아야 합니다 [259, 2024(82)].
    - **2의 보수 체계**를 사용하여 정수를 표현하는 방식에 대한 이해가 필요합니다 [126, 2022(80)].
    - **패리티 기법** (예: 짝수 패리티 기법을 적용한 아스키코드 인코딩)에 대해 알아야 합니다 [126, 2022(79)].
    - **부울식 최소화**: 카르노 맵을 이용하여 주어진 진리표를 최소화된 부울식으로 변환할 수 있어야 합니다 [2025(79)].
    - **TPS 계산**: 개별 업무의 TPS와 운영 시 요청 수를 바탕으로 시스템의 전체 최대 TPS를 계산할 수 있어야 합니다 [2025(80)].
    - **SIMD(Single Instruction Multiple Data)**: 하나의 명령어로 여러 개의 독립적인 연산을 동시에 처리하는 기술을 이해해야 합니다 [2025(84)].
    - **부동소수점 연산**: 덧셈 과정에서 정규화, 지수 조정, 가수 덧셈 등의 과정이 필요함을 이해해야 합니다 [2025(78)].
    - **평균 메모리 접근 시간(AMAT)**: 클록 사이클, 캐시 접근 시간, 실패 손실, 실패율을 이용하여 AMAT를 계산할 수 있어야 합니다 [2025(85)].
- **운영체제**
    
    - **도커 컨테이너**에 대한 이해가 중요합니다. **가상화된 공간을 프로세스 단위의 격리 환경으로 만들고, 호스트 커널을 공유하며 애플리케이션 구동에 필요한 라이브러리와 실행 파일만 존재**하는 특징을 알아야 합니다 [48, 2021(77)]. 또한 **게스트 운영체제와 하이퍼바이저가 존재하지 않아 성능 손실이 적다는 장점**이 있습니다 [49, 2021(77)].
    - **리눅스 파일 관리**에 대한 이해가 필요하며, **파이프 파일, 특수 파일(장치), inode** 등의 개념을 알아야 합니다 [194, 2023(83)].
    - **CPU 스케줄링 정책** (예: 선점(preemptive) 스케줄링)의 특징 및 문맥 교환 발생 빈도 등을 파악해야 합니다 [126, 2022(81)].
    - **가상 메모리 관리**를 위한 **페이지 교체 알고리즘** (예: LRU)에 대해 알아야 합니다 [192, 2023(76)].
    - **SSD의 쓰레기 수집(garbage collection) 효율성 증대**를 위한 운영체제 지원 기능(예: TRIM)을 이해해야 합니다 [259, 2024(81)].
    - **리눅스의 파일 종류** (예: FIFO 파일, 일반 파일, 디렉토리 파일)를 구분할 수 있어야 합니다 [258, 2024(78)].
    - **다단계 피드백 큐(Multi-level Feedback Queue) CPU 스케줄링**: 각 큐의 CPU 할당 시간, 큐 간 프로세스 이동 규칙 등을 이해해야 합니다 [2025(83)].
    - **가상 기억장치**: 페이지 부재(fault) 발생 시의 처리 과정을 이해해야 합니다 [2025(86)].
    - **TPS 계산**: 개별 업무의 TPS와 운영 시 요청 수를 바탕으로 시스템의 전체 최대 TPS를 계산할 수 있어야 합니다 [2025(80)].
- **네트워크 구조**
    
    - **네트워크 다이어그램에서 자원 평준화(resource leveling) 방법**을 이해해야 합니다 [3, 2021(4)].
    - **패킷교환(packet switching)과 회선교환(circuit switching)의 특징**을 비교하여 이해해야 합니다. 특히 **회선교환은 종단 간의 전송률을 보장할 수 있고 링크 자원의 공유가 허용되지 않는다는 특징**을 알아야 합니다 [52, 2021(84)].
    - **라우팅(routing)에 대한 설명**을 파악해야 합니다. **경로설정 테이블의 구성, 유니캐스트/멀티캐스트 프로토콜의 존재, IGP/EGP 구분** 등을 알아야 합니다 [127, 2022(82)].
    - **5G 기반 스마트시티 영상 정보 수집 시스템에 대한 요구사항** (이벤트 메타데이터 기반 즉시 대응 처리, 추출된 이벤트 객체 기반 프레임 메타데이터/이벤트 메타데이터 생성)을 알아야 합니다 [49, 2021(78)].
    - **SDN(Software Defined Networking) 오버레이 수행 방법** (VXLAN, NVGRE, NVO3) 및 **SDN의 컨트롤러 기술**에 대해 이해해야 합니다 [53, 129, 2021(86), 2022(88)].
    - **OSI 7계층에서 주요 계층별 대표 프로토콜**을 올바르게 짝짓는 것이 중요합니다 [56, 2021(93)].
    - **스위치(switch) 유형별(L2, L3, L4, L7) 특징 및 기능**을 이해해야 합니다 [56, 2021(94)].
    - **마이크로서비스 아키텍처(micro service architecture)**의 특징 (작은 코드베이스, 분산 제어 구조, 독립적 자동화 배치, 테스트 용이성 등)을 파악해야 합니다 [57, 2021(95)].
    - **IP 터널링**에 대한 이해가 필요합니다 [50, 2021(80)].
    - **네트워크 장비를 기능에 따라 OSI 7 계층과 올바르게 매핑**하는 것을 알아야 합니다 [125, 2022(76)].
    - **성형(star) 토폴로지**의 특징을 이해해야 합니다 [125, 2022(77)].
    - **LAN 스위칭 기술** (Cut-Through, Interim Cut-Through, Store-and-Forward 스위치)을 구분하는 기준(전송 로직)을 알아야 합니다 [130, 2022(89)].
    - **IPv4에서 단편화(fragmentation)가 발생된 패킷의 헤더 중 변경되지 않는 정보** (예: 식별자)를 알아야 합니다 [130, 2022(90)].
    - **멀티캐스트 라우팅 프로토콜** (예: CBT)을 이해해야 합니다 [130, 2022(91)].
    - **SCTP(Stream Control Transmission Protocol)의 연결설정 과정**을 파악해야 합니다 [130, 2022(92)].
    - **NAT(network address translation)** 및 그 방식(SNAT, PAT)에 대한 이해가 필요합니다 [131, 2022(93)].
    - **이더넷 MAC(Media Access Control) 주소 유형** (유니캐스트, 멀티캐스트, 브로드캐스트)을 구분해야 합니다 [197, 2023(87)].
    - **라우팅 기법** (예: 소스 라우팅)의 특징을 알아야 합니다 [261, 2024(86)].
    - **IP 주소와 마스크가 주어진 경우 네트워크 주소를 계산**할 수 있어야 합니다 [199, 2023(92)].
    - **BGP4(Border Gateway Protocol Version 4)**에 대한 이해가 필요합니다 [197, 2023(89)].
    - **PPP(Point-to-Point Protocol)와 관련된 프로토콜** (LCP, NCP, IPCP, CHAP)의 기능을 알아야 합니다 [198, 2023(91)].
    - **TCP와 UDP 프로토콜 헤더에 공통적으로 존재하는 필드** (체크섬)를 알아야 합니다 [193, 2023(77)].
    - **RIP(Routing Information Protocol)**의 특징과 한계 (정적 라우팅 프로토콜이 아님)를 이해해야 합니다 [264, 2024(92)].
    - **디폴트 게이트웨이 IP 주소**를 파악하는 방법을 알아야 합니다 [261, 2024(87)].
    - **IPv4 헤더의 특징**과 단편화 관련 필드를 이해해야 합니다 [262, 2024(88)].
    - **모뎀, 리피터, L2 스위치, USB 허브 등 네트워크 장비의 기능**을 구분할 수 있어야 합니다 [263, 2024(90)].
    - **FSK(Frequency Shift Keying) 변조 방식**의 특징을 알아야 합니다 [263, 2024(91)].
    - **DHCP(Dynamic Host Configuration Protocol) 메시지**의 종류와 기능을 이해해야 합니다 [133, 2022(99)].
    - **TCP 프로토콜 애플리케이션과 잘 알려진 포트 번호**를 연결할 수 있어야 합니다 [266, 2024(98)].
    - **ARP Request 발송 시 패킷을 수신하는 노드**를 파악할 수 있어야 합니다 [266, 2024(99)].
    - **TCP 기반의 연결 설정 과정**을 이해해야 합니다 [266, 2024(100)].
    - **데이터 압축**: 무손실 압축과 손실 압축의 차이점을 이해해야 합니다 [2025(88)].
    - **네트워크 현상**: 지터(jitter), 패킷 손실(packet loss), 지연시간(latency), 전파 지연(propagation delay)의 개념을 구분할 수 있어야 합니다 [2025(90)].
    - **VLAN 프로토콜**: IEEE 802.1Q의 특징과 프레임 태깅 방식을 이해해야 합니다 [2025(94)].
    - **파장(Wavelength) 계산**: 신호의 속도와 주파수가 주어졌을 때 파장을 계산할 수 있어야 합니다 [2025(89)].
    - **네트워크 관리**: netstat 명령어를 사용하여 백도어 등 비정상적인 연결 상태를 확인할 수 있음을 알아야 합니다 [2025(96)].
    - **SDN(Software Defined Networking)**: 제어 평면과 데이터 평면의 분리, OpenFlow 프로토콜 등 SDN의 특징을 이해해야 합니다 [2025(95)].
    - **신호 변환**: 차분 맨체스터(Differential Manchester) 인코딩 방식을 이해하고 신호를 해석할 수 있어야 합니다 [2025(93)].
- **프로토콜**
    - **POP3 서비스 명령어**: LIST, DELE, RETR 등 POP3에서 사용하는 명령어의 기능을 알아야 합니다 [2025(91)].
    - **IPv6 헤더**: Hop Limit, Flow Label, Traffic Class 등 IPv6 헤더 필드의 기능을 이해해야 합니다 [2025(82)].
    - **오류 제어**: CRC 하드웨어 구현(LFSR)을 보고 생성다항식을 유추하거나, 다양한 오류 검출 방식(BCD, 패리티 등)을 이해해야 합니다 [2025(87), 2025(92)].
- **머신러닝/AI**
    - **머신러닝 모델 성능 평가 지표** (정확도, 정밀도, F1-점수, 재현율)에 대한 올바른 정의를 알아야 합니다 [49, 2021(79)].
    - **딥러닝 방식** (예: GAN)의 특징을 알아야 합니다 [129, 2022(86)].
    - **활성화 함수** (예: 시그모이드 함수)의 특징과 문제점 (기울기 소실)을 이해해야 합니다 [131, 2022(94)].
    - **시계열 데이터 예측**에 주로 사용되는 딥러닝 모델(예: 순환신경망)의 특징을 알아야 합니다 [134, 2022(96)].
    - **kNN(k-Nearest Neighbor) 알고리즘**의 특징 (간단하고 이해하기 쉬움, 새로운 데이터 할당, 계산 시간)과 학습 유형 (지도 학습)을 이해해야 합니다 [199, 2023(93)].
    - **신경망 기반의 기계학습 알고리즘** (GAN, DBN, VAE)과 아닌 것(SVM)을 구분해야 합니다 [200, 265, 2023(94), 2024(96)].
    - **신경망 학습을 위한 활성 함수** (ReLU)와 **최적화 알고리즘** (RMSProp, SGD, Adam)을 구분해야 합니다 [201, 2023(97)].
    - **자연어 처리 모델** (BERT, GPT, CBOW)의 특징과 차이점을 이해해야 합니다 [201, 2023(98)].
    - **전이 학습(Transfer Learning)** 개념을 이해해야 합니다 [202, 2023(99)].
    - **인공지능 학습용 데이터 품질관리 가이드라인**에 제시된 품질관리 활동의 수행 순서를 알아야 합니다 [218, 2024(4)].
    - **학습 유형**: 지도학습, 비지도학습, 강화학습을 구분하고, 각 유형에 해당하는 알고리즘(K-평균 군집화, 나이브 베이즈 분류, 서포트 벡터 머신, 선형 회귀)을 식별할 수 있어야 합니다 [2025(98)].
    - **기계학습**: 과적합(Overfitting)의 개념과 해결 방안(정규화 등)을 이해해야 합니다 [2025(97)].
    - **파인튜닝(Fine-tuning)**: LLM을 특정 작업에 맞게 파라미터를 조정하는 기술을 이해해야 합니다 [2025(100)].
- **가상화/클라우드**
    
    - **OpenStack 코어 서비스** (Cinder, Keystone, Neutron, Nova 등)의 기능과 역할을 알아야 합니다 [55, 2021(90)].
    - **하이퍼바이저**의 유형 (Type1, Type2)과 특징 (직접 하드웨어 제어 여부, 게스트 OS 커널 수정 여부, 오버헤드, 장단점)을 비교하여 이해해야 합니다 [58, 2021(96)].
    - **서버리스 프레임워크 도입**에 필요한 구성 요소 (컨테이너 레지스트리, 컨테이너 오케스트레이터, 마스터 노드, 워커 노드)를 알아야 합니다 [60, 2021(99)].
    - **Spark**에 대한 설명 (인-메모리 방식, 빠른 계산 속도, 언어 지원, 반복적인 계산 문제 해결 적합)을 파악해야 합니다 [133, 2022(98)].
    - **컨테이너를 사용한 가상화**에 대한 이해 (프로세스 수준 가상화, 자원 절감, 독립적 환경 제공)가 필요합니다 [200, 2023(95)].
    - **가상화**에 대한 설명 (전가상화와 반가상화의 차이, 하이퍼바이저의 역할)을 이해해야 합니다 [200, 2023(96)].
