# GEMINI.md

## 프로젝트 개요

이 프로젝트는 정보시스템감리사 기출문제 PDF 파일을 분석하여, 연도별/과목별로 문제를 볼 수 있는 대화형 HTML 페이지를 생성하는 자동화 시스템입니다.

주요 작업은 3단계 파이프라인으로 구성됩니다.

1.  **분석 (Analysis)**: PDF 파일에서 각 문항의 위치, 과목, 번호 등 구조화된 메타데이터를 추출하여 JSON 파일로 저장합니다.
2.  **크롭 (Cropping)**: 분석 단계에서 생성된 메타데이터를 기반으로 원본 PDF에서 각 문항을 고해상도 이미지 파일(PNG)로 잘라냅니다.
3.  **생성 (Generation)**: 추출된 메타데이터와 크롭된 이미지를 사용하여 최종적으로 사용자가 보게 될 단일 HTML 파일을 생성합니다.

## 핵심 워크플로우 및 스크립트 분석

`scripts/` 디렉토리에는 이 파이프라인을 구성하는 핵심 Python 스크립트들이 포함되어 있습니다.

### 1. `pdf_analyzer.py` - PDF 분석 및 메타데이터 추출

-   **목적**: 원본 PDF 파일의 복잡한 레이아웃을 분석하여 각 문항의 정확한 위치(bounding box), 번호, 과목 정보를 담은 구조적인 `JSON` 데이터를 생성합니다. 이 스크립트는 전체 파이프라인의 정확성을 좌우하는 가장 중요한 부분입니다.
-   **핵심 로직**:
    -   **적응형 레이아웃 감지**: 연도별로 미세하게 다른 PDF의 2단(column) 레이아웃(좌표, 너비 등)을 자동으로 감지합니다.
    -   **강화된 문항 위치 탐색**: 정규표현식, 텍스트 검색, 인접 문항 위치 추정 등 여러 단계의 복합적인 방법을 사용하여 페이지 내에서 각 문항 번호의 정확한 좌표를 찾습니다.
    -   **경계 상자 최적화**: 탐색된 문항 위치를 기반으로, 다음 문항의 시작 위치를 고려하여 현재 문항이 잘리지 않도록 크롭 영역(bounding box)의 높이를 동적으로 계산합니다. 2021년도 PDF와 같이 특정 연도의 예외적인 레이아웃 문제를 처리하기 위한 하드코딩된 보정 로직을 포함합니다.
    -   **오류 처리 및 복구**: PDF 파싱 과정에서 누락되거나 잘못 배치된 문항을 식별하고, 전체 문항의 연속성을 기반으로 누락된 문항의 위치를 보간(interpolate)하거나 잘못된 페이지에 할당된 문항을 올바른 페이지로 재배치하는 등 강력한 후처리 및 복구 메커니즘을 갖추고 있습니다.
-   **입력**: `source/{year}_auditor.pdf`
-   **출력**: `{year}/{year}_data.json`

### 2. `question_cropper.py` - 문항 이미지 생성

-   **목적**: `pdf_analyzer.py`가 생성한 `JSON` 메타데이터를 사용하여, 원본 PDF 파일로부터 각 문항의 이미지를 잘라내어 개별 파일로 저장합니다.
-   **핵심 로직**:
    -   `JSON` 파일에 정의된 각 문항의 페이지 번호와 bounding box 정보를 읽습니다.
    -   `PyMuPDF` 라이브러리를 사용하여 PDF의 해당 위치를 300 DPI의 고해상도 `PNG` 이미지로 렌더링하고 저장합니다.
    -   크롭 작업 전, 메타데이터에 120개 문항이 모두 존재하는지 검증하여 작업의 완전성을 보장합니다.
-   **입력**: `source/{year}_auditor.pdf`, `{year}/{year}_data.json`
-   **출력**: `{year}/question/{year}_{question_num}.png` 형식의 이미지 파일들

### 3. `html_generator.py` - 최종 HTML 페이지 생성

-   **목적**: 분석된 메타데이터와 생성된 문항 이미지들을 취합하여 사용자가 최종적으로 보게 될 `index.html` 파일을 생성합니다.
-   **핵심 로직**:
    -   `Jinja2` 템플릿 엔진을 사용합니다.
    -   각 연도의 `{year}_data.json` 파일을 로드하여 모든 문항 정보를 준비합니다.
    -   준비된 데이터를 `templates/index.html` 템플릿에 주입하여, 연도별 탭, 과목별 필터링, 이미지 모달 팝업 등의 동적 기능이 포함된 완성된 HTML 파일을 렌더링합니다.
-   **입력**: 모든 연도의 `{year}_data.json` 파일, `templates/index.html`
-   **출력**: `index.html`

### 4. `page_mapping_validator.py` - (개발자용) 검증 유틸리티

-   **목적**: `pdf_analyzer.py`의 분석 결과(JSON)가 얼마나 정확한지 검증하고 디버깅하기 위한 개발자용 보조 스크립트입니다.
-   **핵심 로직**:
    -   `pdf_analyzer.py`가 생성한 `JSON` 데이터와 실제 PDF 파일의 텍스트 내용을 교차 검증합니다.
    -   `JSON`에 있는 문항이 실제 PDF 페이지에 없는 경우, 또는 그 반대의 경우를 찾아냅니다.
    -   검증 보고서를 생성하고, 잠재적으로 수정된 `JSON` 파일을 생성하여 분석 로직 개선에 활용될 수 있습니다.
-   **참고**: 이 스크립트는 `main.py`를 통해 실행되는 자동화 파이프라인에는 포함되지 않으며, 분석 알고리즘의 정확도를 높이기 위한 목적으로 별도로 사용됩니다.
