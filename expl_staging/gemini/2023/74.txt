정답: 4 (④)

**해설:**
전통 꼰대 오라클, MySQL 족보 **(관계형 DB / RDBMS)** VS 탈중앙 유연성 트렌드 **(NoSQL 진영)** 의 비교 특성 표 갈라치기 평가전입니다. 

**상세 분석:**
* **(가) 스키마 없음(Schemaless), 변경 프리:** RDBMS는 테이블 만들 때 VARCHAR(20) 안 맞추면 에러 뱉고 튕기지만, NoSQL(몽고DB 문서 등)은 꼴리는 대로 JSON 필드 더 넣어도 걍 저장됨. ➔ **NoSQL 특성 O**
* **(나) 클러스터 환경(분산 Scale-Out 수평확장) 떡상:** RDBMS는 덩치(Scale-UP, 램 올리기) 키워서 서버비 깨지지만, NoSQL은 싸구려 서버 100대 병렬로 옆으로 늘어놓고 파일 떨구기에 아키텍처가 최적화 되어있음. ➔ **NoSQL 특성 O**
* **(다) 검색 단순 무식함:** RDBMS는 JOIN 10개 묶어서 지독한 통계 뽑기에 능하지만, NoSQL은 분산되어 있어서 Key 주면 Value 딱 하나 물고 오는 단순 총알 검색 특화임. 복잡 쿼리 혐오증. ➔ **NoSQL 특성 O**
* **(라) 트랜잭션 일관성 포기 선언 (ACID 안녕, BASE 환영):** RDBMS가 무결성 도장 깨기에 미쳤다면, NoSQL은 "은행 아니면 가끔 싱크 안 맞아도 돼(궁극적 일관성, Eventual Consistency)" 라며 속도를 택함. ➔ **NoSQL 특성 O**
* **모조리 다 NoSQL(MongoDB, 카산드라, 하둡 Hbase 등) 찬양 설명이므로 가,나,다,라 몽땅 정답 ④입니다.**
