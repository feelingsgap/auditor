정답: 2 (②)

**해설:**
웹 프로그래밍의 밥줄 통신 규약, **RESTful HTTP 메소드 장인 규격**에서 뱃속 깊숙이 짐을 숨겨 옮기는 당나귀 녀석을 찾는 문제입니다. 

**상세 분석:**
* **지령:** "신규 데이터를 등록(Create/Insert) 하고 싶은데, url 뒤에 너저분하게 물음표 달고 꼬리표로 보내기 싫음. **무조건 HTTP 메시지 '몸체(Body)' 등딱지 안에 파일이나 엄청난 데이터를 숨겨 넣고 밀봉해 서버로 은밀하게 던져야 함.**"
* **① GET:** 데이터 요청 열람실. URL 꼬리표(Query String)에 검색어 다 노출시켜서 보냄. 보안 허벌, 무거운 파일 못 올림. (Read 특화)
* **④ HEAD:** 얌체공. GET이랑 똑같은데 본문 다운은 통신비 아까우니까 패스, 헤더(수정 날짜 등 껍데기)만 보여달라는 핑(Ping) 테스트용 얌체.
* **③ DELETE:** 서버 암살자. "3번 게시글 삭제점" 명령어만 날림 특화.
* **[밀반입 스페셜리스트] ② POST:** 보안 문서 캡슐 등 덩치 큰 첨부파일이나 장문의 일기 쓰기 데이터 등 "가공 안 된 찐 데이터 뭉탱이"를 HTTP Body 본문 배딱지 안에 꼭꼭 숨겨 담아 백엔드 서버 DB로 **Create (생성/등록)** 뚝배기를 깰 때 무조건 타는 전용 리프트 메소드입니다. 진리입니다.
