정답: 3 (③)

**해설:**
형상 관리의 지배자, **Git 명령어** 핵심 동작의 원리를 묻고 있습니다.

**상세 분석:**
* **① git push:** (반대 방향 설명) 내 PC 로컬에서 짠 코드를 Github(원격)으로 우주선 쏘듯 밀어 올리는 명령입니다. 
* **② git fetch:** (반쪽짜리 설명) 원격 저장소에 남들이 바꾼 걸 내 PC로 몰래 다운만 받아오고(관람 모드), 내 코드와 병합(Merge)시키는 코딩 충돌 액션은 절대 하지 않는 소심한 명령입니다.
* **[정답 추출] ③ git pull:** 
  ➔ `git fetch (다운로드) + git merge (내 파일과 합체 융합)` 두 가지가 원소 스킬로 결합된 궁극기입니다. 깃허브 최신 데이터를 내 컴퓨터로 가져오면서 내 코드와 바로 섞어버립니다. (정확한 설명)
* **④ git clone:** (너무 과한 설명) 빈 깡통 PC에 처음 입사했을 때 Github 통째로 복사해오는 명령이지, 충돌 병합과는 아무 상관 없습니다.
