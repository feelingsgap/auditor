정답: 2 (②)

**해설:**
트랜잭션 격리성(Isolation Level)이 무너질 때 생기는 3대 개판 대참사(Dirty, Non-repeatable, Phantom) 중 **'도깨비 신입사원' 귀신 목격 증후군**을 찾아내는 상황극입니다.

**타임라인 밀착 취재:**
* T1: (통계부서) "통영에 20대 사는 사람 몇 명이야? SELECT COUNT" ➔ 100명 나옴. 
* T2: (영업부서 끼어듦) "어? 통영 사는 김민재(20대) 신규 회원가입 폼 들어왔다! INSERT 밀어 넣고, COMMIT 도장 쾅쾅!"
* T1: (통계부서 2차 쿼리) "다시 나이별 인원수 세볼까. SELECT COUNT" ➔ 엇 시발? 1초 전엔 없던 101명으로 투표수가 불어났잖아?! 
* **[판사 판결] Phantom Read (유령/환상 데이터 읽기 현상):**
  ➔ 한 트랜잭션(T1) 안에서 똑같은 다중 행 검색(COUNT 나 범위 SELECT)을 2번 날렸을 뿐인데, 그 짧은 찰나에 남(T2)이 뒤쪽으로 **"INSERT(신규 새 데이터 밀어 넣기) 하거나, DELETE(폭파시켜버림)"** 하고 튀어서 총 행(Row) 개수가 귀신처럼 나타났다가 사라졌다 하는 극강의 호러 환선 현상을 뜻합니다.
  *(참고: 특정 행 데이터 값을 100원에서 200원으로 슬쩍 바꾼 건 Non-Repeatable Read 라고 부릅니다. 이 문제는 행 개수가 추가되는 문제이므로 유령 데이터 읽기 즉, Phantom Read가 적합)* 정답 ②.
