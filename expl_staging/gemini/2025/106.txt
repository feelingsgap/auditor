정답: 3 (③)

**해설:**
사용자가 브라우저에 네이버 주소를 제대로 쳤는데도 엉뚱하게 해커의 가짜 농협 피싱 사이트로 납치되는, 악명 높은 **DNS 스푸핑(Spoofing / 캐시 포이즈닝)** 해킹 공격의 더러운 동작 매커니즘을 파헤칩니다.

**상세 분석:**
* **① (O):** DNS 체계는 "야, 네이버 주소 IP가 수첩에 뭐로 적혀있어?" 라고 물을 때 답장 타이밍이 빨라야 하므로 보안 연결/확인 스텝업이 싹 거세된 UDP(53번 포트)를 기본통신으로 씁니다. 그래서 해커가 중간에서 가짜 응답을 확 끼어 쏴버리면 클라이언트가 진짜인 줄 깜빡 속아 넘어갑니다.
* **② (O):** 해커가 아무 때나 쏘는 게 아니라, 사용자가 언제 질의를 쏘는지 그 패킷의 트랜잭션 ID 시리얼을 정확히 훔쳐봐야 짝을 맞출 수 있으므로 질의망을 스니핑으로 몰래 엿듣고 대기해야 합니다.
* **④ (O):** 이 공격이 성공하면 피해자는 주소창에 www.naver.com 을 정상적으로 오타 하나 없이 똑바로 쳤어도 접속되는 종착역의 IP 좌표가 해커 서버 IP로 맵핑되어 가짜 낚시 창에 떨어집니다.
* **③ (정답/용어 오답 날조):** "DNS 통신 서버의 캐시 정보를 **삭제(Delete)** 하는 방법을 이용한다?" 
  * 이 해킹은 흔적이 안 남게 정보를 지우는 도스 파괴 테러 공격이 아닙니다. 정상적인 IP 대신에 해커가 "야야! 아까 물어봤던 네이버 서버 IP 저기 해커 서버 아이피 요놈이야!" 하면서 **악성 구라 IP를 피해자의 브라우저나 라우터 DNS 캐시 메모리에 아주 더럽게 감염·오염 주입(Poisoning/변조 삽입)** 시켜버리는 더티 낚시 수법입니다. 캐시 정보가 고갈 삭제되는 게 아니라 더러운 오물 정보로 왜곡 덮어씌워집니다.

**📌 DNS Spoofing 핵심 요약:**
* DNS 스푸핑의 메인 기법 행위 = 내 로컬 캐시 메모리 데이터의 소거 삭제(X)가 아니라, 가짜 피싱 IP 주소를 강제로 쑤셔 넣어 영구 저장시키는 **캐시 오염/독점 포이즈닝(Cache Poisoning) 테크닉이다.**
