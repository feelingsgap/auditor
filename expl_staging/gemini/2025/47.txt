정답: 3 (③)

**해설:**
RESTful(Representational State Transfer) 기반 웹 아키텍처에서 조작하고자 하는 리소스의 행위에 가장 적합한 HTTP 메서드(동사)를 짝지은 것을 고르는 문제입니다.

**상세 분석:**
RESTful API 관례에서 리소스(자원)에 대한 CRUD 작업은 지정된 HTTP 메서드에 1:1로 매핑됩니다.
* **가. 회원정보 조회 (READ):** HTTP 프로토콜 자체에 `READ`라는 비표준 동사는 존재하지 않습니다. 자원 조회는 무조건 **`GET`** 메서드를 써야 합니다. (부적절)
* **나. 회원가입 (POST):** 백엔드 DB에 새로운 User 자원을 파생/생성(Create)하는 행위이므로, 본문(Body)을 태워 보내는 **`POST`** 메서드가 정확합니다. (적절)
* **다. 회원정보 수정 (UPDATE):** 이 역시 HTTP 프로토콜에 `UPDATE`라는 동사는 없습니다. 자원의 전체 덮어쓰기 교환은 **`PUT`**, 자원의 부분적 패치 수정은 **`PATCH`**를 사용해야 합니다. (부적절)
* **라. 회원탈퇴 (DELETE):** 삭제(Delete) 행위는 HTTP에 이미 내장된 명시적인 **`DELETE`** 메서드를 사용합니다. 매우 직관적이고 적절합니다. (적절)

따라서 표준 HTTP Method 규칙을 온전히 지킨 그룹은 (나), (라)입니다.

**📌 핵심 요약:**
* RESTful 4대 천왕: 생성(`POST`), 조회(`GET`), 수정(`PUT` 또는 `PATCH`), 삭제(`DELETE`).
* `READ`, `UPDATE`는 명세에 없는 임의의 단어일 뿐 아키텍처 메서드가 아닙니다.
