정답: 4 (④)

**해설:**
IT 소프트웨어 생태계에서 날아다니는 네트워크 API 통신 및 파일 백업용 주요 표준 Data Format(데이터 구조 포맷) 들의 특징적 생리와 호환 한계점을 매칭 시켜 진단하는 문항입니다.

**상세 분석:**
* **① Key-Value Store:** Redis 같은 램 상주 NoSQL 모델에서 인덱스 번호표(Key) 하나 딱 찌르면 즉석에서 값(Value)이 번개처럼 팍 튀어나오는 초강력 해시맵 테이블 단방향 구조. 검색 빠름의 진수입니다. 적절한 설명 (O).
* **② JSON:** {} 중괄호로 묶고 "속성명:값" 딱지를 붙여 배열하며, 마트료시카 인형처럼 속 안으로 트리처럼 내부를 무한 중첩으로 겹겹이 계층을 쌓아 파고들어 담는 현대 통신 망 개발 표준 1티어 포맷. 아주 유연하고 범용적입니다. 적절한 설명 (O).
* **③ XML:** 거칠고 뾰족한 태그 껍질인 `<부모> <자식> 내용 </자식> </부모>` 형태를 통해 트리 계층을 만들어내는 무거운 구조 포맷입니다. JSON이 세상을 먹기 이전 시대의 레거시 정부 공공망 연동이나 안드로이드 UI 뼈대 레이아웃 등에 많이 쓰입니다. 확실한 깊이(Depth) 중첩 구조 지원. 적절한 설명 (O).
* **④ CSV (정답, 부적절한 해설 검출):** 흔히 스프레드시트나 엑셀 다운로드 뽑을 때 쓰는, 문장 중간을 오직 Comma(쉼표)로만 칼같이 쪼개서 가른 가장 저렴하고 플랫한 깍두기 배열(직사각형 행/열 깡그라미) 표장 형식입니다. 매우 정직하게 납작한 2차원 평면 매트릭스 표로만 구현됩니다. JSON이나 XML처럼 A 품목 데이터 안에 B 객체가 종속되어 있고 다시 B 객체 뱃속에 C 속성 배열이 안겨 있는 깊은 차원의 입체적 **'중첩(Nested) 계층 구조 뎁스'를 CSV는 표현해내는 능력 자체가 태생 유전자 배열 로직적으로 아예 결여되어 불가능합니다!** 고로 중첩 표현 가능이라는 뒷말은 판타지 소설 구라입니다.

**📌 직관적 핵심 요약:**
* JSON (제이슨) & XML = 부모 밑에 자식 밑에 손자 형태의 깊은 3차원 입체 다층 중첩구조 트리 지원 특화.
* CSV = 납작하고 플랫 밋밋한 2차원 표 형식 온리, 중첩(Nested) 트리 불가.
