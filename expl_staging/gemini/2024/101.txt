정답: 1 (①)

**해설:**
리눅스 시스템에서 현재 누가 접속해 있는지, 계정 정보가 뭔지 염탐해서 **해킹의 첫 단추인 '정보 수집(사전 조사)'** 에 자주 악용되는 추억의 명령어(서비스)를 묻습니다.

**상세 분석:**
* **① finger:** '손가락으로 찌르다'. 외부에서 해당 시스템에 접속 중인 사용자의 계정 이름, 실제 이름, 터미널 위치, 심지어 마지막 로그인 시간까지 시시콜콜하게 다 털어주는 친절한 데몬 서비스(`fingerd`)입니다. 친절함이 보안에는 독이 되기 때문에, 공격자가 계정 존재 여부를 파악하는 사전 정찰(Reconnaissance)에 1순위로 악용되어 현재는 대부분 보안 설정에서 아예 포트 79번을 닫거나 서비스를 죽여놓습니다. (정답)
* 오답 체크:
  * ② echo ➔ 그냥 터미널 메아리 출력 명령어.
  * ③ NFS ➔ 네트워크 드라이브 공유 마운트. (사용자 신상정보 염탐이 주 목적은 아님)
  * ④ talk ➔ 과거 유닉스 사용자끼리 터미널 창 반 가르고 쓰던 1:1 실시간 채팅 명령어.

**📌 리눅스 계정/자격증명 염탐기:**
* `finger` ➔ 계정 털기 (보안 권고: Disable fingerd)
* `who`, `w`, `users` ➔ 내부에서 접속자 확인
