정답: 1 (①)

**해설:**
Type Code(비디오 종류 번호, 할인 행사 종류 번호)가 한 클래스 내에 난잡하게 하드코딩 되어 있을 때, 이를 잘게 써는 다형성 지향 **리팩토링 구조 설계도(다이어그램)** 를 묻습니다.

**상세 분석:**
* **기존 구조:** 뚱뚱한 비디오 클래스 하나 안에 "너 VHS니 CD니?", "너 할인 가격이니 오리지널 가격이니?" 라며 조건문 if else 만 수십 개가 들어찬 쓰레기 코드입니다.
* **리팩토링 지향점:** 이런 무의미한 분류 번호를 뜯어내서 분리시키는 **'분류 코드 서브클래싱화(Replace Type Code with Subclass/State)'** 작업을 해야 합니다.
* 비디오 본체는 내버려 둔 채, 가격(Price) 계산 로직을 관장하는 다형성 부모를 찍어내고, 그 아래에 놈팽이 할인(Discount)과 정가(Normal)를 쪼개 넣습니다. 또 비디오 매체 종류도 분리해야 합니다.
* **①번 그림 해석:** 정답이 되는 다이어그램의 특징은 두 개의 독립적인 분류 차원(물리적 포맷과 가격 정책)을 철저히 독립된 계층 트리로 팩토리 분리하여 객체 간소화를 달성한 정석 모델입니다.

**📌 리팩토링 법칙 요약:**
* 독립적인 조건 분류 개념은 한 클래스에 짬뽕하지 않고 별도의 상태(State) 클레스로 뽑아내어 다형성을 먹인다.
