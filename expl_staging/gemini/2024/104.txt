정답: 3 (③)

**해설:**
해커들이 게시판 첨부 파일에 악성 웹셸(.jsp, .php)을 숨겨 올려서 서버를 터트리는 공격을 막는 방어법 중, **"오히려 이러면 서버를 해커에게 갖다 바치는 꼬라지"** 인 자살골 대응법을 찾는 문제입니다.

**상세 분석:**
* 공격의 메커니즘을 봅시다. 해커가 악성 스크립트를 올립니다. 그리고 자기가 올린 파일 경로로 (예: `http://서버/upload/hack.jsp`) 접속하면 서버가 그걸 실행시켜줘서 해킹이 완료됩니다.
* 방어 수단은 다음과 같습니다:
  * ① 실행 속성 제거. (실행 불가 맹물로 만듦) O
  * ② 화이트 리스트 기반 확장자(.jpg, .pdf) 통과. O
  * ④ 파일명을 랜더밍하여(`sdjf89w23r.a`) 해커가 주소를 유추하여 못 찾아오게 함. O
* **[자살골] ③ "파일 저장 경로를 웹 서버 접속 가능한 문서 디렉터리(Web Root) 내부에 한정시킨다"**
  * 이렇게 하면 해커가 브라우저 URL 창을 통해서 자기가 올린 악성 파일에 다이렉트로 접속하여 실행 버튼을 누를 수 있게 웹 통로를 열어두는 꼴입니다.
  * **정석 대응:** 첨부 파일은 아예 웹 통신(URL 접속)이 뚫리지 않는 **'외부(최상위 파일 시스템 구조 뒷단)'나 아예 분리된 스토리지 서버에 격리 보관**해야 안전합니다. 따라서 3번 대책은 틀렸습니다.

**📌 파일 업로드 취약점 방벽:**
* 절대 웹 루트(Web Root) 디렉터리 내부에 업로드 파일을 저장하지 마라!
