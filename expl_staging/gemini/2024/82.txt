정답: 1 (①)

**해설:**
CPU가 키보드, 마우스 같은 짤짤이 타건에 매번 신경 쓰느라 본업(연산)을 못 하는 사태를 막기 위해, **대형 화물(대용량 데이터) 전송 업무를 전담하는 '어둠의 하청업체 버스 기사'를 고용하는 기술**입니다.

**상세 분석:**
* 낡은 방식(Programmed I/O 또는 Polling): CPU가 직접 디스크에서 바이트 하나 퍼서 메모리로 나릅니다. 우주 최고급 두뇌가 택배 상하차를 직접 하는 꼴입니다.
* **해결책 DMA (Direct Memory Access):**
  * CPU가 DMA 제어기(하청 소장)에게 "이 하드디스크의 10GB짜리 영화 파일, 메모리 1번 구역으로 다 옮겨놔. 다 끝나면 톡(인터럽트) 날리고." 라고 지시만 내립니다.
  * 그럼 CPU는 하차장에서 빠져나와 딴 일을 하고, DMA 장치가 메인 버스를 장악하여 **자기가 직접 I/O 기기에서 메모리로 데이터를 고속 펌프질** 해버립니다.
  * 따라서 "CPU 개입 없이", "기억장치와 I/O 장치 간" 이라는 키워드만 나오면 묻지도 따지지도 않고 **DMA**가 정답입니다.

**📌 I/O 제어의 진화 단계:**
1. Polling (CPU가 직접 계속 감시 대기)
2. Interrupt (CPU가 딴짓하다 부르면 가서 처리)
3. **DMA (CPU 지시 한 방으로 대규모 화물 직접 이송)**
4. 채널/IOP (독자적인 초소형 I/O 전용 프로세서 탑재)
