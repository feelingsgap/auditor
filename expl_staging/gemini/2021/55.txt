정답: ②

**해설:**
데이터베이스 가상 논리 보안 테이블 껍데기 조망인 뷰(VIEW) 컴포넌트에 대한 생성과 제약 DDL(데이터 정의어) 지식입니다.
*   **(가) 참:** 무작정 Select 조회용이 아니라 뷰를 바탕으로 하위 원본 테이블에 Insert 삽입을 꽂을 때, 복잡한 다중 조인이 섞였거나 주요 낫널 PK값이 누락된 억지 뷰 기반 삽입 DML 시도는 시스템DBMS 차원에서 에러를 뱉고 단호하게 거절 거부 차단됩니다.
*   **(다) 참:** `SUM(매출), AVG(나이)` 같이 여러 데이터의 진액을 짜내어 통계 압축한 집계함수로 범벅이 되어 가상으로 만들어진 집계 뷰 레코드 표 셀에 포커스를 두고, "아 나 이 나이 평균 50을 100으로 업데이트 고칠래!" 라며 뷰 속에서 혼자 역으로 갱신 수정을 시도 허용하는 것은, 미분적분 원본 값을 어떻게 역산 맵핑 분배 배정할지 모순적 불가역 논리 파괴가 일체 수반되므로 DBMS는 이를 원천 봉쇄하여 기본적으로 절대 갱신을 불허합니다.
*   *(나) 오답거짓:* 뷰 생성 CREATE 명령어 작성 시 뼈대에 `GROUP BY` 그룹 통계 구문을 맘껏 문법적으로 넣어서 조회용 뷰로 만드는 것은 얼마든지 합당하고 자유롭게 가능합니다. (단지, 그걸 기반으로 역방향 수정/갱신 업데이트가 제한될 뿐, 생성 자체는 합법입니다)
*   *(라) 오답거짓:* 구체적인 실체화 캐싱 하드웨어 디스크 복제 뷰인 Materialized View(MView)는 말 그대로 `CREATE MATERIALIZED VIEW`라는 지 전용 DDL 생태계 구문명령 문법을 쓰며, 일반 뷰는 `CREATE VIEW` 씁니다. CREATE TABLE 테이블 껍데기를 꺼내다 설명하는 4번 설명 부는 형편없는 헛소리입니다.
