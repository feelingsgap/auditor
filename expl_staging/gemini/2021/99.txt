정답: ①

**해설:**
최신 경량 클라우드 컴퓨팅 트렌드인 TTAK 국산 오픈소스 서버리스(Serverless) 프레임워크(Knative/도커 등) 도입 구축 가이드라인 흐름입니다.
*   **가.** 빌드된 컨테이너 소프트웨어 이미지들을 한데 착착 예쁘게 모아 보급 적재하는 '도커허브' 같은 이미지 창고, 저장소를 **컨테이너 레지스트리(Registry)**라 합니다. (㉮)
*   **나.** 이 배포된 기능 함수 컨테이너들을 살리면 올려주고, 죽으면 부활시키고, 교통정리 분배하는 쿠버네티스 같은 '군사 지휘 운영 지휘자' 를 **컨테이너 오케스트레이터(Orchestrator)**라 합니다. (㉯)
*   **다.** 운영 클러스터 부대 편제는, 명령 하달과 API 감시, 스케줄을 총괄 통치하는 컨트롤 코어가 속한 **마스터 노드 (Master Node / ㉰)**와, 이 지시를 받아 직접 컨테이너의 파드(Pod) 기능을 짐 짊어지고 구동하여 일을 하는 **워커 노드 (Worker Node / 일꾼 / ㉱)** 그룹으로 확고히 구분 편제됩니다.
순서의 이 흐름을 가장 아름답게 조합된 것이 1번입니다.
