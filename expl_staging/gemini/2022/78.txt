정답: ②

**해설:**
통신 프로세스 중 데이터 변환 매핑 방향으로, MAC 로컬 하드웨어 주소를 IP 라우팅 논리 주소로 해석하여 뒤바꿔주는 구조를 찾는 문제입니다.

**통신 프로토콜 분류:**
*   **RARP (Reverse ARP): (정답)** 하위 전송 단위 주소인 고유한 48bit 물리 주소, 즉 `데이터 링크 통신 주소인 MAC 주소 정보`를 토대로 → DHCP를 사용하지 않는 디스크 제어기기나 논리 단말이 `소속된 상위 IP(네트워크 계층) 주소값을 매핑 할당 요청하여 획득하기 위한 해석 프로토콜`입니다. ARP 구동을 역순행으로 변환합니다.
*   **ARP:** 가장 보편적으로 논리적인 송/수신 호스트들의 IP 주소(네트워크 계층) 명령을 매핑 기반하여 → 하드웨어 목적지 물리 카드 MAC 주소(데이터 링크 계층)로 확인·매핑(변환)해줍니다.
*   **ICMP:** 시스템 ping 명령처럼 IP 프로토콜 오류와 네트워크 연결 패킷 전송 가능 여부(진단) 등을 제어해줍니다.
*   **NAT:** 사설 내부망 IP를 대외적인 공개형(공인망) 외부 통신 IP 규격으로 주소 변환(통역)시킵니다.
