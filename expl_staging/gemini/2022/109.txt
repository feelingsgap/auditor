정답: ①

**해설:**
리눅스 서버 인프라의 표준 방화벽 제어 프레임워크인 **`iptables`** 의 커맨드 라인 제어 룰 구문을 조작하는 문제입니다.
문제상황: `[ 192.168.20.22 이라는 특정 IP로부터 들어오는(수신, Input) 트래픽을 방어 차단(드랍)하라 ]`

**iptables 명령어 구문 작성 법칙:**
*   명령 실행 선언 체인: 보통 정책 추가는 룰셋 아래에 붙이는 Append 이므로 **(`-A`)**
*   어떤 데이터 방향 체인인가?: 서버 내부로 인입되는 데이터 길이므로 **`INPUT`** 
*   출발지/송신지 소스 파밍: Source Address를 명명해야 하므로 (`-s` *생략된 보기*) IP 주소 `192.168.20.22` 적용
*   조건 매칭 시 액션 행위 점프 체인: 목표한 대로 가차 없이 폐기 차단시켜야 하므로 **(`-j DROP`) 혹은 (`-j REJECT`)**

이 조건들을 모두 준용하여 완벽히 방화벽 차단 명령어 정책을 부합하게 서술한 지문은 `iptables -A INPUT 192.168.20.22 -j DROP` 인 **①번** 항목뿐입니다. (3번 ACCEPT는 허용 문구이므로 반대임)
