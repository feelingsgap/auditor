**정답: ① replace error code with exception (오류 코드를 예외로 대체)**

**해설:**
리팩토링 카탈로그에서 오류 처리 방식을 개선하는 기법입니다.
* **적용 전:** 예전 전통적인 C 스타일 오류 관리로, 논리적 오류가 발생했을 때 특이한 정수 리턴값(예: `-1`)을 밖으로 던져 호출자가 일일이 if문 리턴값으로 에러를 잡아내야 했습니다.
* **적용 후:** Java 등의 객체지향적 에러 핸들링인 `throw new Exception` 메커니즘을 적용했습니다. 이제 메소드는 `void`로 정상 실행 시 리턴을 없애 깔끔해졌으며, 에러 발생 시 명시적으로 예외 객체를 던져 시스템 상위 호출 레벨 예외 체인에서 깔끔하게 잡도록 했습니다.
이 리팩토링의 정식 명칭은 **Replace Error Code with Exception(오류 코드를 예외로 전환)**입니다.
