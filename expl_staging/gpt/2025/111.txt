정답: ② (2번)

해설:
Prepared Statement와 Stored Procedure가 방어할 수 있는 취약점을 묻는 문제다.

이 기법들은 사용자 입력을 SQL 구조와 분리해 처리하므로 `SQL 삽입` 공격을 막는 데 효과적이다.

각 보기 분석:
① 쿠키 변조 — 직접적인 방어 대상이 아니다.
② SQL 삽입 — 매개변수 바인딩과 절차형 호출로 방어할 수 있으므로 정답이다.
③ 파일 업로드 — 별도의 검증과 제어가 필요하다.
④ 취약한 비밀번호 — 인증 정책 문제다.

핵심 정리:
Prepared Statement와 Stored Procedure는 입력값을 SQL 구문과 분리해 SQL Injection을 줄인다.
