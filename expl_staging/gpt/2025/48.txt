정답: ① (1번)

해설:
경비시스템에서 주간/야간 조건 분기를 별도 클래스로 분리하는 방식으로 바꾸는 데 적용된 설계 패턴을 묻는 문제다.

변경 전에는 `if (주간)`, `else if (야간)`처럼 상태에 따라 분기한다.
변경 후에는 주간 클래스와 야간 클래스가 각각 같은 메서드 집합을 구현하면서 상태별 행동을 분리한다.

이는 상태를 객체로 캡슐화해 상태에 따라 동작을 바꾸는 State Pattern의 전형적인 구조다.

각 보기 분석:
① State Pattern — 정답
② Adapter Pattern — 인터페이스 변환 목적이라 다르다.
③ Builder Pattern — 객체 생성 절차 분리에 관한 패턴이다.
④ Template Method Pattern — 알고리즘 골격 정의와는 다르다.

핵심 정리:
조건 분기를 상태별 클래스로 치환해 행동을 분리하면 State Pattern으로 본다.
