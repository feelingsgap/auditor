정답: ①

해설:
오픈소스 라이선스별 의무 조건을 이해하고 라이선스 침해 여부를 판단하는 문항이다.

가. GPL3.0 라이브러리를 정적 링크로 연결, 상용 배포, 소스 미공개
→ 라이선스 침해. GPL3.0은 강한 Copyleft로 정적·동적 링크 불문 소스 공개 의무 발생.

나. LGPL3.0 라이브러리를 정적 링크로 연결, 상용 배포, 소스 미공개
→ 침해로 단정 불가. LGPL 3.0 §4(Combined Works)는 정적 링크 시 "재링크(re-linking) 가능성 보장"을 요구하며, 이는 앱 소스 공개 또는 오브젝트 파일 제공 중 택1로 충족된다. 앱 소스 자체를 공개할 의무는 없으므로, "소스 미공개"만으로 침해라 할 수 없다.

다. Apache2.0 라이브러리를 정적 링크로 연결, 상용 배포, 소스 미공개
→ 침해 아님. Apache2.0은 Permissive 라이선스로 소스 공개 의무 없음.

각 보기 정리:
가 → 침해 ✓
나 → 침해 아님 (LGPL §4의 재링크 보장 의무는 소스 공개 없이도 충족 가능)
다 → 침해 아님

① 가 → 정답.
② 가, 나 → LGPL 정적 링크를 GPL과 동일시한 오해. 오답.
③ 가, 나, 다 → Apache2.0은 Permissive. 오답.
④ 모두 해당 없음 → 가는 GPL 위반. 오답.

핵심 정리:
- GPL: 어떤 방식으로 링크해도 결합저작물 전체 소스 공개 의무.
- LGPL §4: 앱 오브젝트 파일 제공(또는 앱 소스 공개) 택1로 재링크 가능성 보장 = 의무 충족. 소스 비공개 자체는 허용.
- Apache2.0: Permissive, 소스 공개 의무 없음.
- 공식 정답 ①. "LGPL 정적 링크 = GPL 동일"은 일부 교재 오류이며 원문과 불일치.
