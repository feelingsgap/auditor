정답: ④ (4번)

해설:
아키텍처 분석/평가 방법 중 비용-편익 분석을 기반으로 아키텍처 전략을 선정하는 절차를 묻는 문항이다. 입력으로 아키텍처, 아키텍처 전략들, N개의 품질속성 시나리오를 받고, 단계별로 시나리오 정리 → 제제 → 우선순위 → 효용값 부여 → 전략 도출 → 보정 → 총 이득 계산 → 투자수익률 기반 선택 → 직관 확인 순서를 거친다.

각 보기 분석:
① SAAM(Software Architecture Analysis Method) — 시나리오 기반 아키텍처 평가 방법으로, 직접/간접 시나리오를 통해 아키텍처를 평가하나 비용-편익 계산 절차가 없다.
② ATAM(Architecture Tradeoff Analysis Method) — 품질속성 간 트레이드오프를 분석하는 방법으로, 위험/비위험/감도점/트레이드오프점을 도출한다. 투자수익률 계산 절차는 없다.
③ ARID(Active Review for Intermediate Designs) — 설계 초기 단계의 능동적 검토 방법. 비용-편익 분석과 관련 없다.
④ CBAM(Cost Benefit Analysis Method) — 비용-편익 분석 방법으로, 품질속성 시나리오별 효용값과 비용을 산정하여 투자수익률(ROI) 기반으로 아키텍처 전략을 선택한다. 제시된 절차와 정확히 일치. 정답.

핵심 정리:
CBAM: ATAM을 보완하며 품질속성 개선의 경제적 가치를 정량화.
핵심 절차: 시나리오 선정 → 효용값 부여 → 비용 산정 → ROI 계산 → 전략 선택.
ATAM은 트레이드오프 분석, CBAM은 비용-편익(ROI) 기반 선택이라는 점에서 구분된다.
