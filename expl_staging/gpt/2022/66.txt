정답: ④ (4번)

해설:
지연 갱신(deferred update)은 트랜잭션 완료(commit) 전까지 실제 DB에 반영하지 않고 로그에만 기록하는 방식이고, 즉시 갱신(immediate update)은 트랜잭션 실행 중에 바로 DB에 반영하는 방식이다.

각 설명 분석:
가. 지연 갱신에서 commit 이전 실패 시 UNDO 불필요 — 맞다. 아직 DB에 반영된 내용이 없으므로 별도 취소 작업이 불필요하다.
나. 즉시 갱신은 commit 이후 UNDO와 REDO가 불필요하여 NO-UNDO/NO-REDO — 틀렸다. 즉시 갱신은 commit 이전 실패 시 UNDO가 필요하고, 장애 후 완료된 트랜잭션에 REDO가 필요할 수 있다. NO-UNDO/NO-REDO는 지연 갱신 중에서도 특수한 경우에 해당한다.
다. 즉시 갱신에서 commit 이전 실패 시 UNDO 필요 — 맞다. DB에 이미 반영된 내용을 되돌려야 한다.
라. 지연 갱신에서 완료된 트랜잭션에 REDO 필요할 수 있다 — 맞다. commit은 되었으나 로그의 내용이 DB에 완전히 반영되기 전에 장애가 발생한 경우 REDO가 필요하다.

① 다, 라 — 나가 틀린 설명임을 반영하지 못하고 가를 누락했다.
② 나, 라 — 나는 틀린 설명이다.
③ 나, 다, 라 — 나는 틀린 설명이다.
④ 가, 다, 라 — 세 설명 모두 옳다. 정답.

핵심 정리:
지연 갱신: commit 전 실패 시 UNDO 불필요, commit 후 장애 시 REDO 필요.
즉시 갱신: commit 전 실패 시 UNDO 필요, commit 후 장애 시 REDO 필요.
NO-UNDO/NO-REDO는 지연 갱신의 특정 구현에서만 해당된다.
