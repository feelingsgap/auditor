정답: ① (1번)

해설:
PMBOK의 프로젝트 범위관리(Scope Management)에서 범위 계획 프로세스가 생성하는 산출물들의 생성 순서를 묻는 문항이다. 범위 계획은 요구사항 수집 → WBS 작성 → 일정 계획 → 변경 통제 순으로 진행된다.

각 산출물의 특성과 순서:
- 요구사항문서(Requirements Documentation): 이해관계자 요구를 수집·정의한 결과물. 가장 먼저 생성.
- 작업분류체계 사전(WBS Dictionary): WBS 각 구성요소에 대한 상세 설명. 요구사항문서를 기반으로 WBS 생성 후 작성.
- 활동 목록(Activity List): WBS를 분해하여 일정 계획 시 생성. WBS 사전 이후.
- 변경 요청서(Change Request): 프로젝트 실행 또는 통제 중 발생. 가장 나중.

각 보기 분석:
① 요구사항문서 → 작업분류체계 사전 → 활동 목록 → 변경 요청서 — 올바른 순서. 정답.
② 요구사항문서 → 활동 목록 → 작업분류체계 사전 → 변경 요청서 — 활동 목록과 WBS 사전 순서가 바뀌었다.
③ 활동목록 → 작업분류체계 사전 → 요구사항문서 → 변경 요청서 — 요구사항문서가 가장 먼저이므로 오류.
④ 작업분류체계 사전 → 활동목록 → 요구사항문서 → 변경 요청서 — 순서 전체가 잘못되었다.

핵심 정리:
범위관리 산출물 생성 순서: 요구사항문서 → WBS → WBS 사전 → 활동 목록 → 변경 요청서.
WBS 사전은 WBS 작성 후 각 작업 패키지를 상세 기술한 문서이므로 활동 목록보다 먼저 생성된다.
