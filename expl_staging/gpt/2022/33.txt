정답: ④ (4번)

해설:
MC/DC(Modified Condition/Decision Coverage) 커버리지는 각 조건이 독립적으로 결정 결과에 영향을 미침을 보여야 한다. 주어진 코드의 결정 조건은 `(x > 0) && (y <= 0)`이다.

MC/DC 요건:
- 각 조건(x>0, y<=0)이 전체 결과를 독립적으로 변경하는 테스트 쌍이 필요하다.
- x>0 독립 영향 확인: y<=0이 고정된 상태에서 x>0의 True/False 변화로 결과가 바뀌어야 한다.
- y<=0 독립 영향 확인: x>0이 고정된 상태에서 y<=0의 True/False 변화로 결과가 바뀌어야 한다.

각 보기 분석:
① TC1: True - True — x>0=T, y<=0=T → 결정=True
② TC2: True - False — x>0=T, y<=0=F → 결정=False (TC1과 쌍으로 y<=0 독립성 확인)
③ TC3: False - True — x>0=F, y<=0=T → 결정=False (TC1과 쌍으로 x>0 독립성 확인)
④ TC4: False - False — x>0=F, y<=0=F → 결정=False. TC1+TC2+TC3만으로 이미 MC/DC 100%를 달성할 수 있어, TC4는 추가 커버리지를 제공하지 않는다. 따라서 최소 테스트 쌍에서 불필요하며 "100% 커버리지 달성에 적절하지 않은 것"이다.

핵심 정리:
MC/DC 최소 테스트 케이스: 조건 N개일 때 N+1개로 충분하다.
조건이 2개이므로 TC1, TC2, TC3 세 개면 MC/DC 100%를 달성한다.
TC4(False-False)는 기존 TC3과 결과가 동일하여 독립 영향 증명에 기여하지 않는다.
