정답: ① (1번)

해설:
짝수 패리티(even parity) 기법은 데이터 비트와 패리티 비트를 합쳐 1의 개수가 짝수가 되도록 패리티 비트를 설정한다. 패리티 비트는 MSB(최상위 비트)에 위치한다.

계산 과정:
대문자 A의 ASCII = 10진수 65 → 이진수 1000001
대문자 B = A + 1 = 66 → 이진수 1000010 (7비트 데이터)

대문자 B의 7비트 이진 표현: 1000010
1의 개수 세기: 1, 0, 0, 0, 0, 1, 0 → 1이 2개 (짝수)

짝수 패리티 조건: 패리티 비트 포함 1의 총 개수가 짝수이어야 함.
현재 1의 개수 = 2 (이미 짝수) → 패리티 비트 = 0

8비트 메모리 값 (MSB에 패리티): 0 1000010 = 0100 0010

각 보기 분석:
① 0100 0010 — 패리티 비트 0, 데이터 1000010. 1의 개수 2개(짝수). 정답.
② 0100 1010 — 1의 개수가 달라 B의 인코딩이 아니다.
③ 1100 0010 — MSB=1이면 1의 개수가 3개(홀수)로 짝수 패리티 위반.
④ 1100 1010 — 짝수 패리티와 B의 ASCII 모두 불일치.

핵심 정리:
짝수 패리티: 전체 1의 개수가 짝수 → 데이터 내 1의 수가 짝수이면 패리티 비트=0.
홀수 패리티: 전체 1의 개수가 홀수 → 데이터 내 1의 수가 짝수이면 패리티 비트=1.
ASCII 대문자: A=65, B=66, ..., Z=90.
