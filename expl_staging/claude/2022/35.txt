정답: ② (2번)

해설:
Git의 기본 개념과 명령어에 대한 옳고 그름을 판별하는 문항이다. 가~바 중 올바른 설명을 골라야 한다.

각 보기(가~바) 분석:
가. 브랜치(branch)는 커밋(commit)을 가리키는 포인터 — 올바른 설명이다. Git에서 브랜치는 특정 커밋 SHA를 가리키는 가변 포인터이다.
나. 마스터(master)는 최초 커밋 시 생성되며 커밋할 때마다 가장 최근 브랜치를 가리킨다 — 올바른 설명이다. master는 기본 브랜치로 새 커밋마다 자동으로 최신 커밋을 가리킨다.
다. 토픽 브랜치에서 수행한 작업은 커밋을 통해 자동으로 마스터에 병합된다 — 틀린 설명이다. 토픽 브랜치의 변경사항은 `git merge` 또는 PR(Pull Request)을 통해 명시적으로 병합해야 하며 자동 병합되지 않는다.
라. 마스터 브랜치로 이동하기 위해 `git checkout master`를 사용한다 — 올바른 설명이다.
마. 브랜치의 개개 커밋을 수행하면 단계적으로 브랜치 병합이 이루어진다 — 틀린 설명이다. 커밋은 병합을 일으키지 않는다.
바. `git stash`로 작업 내용을 임시 저장한다 — 올바른 설명이다.

각 선택지 분석:
① 가, 나, 다 — 다가 틀렸으므로 오답.
② 가, 나, 라 — 세 설명 모두 올바르다. 정답.
③ 나, 다, 마 — 다와 마가 틀렸으므로 오답.
④ 다, 마, 바 — 다와 마가 틀렸으므로 오답.

핵심 정리:
브랜치 병합은 자동이 아닌 명시적 merge/rebase 명령 필요.
git stash: 미완성 작업을 스택에 임시 저장, 나중에 git stash pop으로 복원.
git checkout [branch]: 브랜치 전환 명령.
