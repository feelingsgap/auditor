정답: ③ (3번)

해설:
방화벽 패킷 필터링 규칙 테이블을 분석하여 각 보기의 설명이 규칙과 일치하는지 확인하는 문항이다.

규칙 테이블 분석:
- 규칙 1: Internal → External, 서비스 80 → Allow (내부에서 외부 웹 접근 허용)
- 규칙 2: Any → 192.168.1.10, 서비스 80 → Allow (모든 곳에서 웹서버 192.168.1.10 접근 허용)
- 규칙 3: 192.168.1.11 → 10.10.10.21, 서비스 25 → Allow (내부 메일서버에서 외부로 SMTP 허용)
- 규칙 4: 192.168.1.11 → 10.10.10.21, 서비스 110 → Deny (내부 메일서버에서 외부로 POP3 차단)
- 규칙 5: 192.168.1.11 → 10.10.10.21, 서비스 143 → Allow (내부 메일서버에서 외부로 IMAP 허용)
- 규칙 6: 10.10.10.21 → 192.168.1.11, 서비스 25 → Allow (외부 메일서버에서 내부로 SMTP 허용)
- 규칙 7: External → Internal, 서비스 25 → Deny (외부에서 내부로 SMTP 차단)

각 보기 분석:
① 내부와 외부에서 웹 서버(192.168.1.10)에 접근하는 것을 허용한다 — 규칙 1(내부→외부 80 Allow), 규칙 2(Any→192.168.1.10:80 Allow)로 맞는 설명이다.
② 외부의 모든 시스템에서 내부 시스템으로 SMTP 패킷을 보내지 못하도록 설정한다 — 규칙 7(External→Internal:25 Deny)로 맞는 설명이다.
③ 내부 메일 서버(192.168.1.11)에서 외부 메일 서버(10.10.10.21)로 가는 IMAP 접근을 금지한다 — 규칙 5를 보면 192.168.1.11 → 10.10.10.21, 서비스 143(IMAP), Action은 Allow이다. 금지가 아닌 허용이므로 틀린 설명. 정답(적절하지 않은 보기).
④ 내부 메일 서버(192.168.1.11)에서 외부 메일 서버(10.10.10.21)로 가는 SMTP 패킷을 허용한다 — 규칙 3으로 맞는 설명이다.

핵심 정리:
SMTP: 포트 25(발신). POP3: 포트 110(수신). IMAP: 포트 143(수신).
규칙 5는 내부 메일 서버에서 외부로 IMAP(143) Allow → 내부에서 외부 메일 수신 프로토콜 허용.
