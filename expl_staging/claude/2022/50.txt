정답: ④ (4번)

해설:
시퀀스 다이어그램에서 Class1의 m1() 연산에 대한 순환복잡도(CC)와 CBO(Coupling Between Objects)를 계산하는 문항이다.

CC(순환복잡도) 계산:
McCabe의 순환복잡도 = 결정 노드(분기) 수 + 1 (또는 독립 경로 수).
시퀀스 다이어그램의 결합 프레그먼트(combined fragment) 분석:
- loop 프래그먼트: 반복 구조 → 결정 노드 1개 추가
- break 프래그먼트: 조건부 탈출 → 결정 노드 1개 추가
- opt 프래그먼트: 선택 구조 → 결정 노드 1개 추가
CC = 3개의 결정 노드 + 1 = 4

CBO(Coupling Between Objects) 계산:
Class1이 협력하는 다른 클래스의 수 (Class1 자신 제외, Class1을 이용하는 클래스도 제외).
- O21: Class2 인스턴스 → Class2와 결합
- Class3: m3() 호출 → Class3와 결합
- O22: Class2 인스턴스이나 Class1이 직접 결합하지 않음 (Class1이 Class2에만 메시지 전송)
실제로 Class1이 직접 메시지를 보내는 클래스: Class2(m2() 호출), Class3(m4() 호출), O22:Class2(m5() opt)
단, 문제 조건: "CBO를 계산할 때 Class1을 이용하는 다른 클래스는 포함하지 않는다"
Class1이 직접 호출하는 클래스: Class2(O21), Class3, Class2(O22) → Class2와 Class3 = 2개의 고유 클래스
CBO = 2

각 보기 분석:
① CC=3, CBO=3 — CC가 3으로 낮게 계산되었다.
② CC=4, CBO=3 — CBO가 3으로 과다 계산되었다.
③ CC=3, CBO=2 — CC가 3으로 낮게 계산되었다.
④ CC=4, CBO=2 — loop/break/opt 세 결합 프래그먼트로 CC=4, Class2와 Class3 두 클래스와 결합으로 CBO=2. 정답.

핵심 정리:
시퀀스 다이어그램의 CC: loop/alt/opt/break 등 결합 프래그먼트가 분기점이 된다.
CBO: 한 클래스가 직접 결합(사용)하는 고유 클래스 수. 클래스 유형(Class2) 기준으로 계산.
