정답: ① (1번)

해설:
분산 데이터베이스의 투명성(transparency)은 사용자가 분산 환경의 복잡성을 인식하지 않고 단일 시스템처럼 사용할 수 있게 하는 특성이다. 각 투명성 유형의 정확한 정의를 파악해야 한다.

각 보기 분석:
① 이름부여 투명성(naming transparency)이 "분산 DB 설계 방식과 트랜잭션 실행 사이트를 가려주는 것"이라는 설명 — 적절하지 않다. 이름부여 투명성은 데이터 객체가 어느 사이트에 저장되어 있는지와 무관하게 동일한 이름(단일 네임스페이스)으로 접근할 수 있는 것을 의미한다. 트랜잭션 실행 사이트를 가려주는 것은 위치 투명성(location transparency)에 해당한다.
② 분산 투명성(distribution transparency) 또는 네트워크 투명성(network transparency)이 네트워크 세부 사항과 데이터 저장 장소를 가려주는 것 — 분산 투명성의 올바른 설명이다.
③ 위치 투명성(location transparency)이 명령이 데이터 위치와 명령 입력 시스템 위치와 무관한 것 — 위치 투명성의 올바른 설명이다.
④ 중복 투명성(replication transparency)이 데이터 사본 존재를 사용자에게 가려주는 것 — 중복 투명성의 올바른 설명이다.

핵심 정리:
이름부여 투명성: 단일 전역 이름 공간 — 어느 사이트에 있든 동일한 이름으로 접근 가능.
위치 투명성: 데이터가 어느 사이트에 저장되어 있는지 알 필요 없이 접근 가능.
분산 DB 투명성 유형: 이름부여, 위치, 중복, 단편화(fragmentation), 병행, 장애 투명성 등.
