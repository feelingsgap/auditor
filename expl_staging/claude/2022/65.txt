정답: ③ (3번)

해설:
MAC(Mandatory Access Control, 강제 접근 제어)는 데이터 객체와 사용자에게 보안 등급을 부여하고, 등급 간 규칙에 따라 접근을 통제하는 기법이다. Bell-LaPadula 모델이 대표적이며 읽기와 쓰기에 서로 다른 규칙이 적용된다.

각 보기 분석:
① 데이터 객체에 비밀 등급, 사용자에게 허가 등급 부여 — MAC의 기본 개념으로 옳다.
② 보안 등급 TS > S > C > U 순서 — 표준 보안 등급 체계(Top Secret > Secret > Confidential > Unclassified)로 옳다.
③ 갱신(쓰기)을 위해서는 사용자 허가 등급이 객체 비밀 등급보다 같거나 높아야 한다 — 틀렸다. MAC의 *-property(스타 속성)에 따르면 쓰기는 사용자 허가 등급이 객체 비밀 등급과 같거나 낮을 때 허용된다. 높은 등급의 사용자가 낮은 등급 객체에 쓰는 것은 정보 유출 방지를 위해 금지된다.
④ 다단계 보안 기법(multilevel security scheme)이라고도 한다 — MAC의 별칭으로 옳다.

핵심 정리:
MAC 읽기 규칙(Simple Security): 사용자 허가 등급 >= 객체 비밀 등급.
MAC 쓰기 규칙(*-property): 사용자 허가 등급 <= 객체 비밀 등급.
상위 등급 사용자가 하위 객체에 쓰는 것은 정보 하향 유출로 금지된다.
