정답: ② (2번)

해설:
오픈소스 라이선스별 의무 조건을 이해하고 라이선스 침해 여부를 판단하는 문항이다. 핵심은 각 라이선스의 Copyleft 강도와 정적/동적 링크에 따른 의무 범위이다.

각 보기 분석:
가. GPL3.0 라이브러리를 정적 링크 방식으로 연결하여 상업용 소프트웨어 개발·배포하고 소스코드를 공개하지 않음
→ **라이선스 침해.** GPL3.0은 강한 Copyleft로, 정적 링크 시 결합 저작물 전체에 GPL이 적용되어 소스코드 공개 의무 발생. 미공개 = 침해.

나. LGPL3.0 라이브러리를 정적 링크 방식으로 연결하여 상업용 소프트웨어 개발·배포하고 소스코드를 공개하지 않음
→ **라이선스 침해.** LGPL3.0은 동적 링크 시 소스 공개 의무를 면제하지만, **정적 링크 시에는 GPL3.0과 동일하게 소스코드 공개 의무가 발생**한다. LGPL3.0 = GPL3.0 + 추가 허용 사항(섹션 7)이며, 추가 허용 사항은 동적 링크에만 적용된다. 따라서 정적 링크는 GPL3.0 조건 그대로 적용됨.

다. Apache2.0 라이브러리를 정적 링크 방식으로 연결하여 상업용 소프트웨어 개발·배포하고 소스코드를 공개하지 않음
→ **라이선스 침해 아님.** Apache2.0은 Permissive 라이선스로, 소스코드 공개 의무 없이 상업적 이용 가능. 저작권 고지와 라이선스 파일 포함 의무만 있음.

각 보기 정리:
가 → 침해 ✓
나 → 침해 ✓ (LGPL3.0 정적 링크 = Copyleft 의무 적용)
다 → 침해 아님

① 가 — 가만 해당. 나의 LGPL3.0 정적 링크 침해 누락. 오답.
② 가, 나 — 가·나 모두 침해. **정답.**
③ 가, 나, 다 — 다는 침해 아님. 오답.
④ 모두 해당 없음 — 가·나가 침해이므로 오답.

핵심 정리:
- GPL3.0: 정적·동적 링크 모두 강한 Copyleft(소스 공개 의무).
- LGPL3.0: 동적 링크 시 소스 공개 면제, **정적 링크 시 GPL3.0과 동일하게 소스 공개 의무**.
- Apache2.0: Permissive, 소스 공개 의무 없음.

라이선스 강도: GPL > LGPL(정적링크) > LGPL(동적링크) > Apache2.0

[이의제기 경위: 공식 정답이 ①(가)로 발표되었으나, LGPL3.0 정적 링크 시에도 Copyleft 의무(소스 공개)가 적용되므로 나도 침해에 해당한다. 따라서 올바른 정답은 ②(가, 나)이다.]
