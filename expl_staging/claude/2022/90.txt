정답: ③ (3번)

해설:
IPv4 단편화(fragmentation)는 패킷이 MTU(Maximum Transmission Unit)를 초과할 때 여러 개의 작은 단편으로 분할하는 과정이다. 단편화 시 수신 측이 원래 패킷을 재조합하려면 어떤 단편들이 같은 원본 패킷에서 왔는지 식별할 수 있어야 한다.

각 보기 분석:
① 전체 길이 정보 — 단편화 시 변경된다. 원래 패킷의 전체 길이가 아닌 각 단편의 길이로 바뀐다.
② 체크섬(checksum) 정보 — 단편화 시 변경된다. 헤더 내용이 바뀌므로 각 단편마다 체크섬이 재계산된다.
③ 식별자(identification) 정보 — 변경되지 않는다. 모든 단편은 원래 패킷의 식별자를 동일하게 유지한다. 수신 측은 같은 식별자를 가진 단편들을 같은 원본 패킷으로 인식하여 재조합한다. 정답.
④ 단편화 오프셋(fragmentation offset) 정보 — 단편화 시 변경된다. 각 단편이 원래 패킷에서 시작하는 위치(바이트 오프셋)를 나타내며 단편마다 다른 값을 가진다.

핵심 정리:
식별자(Identification): 같은 원본 패킷의 모든 단편이 공유하는 동일한 값. 재조합의 기준.
단편화 시 변경되는 필드: 전체 길이, 체크섬, 단편화 오프셋, MF(More Fragments) 플래그.
단편 재조합: 수신 측에서 식별자 + 오프셋을 이용해 원래 패킷 복원.
