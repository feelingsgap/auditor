정답: ③ (3번)

해설:
데이터베이스 암호화 기술의 방식별 특성에 대한 설명 중 옳지 않은 것을 찾는 문항이다. DB 암호화는 크게 API 방식과 Plug-In 방식으로 구분된다.

각 보기 분석:
① Plug-In 방식은 DB 서버에 암복호화 모듈이 설치되는 방식으로, 구축 시 응용 프로그램의 수정을 최소화할 수 있다.
   맞음. Plug-In 방식은 DB 엔진 또는 DB 서버에 직접 암복호화 모듈을 설치하므로, 애플리케이션 코드 변경 없이 DB 레벨에서 투명하게 암호화가 이루어진다.

② API 방식은 DB 서버에 영향을 주지 않아 DB 서버의 성능 저하는 적은 편이지만, 구축 시 응용 프로그램 전체 또는 일부의 수정이 필요하다.
   맞음. API 방식은 애플리케이션에서 암복호화 API를 직접 호출하므로 DB 서버 부하는 줄지만, 코드 수정이 필수이다.

③ Plug-In 방식은 암복호화 모듈이 라이브러리 형태로 각 애플리케이션 서버에 설치되고, 응용 프로그램에서 해당 암복호화 모듈을 호출하는 방식이다.
   틀림. 이 설명은 API 방식에 대한 설명이다. Plug-In 방식은 DB 서버 측에 모듈이 설치되는 방식이며, 각 애플리케이션 서버에 라이브러리를 배포하는 것은 API 방식의 특징이다. 정답.

④ API 방식과 Plug-In 방식의 장점을 혼용한 하이브리드 방식에서는 대용량 트랜잭션 처리에 Plug-In 방식을 사용하고, 성능 저하가 발생할 수 있는 부분은 API 방식을 사용한다.
   맞음. 하이브리드 방식은 상황에 따라 두 방식을 혼합하여 성능과 편의성을 절충한다.

핵심 정리:
API 방식: 애플리케이션 레이어에서 암복호화 호출 → 앱 수정 필요, DB 서버 부하 최소.
Plug-In 방식: DB 서버/엔진 레이어에서 투명 암호화 → 앱 수정 최소, DB 성능에 영향.
TDE(Transparent Data Encryption): DB 파일 자체를 암호화하는 스토리지 레벨 방식 (SQL Server, Oracle 지원).
