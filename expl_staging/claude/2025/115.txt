정답: ③ (3번)

해설:
웹 애플리케이션 개발 시 입력 데이터 검증 소홀로 발생하는 보안 취약점과 가장 거리가 먼 것을 찾는 문항이다.

각 보기 분석:
① 포맷 스트링 삽입(Format String Injection)
   관련 있음. printf() 등의 포맷 스트링 함수에 사용자 입력이 직접 전달될 때 발생한다. 입력값에 대한 검증 부재로 발생하는 취약점이다.

② 신뢰되지 않는 URL 주소로 자동 접속 연결(Unvalidated Redirects and Forwards)
   관련 있음. 입력된 URL을 검증 없이 리다이렉트에 사용할 때 발생한다. OWASP Top 10에 포함된 입력 검증 관련 취약점이다.

③ 경쟁 조건(Race Condition)
   거리가 멈. 경쟁 조건은 두 개 이상의 프로세스/스레드가 공유 자원에 동시에 접근하여 발생하는 타이밍 문제이다. 입력 데이터 검증과 무관하며, 동시성 제어(락, 뮤텍스) 문제에 해당한다. 정답.

④ 메모리 버퍼 오버플로우(Buffer Overflow)
   관련 있음. 입력 데이터의 크기를 검증하지 않아 버퍼 경계를 초과하는 데이터가 인접 메모리를 덮어쓰는 취약점이다.

핵심 정리:
입력 데이터 검증 소홀 관련 취약점: SQL Injection, XSS, Buffer Overflow, Format String, Path Traversal, Command Injection 등.
경쟁 조건(Race Condition): TOCTOU(Time-of-Check to Time-of-Use) 취약점이 대표적. 입력 검증보다는 동시성 제어 문제.
입력 검증 원칙: 화이트리스트 방식, 길이/형식/범위 검증, 서버 측 검증(클라이언트 측만으로 불충분).
