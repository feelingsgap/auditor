정답: ④ (4번)

해설:
Java 코드를 UML 클래스 다이어그램으로 올바르게 표현하는 문항이다.

Java 코드 분석:
- interface Service { public void doWork(); }
- class Person { ... }
- class Task { ... }
- class Staff extends Person implements Service {
    public void doWork() { Task t = new Task(); ... }
  }

코드에서 도출되는 세 가지 관계:
1. Staff extends Person → 일반화(Generalization): 실선 + 빈 삼각형 화살표, Staff에서 Person으로
2. Staff implements Service → 실현(Realization): 점선 + 빈 삼각형 화살표, Staff에서 Service로
3. Staff.doWork()에서 Task t = new Task() → 의존(Dependency): 점선 + 열린 화살표, Staff에서 Task로

각 보기 분석:
① Staff → Service 연결선이 실선으로 표현됨 — implements(구현)는 점선이어야 하므로 오답.
② Staff → Person 연결선이 점선으로 표현됨 — extends(상속)는 실선이어야 하므로 오답.
③ Staff → Service 화살표 방향이 반대(Service에서 Staff로)로 표현됨 — 실현 관계의 화살표는 Staff에서 Service 방향이어야 하므로 오답.
④ Service ◁····Staff(점선+빈 삼각형), Person ◁——Staff(실선+빈 삼각형), Task←····Staff(점선+열린 화살표) — 세 관계가 모두 올바르게 표현됨. 정답.

핵심 정리:
UML 관계 선 종류:
일반화(extends): 실선 + 빈 삼각형(▷) → 부모 클래스 방향
실현(implements): 점선 + 빈 삼각형(▷) → 인터페이스 방향
의존(dependency): 점선 + 열린 화살표(→) → 사용하는 클래스 방향
연관(association): 실선 + 열린 화살표(→)
