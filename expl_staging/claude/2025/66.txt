정답: ④ (4번)

해설:
다중 단위 로킹(multiple granularity locking)에서 IX와 SIX 로크의 양립성(compatibility)을 묻는 문항이다.

로크 의미:
IX (Intention eXclusive): 하위 노드에 X 로크를 걸 의도. 하위 노드를 독점적으로 수정할 계획.
SIX (Shared + Intention eXclusive): 현재 노드에 S 로크 + 하위 노드에 X 로크 의도. 현재 노드를 공유 읽기하면서 일부 하위 노드를 수정.

양립성 분석:
(가) 현재 IX 보유 - 요청 IX:
IX vs IX: 둘 다 하위 노드를 수정할 의도이나, 현재 노드 자체에는 배타적 잠금을 걸지 않음.
서로 다른 하위 노드를 수정할 수 있으므로 허용(호환). (가) = 허용

(나) 현재 IX 보유 - 요청 SIX:
SIX는 현재 노드에 S 로크를 포함. 그러나 IX를 보유한 트랜잭션은 하위 노드에 X 로크를 걸 예정이며, 이는 S 로크와 충돌(X와 S는 비양립).
대기(불허). (나) = 대기

(다) 현재 SIX 보유 - 요청 IX:
SIX는 현재 노드에 S를 포함. 요청 IX는 하위에 X를 걸겠다는 의도로 현재 노드의 S와 하위 X가 충돌.
대기(불허). (다) = 대기

(라) 현재 SIX 보유 - 요청 SIX:
둘 다 현재 노드에 S + 하위 X 의도. S는 공유 가능하나 각자의 하위 X 의도가 충돌.
대기(불허). (라) = 대기

각 보기 분석:
① 가:허용, 나:허용, 다:허용, 라:허용 — 나, 다, 라가 허용일 수 없음. 오답.
② 가:허용, 나:허용, 다:대기, 라:대기 — 나(IX+SIX)가 허용일 수 없음. 오답.
③ 가:허용, 나:대기, 다:허용, 라:대기 — 다(SIX+IX)가 허용일 수 없음. 오답.
④ 가:허용, 나:대기, 다:대기, 라:대기 — 분석과 일치. 정답.

핵심 정리:
IX+IX = 허용 (하위 노드 분리 가능).
IX+SIX, SIX+IX, SIX+SIX = 대기 (SIX의 S 성분이 하위 X 의도와 충돌).
다중 단위 로킹 계층: DB > 테이블 > 페이지 > 행. 상위 노드에 의도 로크를 걸어 하위 로크 충돌을 사전 감지.
