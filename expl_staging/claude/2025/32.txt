정답: ① (1번)

해설:
SysML(System Modeling Language)의 Requirement Diagram이 표현하는 정보를 묻는 문항이다.

SysML Requirement Diagram의 역할:
SysML Requirement Diagram은 요구사항을 표현하고, 요구사항 간의 관계 및 다른 모델 요소와의 관계를 시각적으로 나타내는 다이어그램이다.

각 항목 분류:
가. 요구사항의 계층구조를 가시적으로 표현한다. ✓ — 포함(ContainReqt) 관계로 요구사항 간 계층 구조를 표현할 수 있다.
나. 다른 모델 요소들과의 관계를 표현한다. ✓ — Satisfy, Refine, Verify, DeriveReqt 등의 관계로 블록·유스케이스 등 다른 모델 요소와의 관계를 표현한다.
다. 개별 요구사항에 대하여 표현한다. ✓ — 요구사항을 요소(requirement element)로 모델링하여 개별 요구사항 정보(ID, 설명 등)를 표현한다.
라. 입력, 출력, 처리의 흐름 관점에서 표현한다. ✗ — 이는 Activity Diagram 또는 Block Definition Diagram의 역할이다.
마. 시스템 특성의 제약사항에 대한 수식을 표현한다. ✗ — 이는 Parametric Diagram(파라메트릭 다이어그램)의 역할이다.

가, 나, 다 → 정답 ①

핵심 정리:
SysML Requirement Diagram = 요구사항 계층(가) + 모델 요소 관계(나) + 개별 요구사항(다).
흐름 표현 = Activity Diagram / Parametric Constraint = Parametric Diagram.
SysML 9개 다이어그램: Req Diagram, Use Case, Activity, Sequence, State Machine, Block Definition, IBD, Parametric.
