정답: ② (2번)

해설:
XML DTD를 분석하여 파싱 오류가 발생하는 XML 문장을 찾는 문항이다.

DTD 분석:
<!ELEMENT x (a, (b | c), d)*>  → x는 (a, (b|c), d) 그룹의 0회 이상 반복
<!ELEMENT a (#PCDATA)>
<!ELEMENT b (#PCDATA)>
<!ELEMENT c (#PCDATA)>
<!ELEMENT d (#PCDATA)>

각 그룹의 구조: a → (b 또는 c) → d 순서로 반드시 세 자식 요소 포함.
*는 0회 이상 반복이므로 그룹이 없는 빈 x는 유효.

각 보기 분석:
① <x></x>
   그룹이 0회 반복. * 허용. 파싱 오류 없음.
② <x><a /><d /></x>
   a, d만 있고 (b|c)가 없음. DTD는 a 다음에 반드시 b 또는 c가 와야 함. (b|c) 누락 → 파싱 오류 발생. 정답.
③ <x><a /><b /><d /></x>
   a → b(b|c 중 b 선택) → d 순서. 한 그룹 완성. 유효.
④ <x><a /><b /><d /><a /><c /><d /></x>
   그룹1: a, b, d + 그룹2: a, c, d. 두 그룹 반복. 유효.

핵심 정리:
DTD 순서 모델: (a, b, c)는 a, b, c를 순서대로 모두 포함해야 함.
DTD 선택 모델: (b | c)는 b 또는 c 중 하나 선택.
* 수량자: 0회 이상. ? 0또는1회, + 1회 이상.
② 는 (b|c) 선택 요소가 아예 없어 구조적으로 유효하지 않음.
