정답: ④ (4번)

해설:
"웹 애플리케이션을 사용하는 다른 최종 사용자의 클라이언트에서 임의의 스크립트가 실행되는 취약점"에 해당하는 공격 유형을 찾는 문항이다.

각 보기 분석:
① Directory Indexing
   틀림. Directory Indexing은 웹 서버가 특정 디렉터리의 파일 목록을 노출하여 민감한 파일 경로가 공개되는 취약점이다. 스크립트 실행과 무관하다.

② XPath Injection
   틀림. XPath Injection은 XML 기반 데이터를 쿼리하는 XPath 구문에 악의적인 코드를 삽입하여 데이터를 탈취하는 공격이다. 클라이언트 측 스크립트 실행이 아닌 서버 측 쿼리 조작이다.

③ Cross-Site Request Forgery(CSRF)
   틀림. CSRF는 인증된 사용자의 세션을 악용하여 의도하지 않은 요청(계좌이체, 정보 변경 등)을 피해자 대신 서버에 전송하는 공격이다. 스크립트 실행이 아닌 위조 요청 전송이다.

④ Cross Site Scripting(XSS)
   맞음. XSS는 공격자가 웹 애플리케이션에 악성 스크립트(주로 JavaScript)를 삽입하고, 이것이 다른 사용자의 브라우저에서 실행되도록 하는 공격이다. 피해자의 세션 쿠키 탈취, 페이지 변조, 악성 사이트 리다이렉션 등이 가능하다. 정답.

핵심 정리:
XSS 유형:
  반사형(Reflected): URL 파라미터의 스크립트가 응답에 포함되어 즉시 실행.
  저장형(Stored/Persistent): DB에 저장된 스크립트가 페이지 로드 시 반복 실행.
  DOM 기반: 클라이언트 스크립트가 DOM을 조작하여 실행.
대응: 출력 인코딩(HTML Entity Encoding), CSP(Content Security Policy), HttpOnly 쿠키.
