정답: ① (1번)

해설:
HDFS(Hadoop Distributed File System)에서 데이터 읽기와 쓰기의 성능 차이를 설명하는 문항이다.

HDFS 아키텍처:
- NameNode: 메타데이터(파일 경로, 블록 위치) 관리
- DataNode: 실제 데이터 블록 저장
- 기본 복제 인수: 3 (블록을 3개의 DataNode에 분산 저장)
- Rack Awareness: 클라이언트와 가까운 DataNode 우선 접근

읽기(Read) 동작:
NameNode에서 블록 위치 정보를 받아 클라이언트에서 가장 가까운 DataNode에서 직접 블록을 읽음.
→ 단일 DataNode에서 순차적으로 빠르게 읽기 가능.

쓰기(Write) 동작:
1. 클라이언트 → DataNode1(Pipeline 시작)
2. DataNode1 → DataNode2 → DataNode3 순차 복제
3. 모든 복제본 저장 확인(ACK) 후 쓰기 완료
→ 복제본 파이프라인 완료를 기다려야 하므로 읽기보다 느림.

각 보기 분석:
① 읽기는 가까운 DataNode에서 데이터를 가져오기 때문에 빠르고, 쓰기는 모든 복제본 저장을 기다리기 때문에 느리다. — HDFS 실제 동작과 일치. 정답.
② 읽기는 모든 DataNode에서 수집해야 하므로 느리고, 쓰기는 특정 DataNode에만 기록하면 되므로 빠르다. — 읽기는 단일 DataNode에서 수행하고, 쓰기는 복제 파이프라인이 필요. 완전히 반대 설명. 오답.
③ HDFS는 데이터를 항상 NameNode에서 읽고 쓰므로 읽기와 쓰기에서의 성능 차이가 없다. — NameNode는 메타데이터만 관리하며 데이터 전송에는 관여하지 않음. 오답.
④ HDFS는 쓰기가 완료된 후에만 읽기가 가능하기 때문에 쓰기에 비해 읽기가 느리다. — 이미 쓰인 블록은 바로 읽을 수 있으며 읽기가 쓰기보다 빠름. 오답.

핵심 정리:
HDFS 설계 철학: 대용량 순차 읽기에 최적화.
쓰기 병목: 복제 파이프라인(3-way replication ACK).
읽기 최적화: Rack-aware DataNode 선택으로 네트워크 홉 최소화.
