정답: ④ (4번)

해설:
대표적인 데이터 포맷에 대한 설명 중 옳지 않은 것을 찾는 문항이다.

각 보기 분석:
① Key-Value Store: 키(Key)에 값(Value) 형태로 데이터를 저장하며, 고속 검색이 가능한 구조이다. 일반적으로 NoSQL 데이터베이스에서 자주 사용된다.
   → Key-Value Store에 대한 올바른 설명. 적절.

② JSON(JavaScript Object Notation): 속성-값 쌍으로 데이터를 저장하며, 트리 구조를 가진다. 문자열 기반 데이터 표현 방식이며, 네트워크 통신에서 자주 사용된다.
   → REST API 등 네트워크 통신에서 주로 사용. 계층적 구조(중첩 가능) 지원. 적절.

③ XML(eXtensible Markup Language): 태그를 사용하여 계층적 구조의 데이터를 표현하며, 문서 형식의 데이터 저장과 교환에 주로 사용된다.
   → XML에 대한 올바른 설명. 적절.

④ CSV(Comma Separated Values): 쉼표(,)로 데이터를 구분하며, 행(row)과 열(column) 구조를 갖는다. 정형 데이터 저장에 적합하며, 중첩 구조를 표현할 수 있다.
   → CSV는 단순 행·열 구조의 평면 데이터 형식으로, 중첩 구조(계층적 관계)를 표현하는 것이 불가능하다. "중첩 구조를 표현할 수 있다"는 틀린 설명. 정답.

핵심 정리:
CSV: 단순 표(flat table) 형식. 중첩/계층 구조 표현 불가. 엑셀·스프레드시트 호환성 우수.
JSON/XML: 중첩 구조 지원 — 복잡한 계층적 데이터 표현 가능.
데이터 포맷 선택 기준: 단순 정형 데이터 → CSV; 구조화된 교환 → JSON/XML; 고속 조회 → Key-Value.
