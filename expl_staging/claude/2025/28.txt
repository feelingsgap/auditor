정답: ③ (3번)

해설:
주어진 두 가지 설명에 해당하는 테스트 설계 기법을 묻는 문항이다.

설명 분석:
- "테스트 대상과 사용자 사이에 발생할 수 있는 입/출력의 시퀀스에 대한 모델을 바탕으로 테스트 케이스를 설계한다." → 사용자의 실제 사용 흐름(시나리오)을 모델화
- "시스템 테스트, 인수 테스트를 수행할 때 End-to-End 기능 테스트를 위하여 주로 활용된다." → 전체 흐름을 처음부터 끝까지 검증

각 보기 분석:
① State transition testing (상태 전이 테스트) — 시스템의 상태 변화를 테스트. 특정 이벤트에 의해 상태가 전이되는 경우를 테스트하며, 입/출력 시퀀스의 End-to-End 테스트보다는 상태 머신 검증에 특화된다.
② Decision table testing (의사결정 테이블 테스트) — 여러 입력 조건의 조합과 그에 따른 결과를 표로 정리하여 테스트. End-to-End 흐름보다는 논리적 조건 검증에 사용된다.
③ Scenario testing (시나리오 테스트) — 사용자가 시스템을 사용하는 현실적인 사용 흐름(시나리오)을 모델화하고 그 시퀀스를 테스트한다. 시스템 테스트 및 인수 테스트의 End-to-End 기능 검증에 적합하다. 정답.
④ Requirement-based testing (요구사항 기반 테스트) — 요구사항 명세서를 기반으로 테스트 케이스를 도출하는 상위 개념. 특정 기법이라기보다는 테스트 접근 방식이다.

핵심 정리:
시나리오 테스트 = 사용자 관점의 실제 사용 흐름 + End-to-End 기능 검증.
시스템 테스트·인수 테스트(UAT)에서 주로 사용.
상태 전이 테스트 = 상태 머신 검증 / 의사결정 테이블 = 조건 조합 검증과 구분할 것.
