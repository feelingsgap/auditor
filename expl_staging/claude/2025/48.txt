정답: ① (1번)

해설:
금고경비시스템 의사코드 (가)를 (나)로 변경하는 데 적용된 설계 패턴을 묻는 문항이다.

코드 구조 분석:
(가): 단일 클래스에 if/else if 조건문으로 경비 상태(주간, 야간, 알람 등)에 따라 다른 동작을 처리하는 구조.
  - 금고 사용 시 호출되는 메서드: if (주간) → 경비 알림 사용기록; else if (야간) → 경비 알림 이상 발생
  - 이상 발생 시, 알람 동작 시도 마찬가지로 if/else if 분기 처리.

(나): 각 경비 상태를 별도 클래스로 분리한 구조.
  - 주간을 표현한 클래스 / 야간을 표현한 클래스가 각각 "금고 사용 시 호출되는 메서드", "이상 발생 시 메서드", "알람 동작 시 메서드"를 독립적으로 구현.

패턴 식별:
객체의 내부 상태(주간/야간)에 따라 동작이 달라지고, 각 상태를 별도 클래스로 캡슐화 → State Pattern.

각 보기 분석:
① State Pattern — 내부 상태를 별도 클래스로 분리해 상태 전환 시 객체의 행동이 바뀌는 패턴. (가)→(나) 변환과 정확히 일치. 정답.
② Adapter Pattern — 인터페이스가 호환되지 않는 클래스들을 함께 사용하기 위한 패턴. 해당 없음.
③ Builder Pattern — 복잡한 객체의 생성 과정을 단계적으로 분리하는 생성 패턴. 해당 없음.
④ Template Method Pattern — 알고리즘 골격을 상위 클래스에 정의하고 세부 단계를 서브클래스에서 구현. (나)에서 공통 메서드를 인터페이스/추상 클래스로 정의하는 측면이 있지만, 핵심은 상태 분리이므로 State Pattern이 더 정확.

핵심 정리:
State Pattern: if/else if 다중 조건 분기 → 각 상태 클래스로 분리.
상태가 바뀌면 행동이 바뀌는 구조(주간 상태 객체 ↔ 야간 상태 객체 교체).
Strategy Pattern과 유사하나 State는 상태 간 전환이 핵심, Strategy는 알고리즘 교체가 핵심.
