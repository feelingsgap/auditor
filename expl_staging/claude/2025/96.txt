정답: ① (1번)

해설:
트로이목마·백도어가 설치되어 정보를 빼내는지 확인할 때 사용하는 네트워크 관리 명령어를 찾는 문항이다.

백도어 탐지 방법:
백도어는 특정 포트를 열고 외부 공격자와 통신한다. 의심스러운 열린 포트나 비정상적인 연결을 확인하는 것이 핵심.

각 보기 분석:
① netstat -an
   -a: 모든 연결 및 수신 대기(LISTEN) 포트 표시
   -n: 주소·포트를 숫자로 표시(DNS 해석 없이 빠르게 확인)
   모든 TCP/UDP 포트 상태(ESTABLISHED, LISTEN, TIME_WAIT 등)를 확인 가능.
   백도어의 비정상 LISTEN 포트나 ESTABLISHED 연결을 탐지하는 데 가장 적합. 정답.

② netstat -p
   연결에 연관된 프로세스 이름/PID 표시 (Linux에서 유용).
   포트-프로세스 연계 확인 가능하지만, -an이 더 포괄적.

③ netstat -r
   라우팅 테이블 표시. 포트 상태나 연결 목록 확인 불가. 백도어 탐지와 무관.

④ netstat -s
   프로토콜별 통계 정보 표시. 포트/연결 상태 직접 확인 불가.

핵심 정리:
netstat -an: 이상한 포트(예: 4444, 1337 등 악성코드 자주 사용 포트)의 LISTEN 상태 확인.
백도어 탐지 절차: netstat -an 실행 → 수상한 LISTEN/ESTABLISHED 포트 식별 → 해당 프로세스 확인(netstat -p 또는 lsof -i).
현대 환경에서는 ss(socket statistics) 명령어가 netstat 대체로 사용되기도 함.
