정답: ② (2번)

해설:
바차트(Gantt Chart)에서 임계경로(Critical Path)상의 작업을 식별하는 문항이다.

임계경로(Critical Path) 개념:
프로젝트 완료까지의 가장 긴 작업 경로로, 여유 시간(Slack/Float)이 0인 작업들의 연속.
임계경로 상의 작업이 지연되면 프로젝트 전체 일정이 지연된다.

바차트 분석:
바차트에서 각 선택지의 작업 경로를 확인한다.
- A(T4, T8, T11, T12): 특정 작업 그룹
- B(T1, T3, T9): 특정 작업 그룹
- C(T2, T6, T10): 특정 작업 그룹
- D(T7): 단일 작업
- E(T5): 단일 작업

이미지의 바차트에서 M1~M8 마일스톤을 기준으로 임계경로를 추적하면, T1→T3→T9 경로가 여유 없이 프로젝트 종료까지 연결되는 임계경로이다.

각 보기 분석:
① A(T4, T8, T11, T12) — 바차트상 해당 작업들 사이에 여유 시간이 존재. 임계경로 아님.
② B(T1, T3, T9) — 여유 없이 종료 마일스톤까지 연결되는 임계경로 작업. 정답.
③ C(T2, T6, T10) — 해당 작업들 사이에 여유 시간 존재. 임계경로 아님.
④ D, E(T7, T5) — 단일 작업들로 전체 경로 연결이 부족. 임계경로 아님.

핵심 정리:
임계경로: 시작 → 종료까지 가장 긴 경로(Float = 0인 작업들).
바차트에서 임계경로 식별: 여유 없이 연속 배치된 작업 체인을 찾는다.
CPM(Critical Path Method): 임계경로 계산 기법. ES, EF, LS, LF로 Float = LS - ES = LF - EF 계산.
