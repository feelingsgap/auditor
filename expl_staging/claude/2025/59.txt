정답: ③ (3번)

해설:
SQL SELECT 문의 카디널리티(결과 투플 수)를 계산하는 문항이다.

데이터 분포:
선수 릴레이션 (일련번호, 이름, 소속팀, 연봉): 총 100명
- L 소속팀: 10명
- H 소속팀: 40명
- K 소속팀: 30명
- 소속팀 미정(NULL): 20명

가. SELECT 소속팀 FROM 선수;
모든 투플의 소속팀 속성값 반환. 중복 제거 없음.
100개 행 → 카디널리티 = 100

나. SELECT DISTINCT 소속팀 FROM 선수;
소속팀 중복 제거.
고유 값: L, H, K, NULL — SQL에서 DISTINCT는 NULL도 하나의 고유 값으로 취급.
카디널리티 = 4

다. SELECT 소속팀, COUNT(*) FROM 선수 GROUP BY 소속팀 HAVING COUNT(*) >= 20;
소속팀별 그룹:
- L: 10명 → HAVING 불충족, 제외
- H: 40명 → HAVING 충족, 포함
- K: 30명 → HAVING 충족, 포함
- NULL: 20명 → NULL 그룹도 COUNT(*)=20이므로 HAVING 충족, 포함
결과 그룹 수 = 3 (H, K, NULL) → 카디널리티 = 3

각 보기 분석:
① 가:80, 나:3, 다:2 — 가와 다 모두 오산. NULL 20명 미반영.
② 가:80, 나:3, 다:3 — 가 오산. NULL 20명을 미계산.
③ 가:100, 나:4, 다:3 — 모두 올바른 계산. 정답.
④ 가:100, 나:4, 다:4 — 다에서 L(10명)도 포함하는 오산.

핵심 정리:
카디널리티: GROUP BY나 DISTINCT 없으면 전체 행 수.
DISTINCT: NULL도 고유한 하나의 값으로 처리.
GROUP BY + HAVING: NULL도 별도 그룹으로 형성, HAVING의 COUNT(*)는 NULL인 행도 카운트.
