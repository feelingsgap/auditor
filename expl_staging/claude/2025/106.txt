정답: ③ (3번)

해설:
DNS 스푸핑(DNS Spoofing) 공격의 동작 원리와 특성에 대한 설명 중 옳지 않은 것을 찾는 문항이다.

각 보기 분석:
① DNS 질의/응답에 사용하는 UDP 프로토콜의 비연결 취약성을 이용한 공격이다.
   맞음. DNS는 기본적으로 UDP 53번 포트를 사용한다. 연결 설정 없이 패킷을 주고받는 UDP의 비연결형 특성 때문에 출발지 위조가 쉬우며, 이를 DNS 스푸핑에 악용한다.

② 공격자는 DNS 스푸핑을 수행하기 위해 공격대상이 보내는 메시지를 스니핑해야 한다.
   맞음. 공격자는 피해자가 DNS 질의를 보낼 때 이를 스니핑하여 트랜잭션 ID 등을 확인하고, DNS 서버보다 먼저 위조된 응답을 피해자에게 보낸다.

③ 공격대상에게 전달되는 DNS 질의 메시지를 조작하거나 DNS 서버의 캐시 정보를 삭제하는 방법을 이용한다.
   틀림. DNS 스푸핑은 DNS 서버의 캐시 정보를 '삭제'하는 것이 아니라 '오염(Poisoning)/변조'하는 방식을 사용한다. 잘못된 IP 정보를 DNS 캐시에 주입하는 것을 캐시 포이즈닝(Cache Poisoning)이라고 한다. 정답.

④ 공격이 성공한 경우, 공격대상은 정상적인 URL로 접속하지만 실제로는 공격자가 만든 가짜 사이트로 접속하게 된다.
   맞음. DNS 스푸핑의 최종 목적이 이것이다. 피해자는 올바른 도메인을 입력했지만 공격자의 서버(파밍 사이트)로 유도된다.

핵심 정리:
DNS 스푸핑 흐름: 피해자 DNS 질의 스니핑 → 위조 응답 먼저 전송(트랜잭션 ID 위조) → 피해자 캐시에 가짜 IP 저장.
DNS 캐시 포이즈닝: DNS 서버 캐시에 위조된 레코드 주입 → 해당 DNS를 사용하는 모든 사용자 영향.
대응: DNSSEC(DNS Security Extensions), 응답 검증, TLS 사용(HTTPS 강제).
