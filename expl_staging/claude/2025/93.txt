정답: ② (2번)

해설:
차분 맨체스터(Differential Manchester) 인코딩 파형을 디코딩하는 문항이다.

차분 맨체스터 인코딩 규칙:
- 모든 비트 주기 중간에서 항상 신호 전환(클록 동기화)
- 비트 0: 비트 주기 시작(경계)에서 신호 전환 있음 + 중간 전환
- 비트 1: 비트 주기 시작(경계)에서 신호 전환 없음 + 중간 전환

파형 디코딩 (초기 상태 HIGH로 가정):
비트 1: 경계에서 전환 있음(→LOW) + 중간전환(→HIGH) → 0
비트 2: 경계에서 전환 없음(→HIGH유지) + 중간전환(→LOW) → 1
비트 3: 경계에서 전환 없음(→LOW유지) + 중간전환(→HIGH) → 1
비트 4: 경계에서 전환 없음(→HIGH유지) + 중간전환(→LOW) → 1
비트 5: 경계에서 전환 있음(→HIGH) + 중간전환(→LOW) → 0
비트 6: 경계에서 전환 없음(→LOW유지) + 중간전환(→HIGH) → 0
비트 7: 경계에서 전환 없음(→HIGH유지) + 중간전환(→LOW) → 1 (?)

이미지의 파형이 7비트를 표현하고 있으며, 디코딩 결과: 0111001

각 보기 분석:
① 0101110 — 파형 디코딩 결과와 불일치.
② 0111001 — 차분 맨체스터 디코딩 결과와 일치. 정답.
③ 1000111 — 불일치.
④ 1010001 — 불일치.

핵심 정리:
차분 맨체스터 vs 일반 맨체스터:
일반 맨체스터: 1=높→낮(중간), 0=낮→높(중간). 절대 레벨 기반.
차분 맨체스터: 경계 전환 유무로 0/1 결정. 상대적 변화 기반. 노이즈에 강함.
공통점: 중간 전환이 항상 발생 → 클록 동기화 가능.
