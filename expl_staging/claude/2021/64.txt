정답: ① (1번)

해설:
동시성 제어 없이 두 트랜잭션이 동시에 수행될 때 발생하는 문제를 파악하는 문항이다.
T1은 홍길동의 잔액을 100,000원 증가시킨 후 ROLLBACK하고, T2는 T1의 UPDATE 직후(ROLLBACK 전) 전체 잔액의 평균을 조회한다.

시나리오 분석:
T1: UPDATE(홍길동 잔액 +100,000) → (T2가 조회) → ROLLBACK
T2: SELECT AVG(balance) → COMMIT
T2는 T1이 아직 커밋하지 않은, 즉 나중에 롤백될 변경 데이터를 읽어 AVG를 계산한다.

각 보기 분석:
① 오손 데이터 읽기(dirty read) — 커밋되지 않은 트랜잭션의 변경 데이터를 다른 트랜잭션이 읽는 현상. T1이 ROLLBACK되면 T2가 읽은 값은 존재하지 않는 데이터가 된다. 정답.
② 갱신 손실(lost update) — 두 트랜잭션이 같은 데이터를 갱신할 때 하나의 갱신이 덮어써지는 현상. 이 시나리오와 다르다.
③ 반복할 수 없는 읽기(unrepeatable read) — 같은 트랜잭션 내에서 동일 데이터를 두 번 읽을 때 결과가 다른 현상. 이 시나리오와 다르다.
④ 모순성(inconsistency) — 여러 데이터 항목을 읽는 도중 일부는 갱신 전, 일부는 갱신 후 값을 읽어 불일치가 발생하는 현상. 이 시나리오와는 다르다.

핵심 정리:
오손 읽기(dirty read): 아직 커밋되지 않은 데이터를 읽는 것 → 롤백 시 읽은 데이터가 무효화됨.
격리 수준 READ COMMITTED 이상에서 방지 가능.
가장 낮은 격리 수준인 READ UNCOMMITTED에서 발생한다.
