정답: ④ (4번)

해설:
학생(학번, 이름, 학과) 릴레이션에 수학과 60명, 화학과 40명, 물리과 30명, 학과 NULL 10명이 있을 때 각 SQL 쿼리의 결과 투플 수를 구하는 문항이다.

데이터 구성:
전체 학생: 60 + 40 + 30 + 10 = **140명**
학과 값 종류: 수학과, 화학과, 물리과, NULL — 4가지

쿼리별 분석:
가. `SELECT 학과 FROM 학생`
→ WHERE 절 없이 모든 행의 학과 컬럼을 반환. NULL 학과를 가진 학생도 포함. **결과 = 140 투플**

나. `SELECT DISTINCT 학과 FROM 학생`
→ 중복 제거. SQL 표준에서 NULL도 하나의 고유 값으로 처리됨. 수학과·화학과·물리과·NULL = **결과 = 4 투플**

다. `SELECT COUNT(*) FROM 학생 GROUP BY 학과`
→ 학과별 그룹으로 집계. SQL 표준에서 NULL도 별도 그룹으로 처리됨.
  - 수학과 그룹: COUNT(*) = 60
  - 화학과 그룹: COUNT(*) = 40
  - 물리과 그룹: COUNT(*) = 30
  - NULL 그룹: COUNT(*) = 10
→ **결과 = 4 행(투플)**

각 보기 분석:
① 가 130, 나 3, 다 3 — 가는 전체 140행이어야 하고, 나·다는 NULL 포함 시 4여야 한다. 오답.
② 가 130, 나 4, 다 4 — 가가 130으로 잘못됨. 오답.
③ 가 140, 나 3, 다 3 — 가는 맞으나, SQL 표준에서 DISTINCT·GROUP BY는 NULL을 포함한다. 오답.
④ 가 140, 나 4, 다 4 — SQL 표준에 따른 올바른 계산. **정답.**

핵심 정리:
- `SELECT 컬럼 FROM 테이블`: WHERE 없이 모든 행 반환 (NULL 포함). 결과 = 전체 행 수.
- `SELECT DISTINCT 컬럼`: SQL 표준상 NULL도 하나의 고유 값으로 포함.
- `GROUP BY 컬럼`: SQL 표준상 NULL도 별도 그룹으로 처리됨.

[이의제기 경위: 공식 정답이 ①(가 130, 나 3, 다 3)으로 발표되었으나, SQL 표준에 따르면 ④(가 140, 나 4, 다 4)가 정확하다. '가=130'은 SELECT가 NULL을 제외한다는 잘못된 전제에서 출발하며, 이는 SQL 표준과 맞지 않는다. 이 문항은 NULL 처리 방식에 대한 시험 출제 오류로 이의제기 수용이 타당하다.]
