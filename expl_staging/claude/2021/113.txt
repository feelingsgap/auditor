정답: ② (2번)

해설:
제시된 설명은 "악성 스크립트가 포함된 URL을 공격자가 클라이언트에게 노출시켜 클릭을 유도하고, 쿠키 정보를 탈취하거나 피싱 사이트, 불법 광고 사이트로 이동하게 만든다"이다. 이는 Reflected XSS(반사형 크로스 사이트 스크립팅)의 특성과 일치한다.

각 보기 분석:
① Stored XSS — 악성 스크립트를 서버(DB 등)에 저장해 두고, 해당 페이지를 조회하는 모든 사용자에게 스크립트가 실행되는 방식. URL을 통해 직접 전달하는 방식이 아니다.
② Reflective XSS — 공격자가 악성 스크립트가 포함된 URL을 피해자에게 전달하면, 서버가 해당 입력을 그대로 반사(reflect)하여 응답함으로써 클라이언트 측에서 스크립트가 실행된다. 쿠키 탈취, 피싱 유도 등에 사용된다. 설명과 정확히 일치. 정답.
③ SQL Injection — DB 쿼리에 악의적인 SQL 문을 삽입하여 DB를 비인가 접근하는 공격. 스크립트 실행과 다르다.
④ Command Injection — 입력값에 시스템 명령어를 삽입하여 서버에서 실행시키는 공격. 스크립트 실행과 다르다.

핵심 정리:
Stored XSS: 스크립트를 서버에 저장, 모든 방문자에게 실행.
Reflected XSS: 악성 URL → 서버 반사 → 클라이언트 실행, 피싱/쿠키 탈취에 활용.
DOM-based XSS: 서버 응답 없이 클라이언트 측 DOM 조작으로 실행.
