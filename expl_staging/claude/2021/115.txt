정답: ② (2번)

해설:
HTTP 프록시 서버를 이용하는 브라우저가 HTTPS를 적용하는 상황에서의 동작을 묻는 문항이다. HTTPS over HTTP Proxy에서 CONNECT 메서드와 SSL/TLS 핸드셰이크 위치를 정확히 이해해야 한다.

각 설명 분석:
가. 브라우저는 CONNECT 메서드를 사용하여 HTTP 프록시 서버에게 요청을 전달한다 — HTTPS 사용 시 브라우저는 CONNECT 메서드로 프록시에게 터널 설정을 요청한다. 올바른 설명이다.
나. 브라우저는 GET/PUT/POST 등의 메서드를 사용하여 HTTP 프록시 서버에게 요청을 전달한다 — HTTPS 환경에서는 CONNECT 메서드를 사용하며, GET/POST 등은 일반 HTTP 요청 시 사용된다. 잘못된 설명이다.
다. 브라우저와 HTTP 프록시 서버 사이 구간, 그리고 HTTP 프록시 서버와 목적지 웹서버 사이 구간 각각에 별도의 SSL/TLS 핸드셰이크가 수행된다 — HTTPS over Proxy에서는 브라우저-목적지 서버 간 단일 SSL/TLS 터널이 형성되며, 프록시는 터널을 중계할 뿐 별도의 핸드셰이크를 하지 않는다. 잘못된 설명이다.
라. 브라우저와 목적지 웹서버 사이에 SSL/TLS 핸드셰이크가 수행되며, HTTP 프록시 서버는 TCP를 통해 이들을 중계한다 — CONNECT 터널이 수립된 후 브라우저와 목적지 서버 간 직접 SSL/TLS 핸드셰이크가 이루어지고, 프록시는 TCP 레벨에서 중계한다. 올바른 설명이다.

각 보기 분석:
① 가, 나 — 나가 틀림.
② 가, 라 — 모두 올바른 설명. 정답.
③ 나, 다 — 나와 다 모두 틀림.
④ 다, 라 — 다가 틀림.

핵심 정리:
HTTPS over HTTP Proxy: 브라우저 → CONNECT 요청 → 프록시 TCP 터널 생성 → 브라우저-서버 간 SSL/TLS 핸드셰이크.
프록시는 암호화된 내용을 해독하지 않고 TCP 레벨에서 중계(투명 전달)한다.
