정답: ③ (3번)

해설:
PBE(Password Based Encryption)는 패스워드로부터 암호화 키를 유도하는 방식이다. KEK(Key Encryption Key)와 솔트(salt)의 역할을 정확히 이해해야 한다.

각 보기 분석:
① PBE는 Java의 java.crypto 패키지에 내장되어 있으며, PGP에서 키를 관리할 때도 사용되고 있다 — PBE는 Java 암호화 표준에 포함되며 PGP, OpenSSL 등에서 패스워드 기반 키 관리에 사용된다. 올바른 설명이다.
② 키를 암호화하는데 사용하는 키(KEK)와 메시지를 암호화하는데 사용하는 키(CEK)가 필요하다 — PBE 구조에서 패스워드로부터 KEK를 유도하고, CEK(실제 메시지 암호화 키)를 KEK로 암호화하여 저장하는 방식을 사용한다. 올바른 설명이다.
③ KEK를 생성하는데 솔트(salt) 값을 이용하면 중간자 공격(man-in-the-middle attack)을 방지할 수 있다 — 솔트는 레인보우 테이블 공격, 사전 공격(dictionary attack)을 방지하기 위해 사용된다. 중간자 공격(MITM)은 통신 채널의 공격으로 솔트와는 관계없다. 잘못된 설명이다. 정답.
④ KEK를 생성하기 위해 패스워드와 의사난수 생성기로 만들어진 솔트(salt)를 입력으로 사용한다 — PBE에서 KEK = KDF(password, salt, iteration count). 솔트는 난수로 생성된다. 올바른 설명이다.

핵심 정리:
솔트(salt)의 역할: 동일 패스워드라도 다른 KEK 생성 → 레인보우 테이블/사전 공격 방어.
솔트는 MITM 방어가 아닌 오프라인 패스워드 크래킹 방어에 사용.
PBE 구조: 패스워드 + 솔트 → KDF → KEK → CEK 암호화.
