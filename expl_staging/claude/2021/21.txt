정답: ② (2번)

해설:
공공데이터 관리지침(행정안전부 고시, 2019)에서 제시한 공공데이터 특성별 제공 방식 선정 권고사항의 빈칸을 채우는 문항이다.

공공데이터 특성별 제공 방식:
- 업데이트가 빈번한 대용량 데이터 (가): 오픈 API가 적합. 실시간·빈번한 데이터는 API를 통해 동적으로 제공하는 것이 효율적이다.
- 주기적으로 업데이트되는 파일 형태의 데이터 (나): 파일데이터 형식이 적합. 수출입통계, 범죄통계처럼 주기적으로 갱신되는 정형 파일은 파일 다운로드로 제공.
- 정적(Static) 데이터 형태로 연관성이 높은 데이터가 많은 경우 (다): LOD(Linked Open Data)가 적합. 정적이면서 연관 데이터가 많은 인물, 지명 등은 링크드 데이터로 제공.

각 보기 분석:
① (가) 오픈API, (나) LOD, (다) 파일데이터 — (나), (다) 오류.
② (가) 오픈API, (나) 파일데이터, (다) LOD — 세 빈칸 모두 정확. 정답.
③ (가) 파일데이터, (나) LOD, (다) 오픈API — 전체 오류.
④ (가) 파일데이터, (나) 오픈API, (다) LOD — (가), (나) 오류.

핵심 정리:
빈번 대용량 데이터 → 오픈 API.
주기적 파일 데이터 → 파일데이터.
정적·고연관 데이터 → LOD(Linked Open Data).
