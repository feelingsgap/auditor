정답: ③ (3번)

해설:
객체 데이터 모델(object data model)에서 사용하는 OID(Object IDentifier)에 대한 설명 중 가장 적절하지 않은 것을 찾는 문항이다.

OID의 특성:
OID는 객체 지향 데이터베이스에서 각 객체를 유일하게 식별하기 위해 시스템이 자동으로 부여하는 식별자로, 객체의 속성 값과 무관하게 생성된다.

각 보기 분석:
① OID는 객체마다 유일한 값을 가지고 있으므로 하나의 시스템 내에서 OID가 동일한 객체는 있을 수 없다 — 올바른 설명. OID는 전역적 유일성을 보장한다.
② OID는 객체가 생성될 때 시스템에 의해 생성되고 일단 생성된 뒤에는 어떠한 경우에도 변경되지 않는다 — 올바른 설명. OID는 불변(immutable)이다.
③ OID는 관계 데이터 모델에서 사용하는 기본 키와 본질적으로 동일한 성질을 가지고 있다 — 틀린 설명. OID는 시스템이 자동 부여하며 객체 속성과 무관하고 사용자가 제어할 수 없다. 반면 관계 데이터 모델의 기본 키는 데이터 속성 값을 기반으로 하며 사용자가 정의한다. OID는 값 기반이 아닌 포인터/참조 기반 식별이라는 점에서 본질적으로 다르다. 정답.
④ OID는 객체가 제거될 때만 삭제되며 이렇게 삭제된 OID는 다시 재사용할 수 없다 — 올바른 설명. OID는 객체 삭제 시에만 소멸되며 재사용되지 않는다.

핵심 정리:
OID: 시스템 자동 부여, 불변, 속성 값과 무관, 재사용 불가.
관계 DB 기본 키: 사용자 정의, 속성 값 기반, 값 변경 가능(PK 변경 시 참조 무결성 위반 발생).
OID와 기본 키는 식별 메커니즘이 본질적으로 다르다.
