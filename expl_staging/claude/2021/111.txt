정답: ③ (3번)

해설:
TCP 세션 하이재킹(session hijacking) 공격의 탐지 방법을 묻는 문항이다. TCP 세션 하이재킹은 공격자가 합법적인 TCP 세션을 가로채어 통신을 위조하는 공격이다.

각 보기 분석:
① 서버와 시퀀스 넘버를 주기적으로 체크하여 비동기화 상태에 빠지는지 탐지한다 — TCP 세션 하이재킹은 시퀀스 번호(Sequence Number)를 조작하여 세션을 가로채므로, 클라이언트-서버 간 시퀀스 번호 비동기화가 발생한다. 이를 감지하는 것은 유효한 탐지 방법이다.
② 공격자가 중간에 끼어 작동하므로 패킷의 유실 및 재전송이 증가하는지 탐지한다 — 공격자가 중간에서 패킷을 조작하면 패킷 유실과 재전송이 증가할 수 있다. 유효한 탐지 방법이다.
③ SYN 패킷의 비율이 급격히 늘어나는 현상인 SYN Storm이 발생하는지 탐지한다 — SYN Storm은 SYN Flooding(DoS 공격)의 탐지 방법이다. TCP 세션 하이재킹과는 다른 공격으로, 세션 하이재킹 탐지 방법으로 적절하지 않다. 정답.
④ 세션을 빼앗기거나 공격에 실패하는 경우 세션이 리셋되므로 예상치 못한 접속의 리셋을 탐지한다 — 세션 하이재킹 시 기존 연결이 비정상 종료(RST)될 수 있어 유효한 탐지 방법이다.

핵심 정리:
TCP 세션 하이재킹 탐지: 시퀀스 번호 비동기화, 패킷 유실 증가, 예상치 못한 RST.
SYN Storm(SYN Flood): 대량의 SYN 패킷으로 서버 자원을 고갈시키는 DoS 공격. 세션 하이재킹과 구분.
세션 하이재킹은 이미 수립된 세션을 가로채는 공격이다.
