정답: ② (2번)

해설:
TCP/IP 모델의 트랜스포트 계층(Transport Layer)에서 사용하는 포트 번호(Port Number)의 역할과 특성을 묻는 문항이다.

각 보기 분석:
① 소스 포트 번호는 일반적으로 32비트를 사용한다 — 부적절한 설명이다. 포트 번호는 16비트(0~65535)를 사용한다. 32비트는 IPv4 주소의 길이이다.
② 포트 번호는 프로세스 간의 전달(process-to-process delivery)을 위해서 사용된다 — 적절한 설명이다. IP 주소가 호스트-to-호스트 전달을 담당한다면, 포트 번호는 해당 호스트 내의 특정 프로세스를 식별하여 프로세스 간 통신을 가능하게 한다. 정답.
③ TCP에서 사용하는 포트 번호와 UDP에서 사용하는 포트 번호는 중복될 수 없다 — 부적절한 설명이다. TCP 포트와 UDP 포트는 독립적인 번호 공간을 가지므로 동일한 번호를 서로 다른 프로토콜에서 동시에 사용할 수 있다(예: DNS는 TCP 53, UDP 53 모두 사용).
④ 소스 포트 번호는 클라이언트 프로그램에서 주로 사용하는 Well-known 포트 번호와 서버 프로그램에서 주로 사용하는 동적 포트 번호 등이 있다 — 부적절한 설명이다. Well-known 포트(0~1023)는 서버 프로그램이 사용하고, 클라이언트는 동적/사설 포트(1024~65535)를 소스 포트로 사용한다. 설명이 반대이다.

핵심 정리:
포트 번호: 16비트(0~65535), 프로세스 식별에 사용.
Well-known 포트(0~1023): 서버 측 (HTTP 80, HTTPS 443, FTP 21 등).
동적 포트(1024~65535): 클라이언트가 소스 포트로 임시 사용.
TCP와 UDP의 포트 번호 공간은 독립적이다.
