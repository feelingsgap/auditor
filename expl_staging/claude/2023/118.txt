정답: ③ (3번)

해설:
애플리케이션 취약성을 이용한 공격은 소프트웨어 코드·설계 결함을 직접 공략하는 공격을 의미한다. 이와 거리가 먼 것을 찾아야 한다.

각 보기 분석:
① 버퍼 오버플로우 공격 — 애플리케이션이 입력 데이터의 크기를 검증하지 않아 발생하는 취약점을 이용한 공격이다. 소프트웨어 코드 결함을 직접 악용하는 대표적인 애플리케이션 취약성 공격이다.
② SQL 인젝션 공격 — 입력값 검증 미흡으로 인해 악의적인 SQL 쿼리를 삽입하여 데이터베이스를 조작하는 공격이다. 웹 애플리케이션의 코드 취약성을 이용한다.
③ 레인보우 테이블 공격 — 미리 계산된 해시값 목록(레인보우 테이블)을 이용하여 패스워드 해시를 역추적하는 공격이다. 애플리케이션 코드 취약점을 직접 공략하는 것이 아니라, 암호학적 해시 함수의 특성과 사전 계산을 이용하는 패스워드 크래킹 기법이다. 정답.
④ 원격 코드 실행 공격 — 애플리케이션의 취약점을 이용해 원격에서 임의의 코드를 실행시키는 공격이다. 버퍼 오버플로우, 역직렬화 취약점 등 애플리케이션 결함을 악용한다.

핵심 정리:
애플리케이션 취약성 공격: 버퍼 오버플로우, SQL 인젝션, XSS, 원격 코드 실행, 역직렬화 취약점 등.
레인보우 테이블 공격: 패스워드 해시 역추적 기법으로 애플리케이션 코드 취약성과 직접 관련 없음.
레인보우 테이블 대응: 솔트(Salt) 추가로 동일 패스워드도 다른 해시값을 갖게 만든다.
