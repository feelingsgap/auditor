정답: ② (2번)

해설:
HTTP 메소드 중 메시지 몸체(body)를 통해 신규 데이터 등록에 필요한 요청 자료를 서버에 전달하는 메소드를 묻는 문제이다.

HTTP 메소드 특성:
- GET: URL 쿼리 파라미터로 데이터를 서버에 전달. 메시지 body 미사용. 데이터 조회 목적.
- POST: 요청 메시지 body에 데이터를 담아 서버로 전송. 신규 리소스 생성 및 데이터 등록에 사용.
- PUT: 기존 리소스 전체 교체(업데이트).
- DELETE: 리소스 삭제.
- HEAD: GET과 동일하지만 body 없이 헤더만 반환. 리소스 존재 여부, 메타정보 확인.

각 보기 분석:
① GET — URL에 데이터를 포함하여 전송, body 미사용, 조회 목적.
② POST — 메시지 body를 통해 신규 데이터를 서버에 전달. 정답.
③ DELETE — 리소스 삭제 요청. 신규 데이터 등록과 무관.
④ HEAD — 헤더 정보만 반환, body 없음.

핵심 정리:
POST: 메시지 body에 데이터 포함, 서버 측 신규 리소스 생성, 멱등성 없음.
GET vs POST: GET은 URL 노출로 보안 취약, POST는 body로 전송하여 상대적으로 안전.
RESTful API에서 POST는 Create(생성) 작업에 대응한다.
