정답: ④ (4번)

해설:
PCE(Phase Containment Effectiveness)는 특정 단계에서 생성된 결함이 같은 단계 내에서 발견된 비율을 나타내는 지표이다. PCE가 클수록 해당 단계에서 자체적으로 결함을 잘 발견함을 의미한다.

PCE 계산:
PCE = (해당 단계에서 생성되어 같은 단계에서 발견된 결함 수) / (해당 단계에서 생성된 전체 결함 수)

분석 단계: 생성 결함 = 114+27+4+15 = 160, 같은 단계 발견 = 114
PCE(분석) = 114/160 = 0.7125 (71.25%)

설계 단계: 생성 결함 = 93+6+24 = 123, 같은 단계 발견 = 93
PCE(설계) = 93/123 = 0.7561 (75.61%)

구현 단계: 생성 결함 = 213+105 = 318, 같은 단계 발견 = 213
PCE(구현) = 213/318 = 0.6698 (66.98%)

각 보기 분석:
① 분석 - 설계 - 구현 — 71.25 > 75.61이 아니므로 오류.
② 분석 - 구현 - 설계 — 설계 > 분석이 아니므로 오류.
③ 설계 - 구현 - 분석 — 구현 > 분석이 아니므로 오류.
④ 설계 - 분석 - 구현 — 75.61% > 71.25% > 66.98% 순서로 정확히 일치. 정답.

핵심 정리:
PCE(Phase Containment Effectiveness) = 자체 단계 발견 결함 / 해당 단계 생성 결함.
PCE가 높을수록 결함이 다음 단계로 누출되지 않아 품질이 높다.
이 문항에서: 설계(75.6%) > 분석(71.3%) > 구현(67.0%) 순서.
