정답: ④ (4번)

해설:
관계형 데이터베이스와 NoSQL의 특성을 비교하여 NoSQL에 가까운 측면을 모두 고르는 문제이다. 각 항목이 어떤 시스템의 특성인지 판단해야 한다.

각 항목 분석:
(가) 스키마: 스키마가 없거나 변경이 자유롭다 — NoSQL의 특성. 관계형 DB는 엄격한 스키마(DDL) 정의가 필요하지만 NoSQL(문서 DB 등)은 스키마 없이 유연하게 데이터 저장 가능. NoSQL에 가까움.

(나) 확장성: 클러스터 환경에 적합하다 — NoSQL의 특성. NoSQL은 수평 확장(scale-out)에 최적화되어 분산 클러스터 환경에 적합. 관계형 DB는 주로 수직 확장(scale-up). NoSQL에 가까움.

(다) 검색기능: 단순한 데이터 검색 기능을 제공한다 — NoSQL의 특성. NoSQL은 복잡한 JOIN, 집계 연산보다 단순 키-값 검색에 최적화. 관계형 DB는 SQL로 복잡한 질의 처리 가능. NoSQL에 가까움.

(라) 트랜잭션: 일관성 유지를 보장하기 어렵다 — NoSQL의 특성. NoSQL은 ACID를 완전히 보장하지 않고 eventual consistency를 채택. 관계형 DB는 ACID 트랜잭션 보장. NoSQL에 가까움.

각 보기 분석:
① 가, 라 — 나, 다도 NoSQL에 가까우므로 불완전.
② 가, 나, 라 — 다도 NoSQL에 가까우므로 불완전.
③ 나, 다, 라 — 가도 NoSQL에 가까우므로 불완전.
④ 가, 나, 다, 라 — 네 항목 모두 NoSQL에 가까운 특성. 정답.

핵심 정리:
NoSQL 특성: 유연한 스키마, 수평 확장, 단순 검색, 완화된 일관성(eventual consistency).
관계형 DB 특성: 엄격한 스키마, 수직 확장, SQL 복잡 질의, ACID 트랜잭션.
