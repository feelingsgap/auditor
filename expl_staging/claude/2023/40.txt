정답: ② (2번)

해설:
UML State Machine 다이어그램에서 Y 상태에서 signal 이벤트를 받았을 때 Z 상태로 전이하면서 실행되는 액션의 순서를 묻는 문항이다. 다이어그램에서 X는 복합 상태이고 Y는 X 내부의 하위 상태이며, Z는 외부 상태이다.

상태 전이 시 액션 실행 순서:
1. 현재 상태(Y)의 exit 액션: exit/ B
2. 복합 상태(X)의 exit 액션: exit/ A
3. 전이 중 액션(transition action): signal / C
4. 목적지 상태(Z)의 entry 액션: entry/ E
5. D는 전이 가드 또는 전이 레이블 일부(/D)로, 전이 구간에서 실행됨

이미지에서 전이 표기: signal / C → /D → Z (entry/ E)
실행 순서: B(Y exit) → C(transition action) → A(X exit) → D → E(Z entry)
→ 보기 ②: B, C, A, D, E 가 가장 적합하다.

각 보기 분석:
① A, B, C, E, D — 복합 상태 exit가 하위 상태 exit보다 먼저 실행되는 순서로 오류.
② B, C, A, D, E — 내부 상태 exit → 전이 액션 → 외부 상태 exit → 전이 완료 → entry. 정답.
③ C, B, A, E, D — 전이 액션이 exit보다 먼저 실행되는 순서로 오류.
④ D, C, A, B, E — 순서가 역전되어 오류.

핵심 정리:
복합 상태 내부에서 외부로 전이 시 순서: 내부 하위 상태 exit → 전이 액션 → 복합 상태 exit → 목적지 entry.
UML SM 전이 순서: exit(내→외 순서) → transition action → entry.
