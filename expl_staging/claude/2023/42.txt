정답: ③ (3번)

해설:
Git 명령어 각각의 정확한 동작을 묻는 문항이다. Git은 분산 버전 관리 시스템으로 로컬 저장소와 원격 저장소 간 다양한 동기화 명령을 제공한다.

각 보기 분석:
① git push - 원격 저장소의 모든 내용을 로컬 저장소로 복사한다 — 틀린 설명이다. git push는 로컬의 변경사항을 원격 저장소로 전송하는 명령이다. 설명은 git clone 또는 git pull에 가깝다.
② git fetch - 로컬 저장소의 변경사항을 원격 저장소로 보낸다 — 틀린 설명이다. git fetch는 원격 저장소의 변경사항을 로컬로 가져오되 병합(merge)은 하지 않는 명령이다.
③ git pull - git remote 명령을 통해 서로 연결된 원격 저장소의 최신 내용을 로컬 저장소로 가져오면서 병합한다 — 올바른 설명이다. git pull은 git fetch + git merge의 결합 동작으로, 원격의 최신 내용을 로컬로 가져와 현재 브랜치에 병합한다. 정답.
④ git clone - 로컬과 원격 저장소의 변경사항을 비교 대조하고 git merge 명령과 함께 최신 데이터를 반영한다 — 틀린 설명이다. git clone은 원격 저장소를 로컬로 복제하는 최초 1회 명령이다.

핵심 정리:
git clone: 원격 저장소를 로컬에 처음 복제.
git fetch: 원격 변경사항을 가져오되 병합 없음.
git pull: fetch + merge (원격 최신 내용 가져와 병합).
git push: 로컬 변경사항을 원격으로 전송.
