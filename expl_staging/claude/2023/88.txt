정답: ③ (3번)

해설:
"상대 프로세스가 전송한 데이터를 소켓을 통해서 읽을 수 있으며, 연결성 서비스에 사용한다"는 설명에 해당하는 소켓 함수를 묻는 문제이다.

설명 분석:
- 상대 프로세스가 전송한 데이터를 읽는다 → 수신(receive) 함수
- 소켓을 통해서 읽음 → 소켓 수신 함수
- 연결성 서비스에 사용 → TCP 기반 연결 지향(connection-oriented) 서비스

소켓 함수 역할:
- connect(): 서버에 연결 요청을 보내는 클라이언트 함수.
- listen(): 서버가 연결 요청을 기다리는 상태로 설정하는 함수.
- recv(): TCP(연결 지향) 소켓에서 데이터를 수신하는 함수. 연결성 서비스에 사용.
- sendto(): UDP(비연결성) 소켓에서 데이터를 전송할 때 목적지 주소를 지정하여 보내는 함수.

각 보기 분석:
① connect() — 클라이언트가 서버에 연결을 요청하는 함수. 데이터 수신이 아님.
② listen() — 서버가 연결 대기 상태로 전환하는 함수. 데이터 수신이 아님.
③ recv() — TCP 연결 기반 소켓에서 데이터를 수신. 설명과 정확히 일치. 정답.
④ sendto() — UDP 비연결성 소켓에서 데이터 전송 시 사용. 수신이 아닌 송신 함수이며 비연결성 서비스.

핵심 정리:
연결 지향(TCP) 수신: recv()
비연결성(UDP) 수신: recvfrom() (송신 측 주소 정보도 함께 수신)
연결 지향(TCP) 송신: send()
비연결성(UDP) 송신: sendto()
