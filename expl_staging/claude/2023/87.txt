정답: ② (2번)

해설:
이더넷과 같은 링크 계층 프로토콜에서 사용하는 MAC(Media Access Control) 주소 유형 중 적절하지 않은 것을 고르는 문제이다.

링크 계층 MAC 주소 유형:
- 유니캐스트(Unicast): 특정 단일 수신자에게 전송. 모든 NIC는 고유한 유니캐스트 MAC 주소를 가짐.
- 멀티캐스트(Multicast): 특정 그룹에 속한 여러 수신자에게 동시 전송. MAC 주소의 첫 바이트 최하위 비트(I/G 비트)가 1이면 멀티캐스트.
- 브로드캐스트(Broadcast): 네트워크의 모든 장치에 전송. MAC 주소 FF:FF:FF:FF:FF:FF 사용.

애니캐스트(Anycast) 분석:
- 애니캐스트는 IP 계층(Layer 3)의 주소 유형. 동일 주소를 가진 여러 서버 중 가장 가까운 서버로 전송되는 방식.
- 링크 계층(Layer 2)의 MAC 주소 유형에는 해당하지 않음.

각 보기 분석:
① 유니캐스트(Unicast) 주소 — 링크 계층 MAC 주소 유형에 해당. 적절함.
② 애니캐스트(Anycast) 주소 — IP 계층 개념으로 링크 계층 MAC 주소 유형이 아님. 적절하지 않음. 정답.
③ 멀티캐스트(Multicast) 주소 — 링크 계층 MAC 주소 유형에 해당. 적절함.
④ 브로드캐스트(Broadcast) 주소 — 링크 계층 MAC 주소 유형에 해당. 적절함.

핵심 정리:
링크 계층 MAC 주소 유형: 유니캐스트, 멀티캐스트, 브로드캐스트.
애니캐스트: IPv6에서 도입된 Layer 3 주소 유형으로 MAC 계층에는 없음.
브로드캐스트 MAC: FF:FF:FF:FF:FF:FF (48비트 모두 1).
