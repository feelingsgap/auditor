정답: ② (2번)

해설:
웹 서버 보안 강화를 위한 조치 중 적절하지 않은 것을 찾는 문항이다. 보안 원칙은 최소 권한(Least Privilege)으로, 필요한 최소한의 권한만 부여해야 한다.

각 보기 분석:
① IIS의 WebDAV 서비스는 인터넷 상에서 원격 사용자들 간에 파일을 공동으로 사용하게 하므로 비활성화하도록 설정한다 — 적절한 보안 조치이다. 불필요한 서비스(WebDAV)는 공격 면적을 넓히므로 비활성화가 권장된다.
② 웹 서버를 실행시키는 운영체제의 사용자에게 충분한 서비스를 제공하기 위하여 최대한 많은 권한을 허용한다 — 부적절한 보안 조치이다. 최소 권한 원칙(Principle of Least Privilege)에 위배된다. 웹 서버 프로세스 계정에는 서비스 운영에 필요한 최소 권한만 부여해야 한다. 정답.
③ 웹 브라우저가 웹 서버로 디렉토리의 정보를 요청했을 때 디렉토리를 검색해 파일 목록을 보여주는 기능은 불가능하도록 설정한다 — 적절한 보안 조치이다. 디렉토리 리스팅(Directory Listing) 비활성화는 파일 구조 노출을 방지한다.
④ 파일 업로드 및 다운로드 시 허용 가능한 최대 크기를 설정한다 — 적절한 보안 조치이다. 파일 크기 제한으로 DoS 공격 및 대용량 악성 파일 업로드를 방지한다.

핵심 정리:
웹 서버 보안 핵심 원칙: 최소 권한, 불필요한 서비스 비활성화, 디렉토리 리스팅 비활성화.
웹 서버 계정에 과도한 권한 부여는 취약점 악용 시 시스템 전체 침해로 이어질 수 있다.
