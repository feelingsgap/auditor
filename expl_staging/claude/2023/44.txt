정답: ① (1번)

해설:
소프트웨어 컴포넌트 Cc의 안정성(stability) 지표를 계산하는 문항이다. 안정성은 Fan-in과 Fan-out으로 계산되며, 불안정성 지표 I(Instability) = Fan-out / (Fan-in + Fan-out)으로 산출한다.

Cc 컴포넌트 분석 (이미지의 다이어그램 기준):
Cc는 내부에 t, u 모듈을 가진다.
- Cc로 들어오는(Cc에 의존하는) 컴포넌트: Ca(q, r), Cb(s) → Ca와 Cb 2개 + 내부 의존 포함
- 이미지를 보면 Ca와 Cb가 Cc(t, u)를 사용하고, Cc는 Cd를 사용한다.
Fan-in(Cc에 의존하는 외부 컴포넌트): Ca, Cb → 2개가 아닌, q, r, s 3개 모듈이 들어오므로 Fan-in = 3
Fan-out(Cc가 의존하는 외부 컴포넌트): Cd(v) → 1개 → Fan-out = 1
I(Instability) = 1 / (3+1) = 1/4

각 보기 분석:
① Fan-in: 3, Fan-out: 1, I(Instability): 1/4 — 위 계산과 일치. 정답.
② Fan-in: 1, Fan-out: 3, I: 1/4 — Fan-in과 Fan-out이 역전되어 있고 계산도 불일치.
③ Fan-in: 1, Fan-out: 3, I: 3/4 — Fan-in, Fan-out 역전 오류.
④ Fan-in: 1, Fan-out: 3, I: 3/4 — 동일하게 오류.

핵심 정리:
Fan-in: 해당 컴포넌트를 사용(의존)하는 다른 컴포넌트 수 → 높을수록 안정.
Fan-out: 해당 컴포넌트가 사용하는 다른 컴포넌트 수 → 높을수록 불안정.
I(Instability) = Fan-out / (Fan-in + Fan-out): 0에 가까울수록 안정적.
