정답: ③ (3번)

해설:
파일 시스템에서 디스크 클러스터 끝과 파일의 실제 데이터 끝 사이의 빈 공간을 설명하는 용어를 찾는 문항이다. 이 공간에는 이전에 삭제된 파일의 내용이 남아 있을 수 있어 보안 위협이 된다.

각 보기 분석:
① 저널(journal) — 파일 시스템의 변경 사항을 기록하여 장애 시 복구에 사용하는 로그 영역이다. 클러스터 빈 공간과 관련 없다.
② 메타데이터(metadata) — 파일의 이름, 크기, 생성 날짜, 권한 등 파일에 관한 정보를 담는 데이터이다. 클러스터 내 빈 공간이 아니다.
③ 파일 슬랙(file slack) — 파일에 할당된 디스크 클러스터들이 끝나는 위치와 파일의 실제 데이터가 끝나는 위치 사이의 빈 공간이다. 이전에 삭제된 파일의 내용이 남을 수 있어 디지털 포렌식과 보안의 관점에서 중요하다. 지문 설명과 정확히 일치. 정답.
④ 파일 지시자(file descriptor) — 운영체제가 열린 파일을 관리하기 위해 사용하는 정수 식별자이다. 디스크 물리적 공간과 관련 없다.

핵심 정리:
파일 슬랙(file slack): 클러스터 크기와 파일 크기의 차이로 발생하는 잔여 공간.
램 슬랙(RAM slack): 마지막 섹터 내 파일 데이터 이후 섹터 끝까지의 공간 (0으로 채워지거나 메모리 내용이 남음).
디스크 슬랙(disk slack): 클러스터의 마지막 섹터 이후 남은 섹터 공간.
파일 슬랙은 완전 삭제 시 전체 영역을 덮어써야 완전히 제거된다.
