정답: ③ (3번)

해설:
웹서버 보안을 위해 위험한 형식의 파일 업로드를 막기 위한 보안 대책을 묻는 문항이다. 파일 업로드 취약점은 웹셸(Web Shell) 업로드 등 심각한 공격으로 이어질 수 있어 다층적 방어가 필요하다.

각 보기 분석:
① 파일들이 서버에서 실행될 수 없도록 실행 속성을 제거한다 — 업로드된 파일의 실행 권한을 제거하면 악성 스크립트가 서버에서 실행되는 것을 방지할 수 있다. 적절한 보안 대책이다.
② 화이트 리스트 방식으로 허용된 형식의 파일만 업로드를 허용한다 — 허용된 확장자와 MIME 타입을 화이트리스트로 제한하면 위험한 파일 형식(php, jsp, asp 등)의 업로드를 차단할 수 있다. 적절한 보안 대책이다.
③ 파일들이 저장되는 경로가 웹 서버의 문서 디렉터리 내부에 한정되도록 설정한다 — 업로드 파일을 웹 서버의 문서 디렉터리(웹 루트) 내부에 저장하면 오히려 위험하다. 업로드 파일은 웹 루트 외부 경로에 저장하거나, 웹 서버가 직접 실행할 수 없는 별도 공간에 저장해야 한다. 문서 디렉터리 내부에 저장하면 업로드된 웹셸이 URL로 직접 접근·실행될 수 있다. 틀린 설명이므로 정답.
④ 서버에 파일을 저장할 때 파일 이름과 경로를 사용자가 유추할 수 없는 임의의 문자열로 변경하여 저장한다 — 파일명을 랜덤한 문자열로 변경하면 공격자가 업로드한 파일의 위치를 추측하기 어려워진다. 적절한 보안 대책이다.

핵심 정리:
파일 업로드 보안: 화이트리스트 확장자 검증, 실행 권한 제거, 웹 루트 외부 저장, 파일명 난수화.
업로드 경로를 웹 루트 내부로 한정하는 것은 오히려 보안 취약점이 된다.
Content-Type 헤더만 검증하는 것은 우회 가능하므로, 파일 내용(매직 바이트)도 검사해야 한다.
