정답: ① (1번)

해설:
두 가지 상황에서 사용할 수 있는 UML 다이어그램으로 가장 적절한 것을 찾는 문항이다.

제시된 상황:
가. 전체 시스템 수준에서 시스템과 상호작용하는 각 액터의 관점에서 시스템을 모델링하고자 함.
나. 시스템 객체 사이의 어떤 제어흐름을 밝알고 설계, 동기화된 필요성 또는 병렬 수행 가능성을 파악하고자 함.

각 다이어그램 特性 분석:
① Activity Diagram (활동 다이어그램) — 작업 흐름, 제어 흐름, 병렬 처리, 동기화를 표현하는 데 적합하다. 나의 상황에 부합하나, 가의 액터-시스템 상호작용 모델링에는 Use Case Diagram이 더 적합하다.

그러나 Activity Diagram은 두 상황 모두에 사용 가능하다:
가 상황: 시스템 수준에서 액터와 시스템 간 활동 흐름을 스윔레인(swimlane)으로 모델링 가능.
나 상황: 제어흐름, 동기화, 병렬 수행을 포크/조인(fork/join)으로 표현 가능.

② State Diagram — 객체 생명주기의 상태 변화를 모델링. 나 상황의 병렬 수행 표현에 제한적.
③ Class Diagram — 정적 구조(클래스, 관계) 표현. 두 상황 모두 부합하지 않음.
④ Use Case Diagram — 액터와 시스템 간 기능적 상호작용 표현. 가 상황에는 적합하나 나의 제어흐름/병렬 수행 표현에는 부적합.

핵심 정리:
Activity Diagram: 액터별 스윔레인으로 시스템 수준 행위 표현 + 포크/조인으로 병렬 흐름 표현.
두 상황을 동시에 지원하는 UML 다이어그램은 Activity Diagram이다.
Use Case Diagram은 기능 열거에 강하나 제어흐름·동기화 표현에 한계가 있다.
