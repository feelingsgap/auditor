정답: ① (1번)

해설:
SysML(Systems Modeling Language) 다이어그램 종류별 특징을 파악하고, 주어진 두 가지 설명에 해당하는 다이어그램을 찾는 문항이다.

주어진 설명 분석:
가. 운영 시스템에 유효한 시스템 변수 값을 표현할 수 있어 예외 상황의 발생을 탐지할 수 있도록 한다.
나. 제약 조건 변수와 값 속성 간의 바인딩을 통해 값 속성 집합에 수학적 관계를 지정한다.

두 설명 모두 수학적 제약 조건과 변수 간의 관계를 다루며, 이는 Parametric diagram의 핵심 특징이다.

각 보기 분석:
① Parametric diagram — 블록의 값 속성 간 수학적·물리적 제약 관계를 표현하는 다이어그램. 제약 블록(Constraint Block)과 바인딩 커넥터를 사용해 변수 간 수식 관계를 정의하고, 시스템 변수의 유효 범위를 분석하여 예외 상황 탐지에 활용된다. 설명 가·나 모두 해당. 정답.
② Package diagram — 모델 요소의 논리적 그룹화와 의존 관계를 표현. 수학적 제약 조건과 무관.
③ State machine diagram — 시스템 또는 객체의 상태 전이를 표현. 이벤트·조건·액션으로 구성되며 수학적 바인딩 관계가 핵심이 아님.
④ Block definition diagram (BDD) — 블록의 구조적 계층과 속성을 정의하는 다이어그램. 구성 요소 간 수학적 제약 관계 표현은 Parametric diagram의 역할.

핵심 정리:
SysML Parametric diagram: 값 속성(Value Property) 간 수학적 제약을 Constraint Block으로 표현.
바인딩 커넥터(Binding Connector)로 제약 변수와 블록 속성을 연결.
성능 분석, 신뢰성 분석, 트레이드오프 분석 등 정량적 분석에 활용된다.
