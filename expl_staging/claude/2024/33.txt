정답: ④ (4번)

해설:
C 언어 이중 포인터(포인터 배열)와 포인터 연산을 추적하여 출력 결과를 구하는 문항이다.

코드 분석:
int a, b, c;
ary[4] = {10, 20, 30, 40}, {50, 60, 70, 80} (2차원 배열 형태)
int (*ptr)[4] = ary;  // ptr은 4개 int 배열을 가리키는 포인터, ary[0]을 가리킴

a = *(*(ptr + 1) + 3)
  - ptr + 1 → ary[1] ({50, 60, 70, 80})을 가리킴
  - *(ptr + 1) → ary[1] 배열의 시작 주소
  - *(ptr + 1) + 3 → ary[1][3] 위치
  - *(*(ptr + 1) + 3) = ary[1][3] = 80
  → a = 80

b = **ptr++
  - 후위 ++ : 현재 ptr 값(ary[0]을 가리킴)을 사용한 후 ptr를 ary[1]로 증가
  - **ptr (증가 전) = *ary[0] = ary[0][0] = 10
  → b = 10, ptr는 이후 ary[1]을 가리킴

c = **ptr
  - ptr는 현재 ary[1]을 가리킴
  - **ptr = *ary[1] = ary[1][0] = 50
  → c = 50

printf("%d\n", a+b+c) = 80 + 10 + 50 = 140

각 보기 분석:
① 80 — a만의 값. b와 c를 더하지 않은 오답.
② 100 — 포인터 연산 오류로 인한 오답.
③ 120 — 포인터 증가 시점 혼동으로 인한 오답.
④ 140 — a(80) + b(10) + c(50) = 140. 정답.

핵심 정리:
후위 ++ 연산자: 현재 값을 먼저 사용한 후 증가 (b = **ptr++은 증가 전 ptr로 역참조).
int (*ptr)[4]: 4개 int 배열을 가리키는 포인터. ptr+1은 다음 4개 int 블록으로 이동.
이중 역참조(**ptr): 포인터가 가리키는 배열의 첫 번째 원소에 접근.
