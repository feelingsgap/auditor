정답: ② (2번)

해설:
제시된 설명에 해당하는 보안 개념을 찾는 문항이다. 설명은 "조직이 자신에게 적합한 보안 정책을 수립하고 수립된 보안 정책에 의해 보안 관련 조직을 구성하여 일련의 보안 활동을 수행하는 것"이다.

각 보기 분석:
① 권한관리 — 사용자나 시스템에 부여된 접근 권한을 관리·통제하는 활동으로, 조직 차원의 보안 정책 수립과 조직 구성을 포괄하는 개념이 아니다.
② 보안 거버넌스 — 조직의 보안 목표 달성을 위해 보안 전략과 정책을 수립하고, 이를 실행하기 위한 조직 체계를 구성하며, 보안 활동 전반을 지휘·감독하는 체계다. 제시된 설명과 정확히 일치한다. 정답.
③ 위험분석 — 조직의 정보자산에 대한 위협과 취약점을 식별하고, 위험의 크기를 측정하는 과정이다. 정책 수립과 조직 구성을 포괄하는 상위 개념이 아니다.
④ 접근통제 — 인가된 사용자만 시스템, 네트워크, 데이터에 접근할 수 있도록 통제하는 보안 기술이다. 보안 활동 전반을 관장하는 개념이 아니다.

핵심 정리:
보안 거버넌스: 조직의 보안 정책 수립 → 보안 조직 구성 → 보안 활동 수행 체계.
IT 거버넌스의 하위 개념으로, 이사회·경영진 수준의 보안 방향 설정 및 책임 소재 명확화 포함.
ISO/IEC 27001의 ISMS(정보보호 관리체계)와 밀접하게 연관된다.
