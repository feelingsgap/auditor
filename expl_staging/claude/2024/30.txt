정답: ③ (3번)

해설:
코드의 (가)와 (나)에서 나타나는 객체지향 개념을 찾는 문항이다.

코드 분석:
```
ClosedFigure closedFigure;
Rectangle r = new Rectangle();
Circle c = new circle();

closedFigure = r;
closedFigure.getArea();  ......  (가)
closedFigure = c;
closedFigure.getArea();  ......  (나)
```

closedFigure 변수에 Rectangle 객체와 Circle 객체를 번갈아 할당하고 동일한 getArea() 메서드를 호출한다. 같은 메서드 호출이지만 실제 실행 시점에 바인딩된 객체 타입에 따라 다른 동작(Rectangle의 getArea와 Circle의 getArea)이 수행된다.

각 개념 분석:
① 상속(inheritance) — 부모 클래스의 속성과 메서드를 자식 클래스가 물려받는 것. 코드에서 상속 관계는 존재할 수 있으나 (가)(나)에서 직접 나타나는 개념이 아니다.
② 정보은닉(information hiding) — 객체 내부 구현을 외부로부터 숨기는 것. 이 코드에서 직접 나타나는 개념이 아니다.
③ 다형성(polymorphism) — 동일한 인터페이스(getArea())로 서로 다른 타입(Rectangle, Circle)의 객체를 동작시키는 것. (가)와 (나) 모두 동일한 메서드 호출로 각기 다른 동작을 수행한다. 정답.
④ 의존성(dependency) — 한 클래스가 다른 클래스에 의존하는 관계. 이 코드의 핵심 개념이 아니다.

핵심 정리:
다형성(polymorphism): 동일한 메서드 이름으로 객체 타입에 따라 다른 동작 수행.
대표 구현: 오버라이딩(runtime polymorphism), 오버로딩(compile-time polymorphism).
이 코드는 런타임 다형성으로, ClosedFigure 타입 변수에 Rectangle·Circle을 대입하여 동일 메서드 호출 시 각각의 구현이 실행된다.
