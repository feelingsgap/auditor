정답: ① (1번)

해설:
큰 데이터 블록을 전송할 때 CPU의 개입 없이 메모리와 I/O 장치 간에 데이터를 직접 전송하는 메커니즘을 찾는 문항이다.

문제 상황:
큰 데이터 블록 전송 시 CPU가 동작하느라 많은 시간을 소모하고, 시스템 버스를 대부분 I/O에 사용해야 한다. CPU의 개입 없이 I/O 장치와 기억장치 간에 데이터 전송을 수행할 수 있는 메커니즘이 필요하다.

각 보기 분석:
① DMA(Direct Memory Access) — CPU를 거치지 않고 I/O 장치와 메모리 간에 직접 데이터를 전송하는 방식. DMA 컨트롤러가 전송을 관리하고 완료 후 CPU에 인터럽트를 보낸다. 문제의 조건과 정확히 일치한다. 정답.
② Daisy-chain — 인터럽트 우선순위를 결정하기 위해 여러 장치를 직렬로 연결하는 방식. 데이터 전송 방식이 아니다. 틀렸다.
③ Software polling — CPU가 주기적으로 I/O 장치 상태를 확인하는 방식. CPU가 계속 관여하므로 문제의 조건(CPU 개입 없이)과 반대다. 틀렸다.
④ Memory-mapped I/O — I/O 장치를 메모리 주소 공간에 매핑하여 메모리 접근 명령으로 I/O를 제어하는 방식. CPU가 여전히 개입하며 데이터 전송 효율화 메커니즘이 아니다. 틀렸다.

핵심 정리:
DMA: CPU 개입 없이 메모리 ↔ I/O 직접 전송. 전송 완료 후 인터럽트로 CPU에 통보.
DMA 사용 시 CPU는 전송 중 다른 작업 수행 가능 (CPU 효율 향상).
Cycle Stealing: DMA 컨트롤러가 버스 제어권을 일시적으로 가져가는 방식.
