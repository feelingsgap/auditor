정답: ④ (4번)

해설:
릴레이션 R과 S의 속성 A를 통한 이원 조인(two-way join) 구현 방법 중, R과 S가 조인 속성 A에 대해 오름차순으로 정렬되어 있는 경우에만 적용 가능한 방법을 찾는 문항이다.

각 조인 구현 방법 분석:
① 중첩 루프(Nested Loop Join) — 외부 릴레이션의 각 튜플에 대해 내부 릴레이션 전체를 스캔하는 방식. 정렬 여부와 관계없이 항상 적용 가능. 정렬 조건 불필요.
② 인덱스 검사(Index Nested Loop Join) — 내부 릴레이션의 조인 속성에 인덱스가 있을 때 인덱스를 이용해 검색을 가속. 정렬이 아닌 인덱스 존재가 조건.
③ 해시 검사(Hash Join) — 조인 속성에 해시 함수를 적용하여 같은 버킷에 속하는 튜플끼리 조인. 정렬 조건 불필요. 대규모 데이터에 효율적.
④ 정렬 합병(Sort-Merge Join) — 두 릴레이션이 조인 속성으로 정렬된 상태에서, 양쪽을 동시에 스캔하며 병합(merge). 정렬이 선행되어야만 적용 가능한 조건이 있으며, 이미 정렬된 경우 매우 효율적. 정답.

핵심 정리:
Sort-Merge Join 적용 조건: 두 릴레이션 모두 조인 속성 기준으로 정렬된 상태.
정렬 비용이 크지만 이미 정렬된 경우 O(n+m)의 선형 시간 조인 가능.
인덱스가 없고 해시 불가 상황에서 대규모 데이터의 동등 조인에 유리.
비교: Nested Loop=O(n×m), Hash Join=O(n+m) but 메모리 필요, Sort-Merge=O(n+m) but 정렬 필요.
