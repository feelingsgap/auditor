정답: ③ (3번)

해설:
Apache Spark에서 관계형 데이터베이스의 릴레이션에 해당하는 데이터 표현 방식과 그 특성을 설명하는 용어를 찾는 문항이다.

설명 분석:
"관계형 데이터베이스가 데이터 표현을 위한 추상화로 릴레이션을 사용하는 것처럼 Apache Spark은 ㉮라 불리는 데이터 표현 방식을 사용한다. Spark에서 연산자는 하나 이상의 ㉮를 입력으로 받아 출력으로 ㉮를 반환하는 방식으로 계산을 수행한다."

RDD(Resilient Distributed Dataset):
- Spark의 핵심 데이터 추상화 단위.
- 분산 환경에서 불변(immutable)하고 병렬 처리 가능한 데이터 컬렉션.
- 연산자(transformation/action)는 RDD를 입력으로 받아 새 RDD를 반환한다.
- "Resilient" = 장애 복구 가능, "Distributed" = 분산 처리, "Dataset" = 데이터 집합.

각 보기 분석:
① HDFS(Hadoop Distributed File System) — Hadoop의 분산 파일 시스템. 스토리지 레이어로 데이터 표현 추상화 단위가 아니다. 틀렸다.
② HDFS(Hadoop Duplicated File System) — 'Duplicated'는 잘못된 풀이어다. HDFS는 Hadoop Distributed File System이다. 틀렸다.
③ RDD(Resilient Distributed Dataset) — Spark의 데이터 표현 추상화 단위로 설명과 정확히 일치한다. 정답.
④ RDD(Reproducible Duplicated Dataset) — RDD의 풀이가 틀렸다. Resilient Distributed Dataset이 올바른 명칭이다. 틀렸다.

핵심 정리:
RDD = Resilient Distributed Dataset: Spark의 기본 데이터 추상화.
RDD 특성: 불변성(immutable), 지연 평가(lazy evaluation), 장애 내성(fault tolerance, lineage 기반).
Spark 연산: Transformation(새 RDD 생성, 지연 실행) + Action(결과 반환, 즉시 실행).
