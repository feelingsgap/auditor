정답: ③ (3번)

해설:
제시된 C언어 코드의 취약점을 분석하는 문항이다. 코드는 char name[100] 배열을 선언하고 scanf("%99s", name)으로 입력을 받은 후 printf(name)으로 출력하는 구조다.

코드 분석:
```c
char name[100];
printf("Enter your name: ");
scanf("%99s", name);
printf(name);
```

포맷 스트링 취약점 분석:
printf(name)은 name 변수를 포맷 문자열로 직접 사용하고 있다. 사용자가 name에 "%x", "%s", "%n" 등의 포맷 지정자를 입력하면, printf가 이를 형식 문자열로 해석하여 스택 메모리 내용 노출 또는 임의 메모리 쓰기가 가능해진다.

각 보기 분석:
① 스택 오버플로우 취약점 — scanf("%99s", name)에서 %99s로 입력 길이를 99자로 제한하고 있어, 100바이트 배열에 대한 스택 오버플로우는 방지되어 있다. 이 코드의 주된 취약점이 아니다.
② 경쟁 조건 취약점 — 멀티스레드 환경에서 공유 자원에 동시 접근할 때 발생하는 취약점이다. 이 코드와 무관하다.
③ 포맷 스트링 취약점 — printf(name)처럼 사용자 입력값을 포맷 문자열로 직접 전달하면 포맷 스트링 공격이 가능하다. 올바른 코드는 printf("%s", name)이다. 정답.
④ 명령어 주입 취약점 — system() 등 셸 명령을 실행하는 함수에 외부 입력이 전달될 때 발생한다. 이 코드에는 해당 패턴이 없다.

핵심 정리:
포맷 스트링 취약점: printf(user_input) → printf("%s", user_input) 으로 수정 필요.
%n 포맷 지정자를 통해 임의 메모리 쓰기, %s로 메모리 내용 읽기 가능.
보안 코딩 원칙: 사용자 입력을 포맷 문자열로 직접 사용하지 않는다.
