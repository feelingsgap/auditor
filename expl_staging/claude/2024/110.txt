정답: ② (2번)

해설:
외부 IP 주소의 임의 포트에서 내부 FTP 서버(포트 21)로 접속하려는 트래픽을 탐지 및 차단하기 위한 Snort 탐지규칙을 찾는 문항이다.

Snort 규칙 문법 분석:
[액션] [프로토콜] [출발지IP] [출발지포트] → [목적지IP] [목적지포트] (옵션)
- alert: 탐지 및 알림
- drop: 차단
- !192.168.0.0/24: 내부 네트워크가 아닌 외부 IP
- any: 임의의 포트
- 포트 21: FTP 제어 포트

요구사항:
외부(비내부) IP에서 내부 FTP 서버로 접속 시 차단(drop) 또는 탐지(alert) 및 메시지 생성.

각 보기 분석:
① alert tcp !192.168.0.0/24 any → 192.168.0.0/24 21 (msg: "illegal ftp attempt") — 탐지는 하지만 차단은 아닌 alert 규칙이다. 문제가 "탐지 및 차단"을 요구하므로 alert만으로는 차단이 이루어지지 않는다.
② drop tcp !192.168.0.0/24 any → 192.168.0.0/24 21 (msg: "illegal ftp attempt") — 외부(!192.168.0.0/24)의 임의 포트(any)에서 내부(192.168.0.0/24) FTP 포트(21)로의 TCP 트래픽을 차단(drop)한다. 탐지와 차단을 동시에 수행하는 올바른 규칙이다. 정답.
③ alert tcp 192.168.0.0/24 any → !192.168.0.0/24 21 (msg: "illegal ftp attempt") — 내부에서 외부의 포트 21로 나가는 트래픽을 탐지하는 규칙으로, 요구사항과 방향이 반대다.
④ drop tcp 192.168.0.0/24 any → !192.168.0.0/24 21 (msg: "illegal ftp attempt") — 내부에서 외부 FTP 포트로의 트래픽을 차단하는 규칙으로, 요구사항과 방향이 반대다.

핵심 정리:
Snort drop 규칙: 패킷 차단 + 로그 기록.
!IP/CIDR: 해당 네트워크 대역을 제외한 나머지(외부).
출발지/목적지 방향 정확히 구분: 외부→내부 vs 내부→외부.
