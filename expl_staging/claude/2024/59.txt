정답: ② (2번)

해설:
인덱스 ROWID를 이용하여 테이블 블록을 읽는 전체 메커니즘의 올바른 순서를 찾는 문항이다.

ROWID 기반 블록 읽기 메커니즘:
ROWID는 오브젝트 번호, 데이터 파일 번호, 블록 번호와 같은 물리적 요소로 구성되어 있으며, 이를 이용해 특정 블록을 버퍼 캐시에서 조회하거나 디스크에서 읽어온다.

단계별 순서 분석:
ㄹ. 인덱스에서 하나의 ROWID를 읽고 디스크상의 블록 위치정보를 해시 함수에 적용해 해시값을 확인한다.
→ 가장 먼저 ROWID로 해당 블록의 해시 버킷을 결정.

ㄷ. 해시 버킷에 연결된 해시 체인을 스캔하면서 블록 헤더를 찾는다.
→ 해시값으로 해시 버킷을 찾고, 체인을 탐색하여 해당 블록이 이미 버퍼 캐시에 있는지 확인.

ㄱ. LRU 리스트를 스캔하면서 빈 버퍼를 찾는다.
→ 블록이 캐시에 없으면 디스크에서 읽어야 하므로 빈 버퍼 확보를 위해 LRU 리스트 탐색.

ㅁ. Dirty 버퍼를 디스크에 기록해 빈 버퍼를 확보한다.
→ 빈 버퍼가 없으면 Dirty 버퍼(변경되었으나 미기록 버퍼)를 디스크에 플러시하여 공간 확보.

ㄴ. 디스크에서 블록을 읽어 버퍼 캐시에 적재한다.
→ 확보된 빈 버퍼에 디스크에서 읽어온 블록을 적재.

각 보기 분석:
① ㅁ-ㄱ-ㄴ-ㄹ-ㄷ — ROWID 탐색 전에 Dirty 버퍼 기록이 먼저 발생하는 오류.
② ㄹ-ㄷ-ㄱ-ㅁ-ㄴ — ROWID 해시 → 해시체인 탐색 → LRU 빈 버퍼 탐색 → Dirty 버퍼 기록 → 블록 적재. 올바른 순서. 정답.
③ ㄷ-ㄹ-ㄱ-ㄴ-ㅁ — 해시체인 탐색이 ROWID 확인보다 먼저 발생하는 오류.
④ ㄹ-ㄷ-ㅁ-ㄱ-ㄴ — ㄱ(빈 버퍼 탐색) 전에 ㅁ(Dirty 버퍼 기록)이 먼저 발생. 빈 버퍼가 없다는 것을 확인한 후 Dirty 버퍼를 기록해야 하므로 순서 오류.

핵심 정리:
ROWID 블록 읽기 순서: ROWID→해시값 계산→해시체인 탐색→(캐시 미스 시)LRU 빈 버퍼 탐색→Dirty 버퍼 기록→디스크 읽기.
Buffer Cache Hit: 해시체인 탐색 단계에서 블록 발견 시 ㄹ-ㄷ로 완료.
LRU 알고리즘: 가장 오래 사용되지 않은 버퍼를 교체 대상으로 선택.
