정답: ① (1번)

해설:
NAT(Network Address Translation)와 사설 IP 주소에 대한 설명 중 옳지 않은 것을 찾는 문항이다.

사설 IP 주소 범위 (RFC 1918):
- 10.0.0.0 ~ 10.255.255.255 (10.0.0.0/8)
- 172.16.0.0 ~ 172.31.255.255 (172.16.0.0/12)
- 192.168.0.0 ~ 192.168.255.255 (192.168.0.0/16)

각 보기 분석:
① 13.234.23.87은 사설 IP 주소로 활용할 수 있다 — 13.234.23.87은 공인 IP 주소 범위에 속하며 사설 IP 주소가 아니다. RFC 1918에서 정의하는 사설 IP 범위(10.x.x.x, 172.16-31.x.x, 192.168.x.x)에 해당하지 않는다. 틀린 설명이다. 정답.
② 사설 IP 주소와 공인 IP 주소를 1대1로 고정적으로 매핑하는 것을 Static NAT라고 한다 — Static NAT는 고정 1:1 매핑 방식이다. 옳은 설명이다.
③ 내부 네트워크(LAN)에서 사용하는 사설 IP 주소와 외부 인터넷에서 사용하는 공인 IP 주소를 변환하는 것을 의미한다 — NAT의 기본 정의와 일치한다. 옳은 설명이다.
④ IP 주소뿐만 아니라 포트 번호를 사용하는 NAPT(Network Address Port Translation) 또는 PAT(Port Address Translation) 방법이 있다 — NAPT/PAT는 포트 번호까지 변환하여 하나의 공인 IP로 다수의 사설 IP를 지원하는 방식이다. 옳은 설명이다.

핵심 정리:
사설 IP 범위: 10.x.x.x, 172.16-31.x.x, 192.168.x.x (RFC 1918).
13.x.x.x는 공인 IP 범위로 사설 IP로 사용 불가.
Static NAT: 1:1 고정 매핑. Dynamic NAT: 풀에서 동적 할당. PAT/NAPT: 포트 포함 N:1 매핑.
