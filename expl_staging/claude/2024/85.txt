정답: ④ (4번)

해설:
데이터 통신에서 사용하는 에러 검출 방식의 종류를 파악하고, 에러 검출 방식이 아닌 것을 찾는 문항이다.

데이터 통신 에러 검출 방식:
- 블록합검사(Block Sum Check): 블록 내 각 비트 위치별로 패리티를 계산하는 2차원 패리티 방식.
- 패리티검사(Parity Check): 데이터 비트에 패리티 비트를 추가하여 1의 개수의 홀수/짝수를 확인하는 방식.
- 순환중복검사(CRC, Cyclic Redundancy Check): 다항식 나눗셈을 이용한 강력한 에러 검출 방식. 통신 및 저장장치에서 널리 사용된다.
- 허프만코딩(Huffman Coding): 빈도 기반 가변 길이 코드를 생성하는 데이터 압축(무손실 부호화) 방식. 에러 검출 목적이 아니다.

각 보기 분석:
① 블록합검사(Block Sum Check) — 에러 검출 방식이다. 옳다.
② 패리티검사(Parity Check) — 에러 검출 방식이다. 옳다.
③ 순환중복검사(Cyclic Redundancy Check) — 에러 검출 방식이다. 옳다.
④ 허프만코딩검사(Huffman Coding Check) — 허프만 코딩은 데이터 압축 알고리즘으로, 에러 검출과는 무관하다. 정답(에러 검출 방식 아님).

핵심 정리:
에러 검출 방식: 패리티 검사, 블록합 검사, CRC, 체크섬(Checksum).
에러 수정 방식: 해밍 코드(Hamming Code), Reed-Solomon 코드.
허프만 코딩: 무손실 데이터 압축 알고리즘 (에러 검출·수정과 무관).
