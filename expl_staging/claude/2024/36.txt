정답: ② (2번)

해설:
IEEE 29119의 Base Choice 테스트 설계 방법에서 테스트케이스 수를 계산하는 문항이다.

Base Choice 방법:
각 입력 파라미터에서 "기본(base)" 값을 하나씩 선택한 후, 하나의 파라미터만 변경하고 나머지는 기본 값을 유지하는 방식으로 테스트케이스를 생성한다.

테스트케이스 수 계산:
- 기본 테스트케이스 (Base Case): 1개 (모든 파라미터에 기본 값)
- 글꼴 이름 변경: 4개 값 중 기본 1개 제외 → 3개 추가
- 글꼴 크기 변경: 5개 값(10, 12, 14, 16, 18) 중 기본 1개 제외 → 4개 추가
- 스타일 변경: 3개 값(밑줄, 기울임, 보통) 중 기본 1개 제외 → 2개 추가

총합: 1 + 3 + 4 + 2 = 10개

각 보기 분석:
① 5개 — 파라미터 수(3)와 기본 조합만 고려한 오류.
② 10개 — Base Case 1 + (4-1) + (5-1) + (3-1) = 10. 정답.
③ 20개 — 각 파라미터 값의 단순 합산 또는 다른 계산 오류.
④ 60개 — 4 × 5 × 3 = 60으로, 모든 조합을 생성하는 조합적 테스트(Combinatorial Testing)의 총 경우의 수. Base Choice가 아닌 전수 조합.

핵심 정리:
Base Choice 공식: 1 + Σ(각 파라미터의 값 수 - 1)
목적: 전수 조합(Exhaustive)보다 훨씬 적은 테스트케이스로 각 파라미터의 비기본(non-base) 값을 최소 한 번씩 커버.
조합적 테스트(Pairwise 등)와 구분: Base Choice는 한 번에 하나의 파라미터만 변경.
