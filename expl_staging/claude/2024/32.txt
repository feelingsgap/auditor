정답: ② (2번)

해설:
코드 리팩토링 전후를 비교하여 적용된 리팩토링 기법을 식별하는 문항이다.

코드 변화 분석:
적용 전: if (inValue > 50) inValues++;
적용 후: int grade = inValue; if (inValue > 50) grade++;

변화의 핵심: 매개변수 inValue(또는 inValues)에 직접 값을 증가시키는 대신, 새로운 지역변수 grade를 도입하여 매개변수 대신 지역변수를 수정하도록 변경했다.

각 보기 분석:
① 임시변수 분리 기법(Split temporary variable) — 하나의 임시변수가 여러 다른 목적으로 재사용될 때 각 용도마다 별도 변수를 도입하는 기법. 이 코드는 임시변수 재사용 문제가 아니므로 해당 없음.
② 매개변수 할당 제거 기법(Remove assignments to parameters) — 메서드 내에서 매개변수에 직접 값을 할당하는 것은 코드 가독성을 낮추고 부작용을 유발할 수 있다. 이를 해결하기 위해 매개변수 값을 지역변수에 복사한 후 지역변수를 사용하는 기법. 코드 변화와 정확히 일치. 정답.
③ 중간자 제거 기법(Remove middle man) — 위임(delegation)이 과도하게 사용된 경우 중간 클래스를 제거하는 기법. 이 코드와 무관.
④ 조건 다형성 대체 기법(Replace conditional with polymorphism) — 조건문(if/switch)을 다형성(상속·오버라이딩)으로 교체하는 기법. 이 코드의 변화와 무관.

핵심 정리:
Remove assignments to parameters: 매개변수 직접 수정 → 지역변수 복사 후 지역변수 수정.
매개변수는 호출자가 넘긴 값이므로 메서드 내부에서 변경하면 의도가 모호해지고 pass-by-reference 언어에서는 부작용이 발생할 수 있다.
이 기법으로 메서드의 의도가 명확해지고 부작용이 방지된다.
